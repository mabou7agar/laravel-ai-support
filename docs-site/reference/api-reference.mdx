The Mintlify `API` tab now renders the current OpenAPI reference for the v1 endpoints and the federation contract.

Use this page for the route-family overview, compatibility notes, and legacy endpoints that are intentionally kept outside the generated OpenAPI spec.

## API Surface at a Glance

There are three route families:

- app-facing v1 APIs (`/api/v1/*`) for product features
- legacy app APIs (`/ai-demo/*`, `/api/ai-chat/*`) for compatibility
- node federation contract (`/api/ai-engine/*`) for master/child communication

For PHP runtime usage, the parallel entrypoints are:

- `UnifiedEngineManager` / `Engine` facade for fluent application-facing calls
- `AIEngineService` for direct `AIRequest` execution

### Direct Generation (`/api/v1/ai/generate`)

- `POST /text`
- `POST /image`
- `POST /preview`
- `GET /preview/jobs/{jobId}`
- `POST /reference-pack`
- `GET /reference-pack/jobs/{jobId}`
- `POST /video`
- `GET /video/jobs/{jobId}`
- `POST /transcribe`
- `POST /tts`

Authenticated calls to these endpoints are credit-enforced (including image/audio).

Reference-pack specific payload fields:

- `look_id`: the app-selected approved look identifier
- `look_payload`: optional label/instruction overrides for that approved look
- `selected_looks`: ordered list of exact stored looks for deterministic multi-look packs
- `look_mode`: `strict_selected_set`, `strict_stored`, `guided`, or `vendor`
- `strict_stored_looks`: boolean alias for `look_mode=strict_stored`

Reference-pack mode defaults:

- no `look_id`: `vendor`
- `look_id` with no explicit mode: `guided`
- `selected_looks` with more than one item and no explicit mode: `strict_selected_set`
- `strict_stored_looks=true`: `strict_stored`

Use `strict_stored` for one approved canonical look. Use `strict_selected_set` when one pack must cover multiple approved stored looks in exact order. Use `guided` or `vendor` when exploration is the goal.

Video generation accepts Seedance motion reference fields:

- `reference_video_urls`: up to 3 reference video URLs, forwarded to fal.ai as `video_urls`
- `animation_reference_url`: single-video alias for `reference_video_urls`
- `animation_reference_urls`: multi-video alias for `reference_video_urls`
- `reference_audio_urls`: up to 3 reference audio URLs, forwarded as `audio_urls`; fal.ai requires at least one image or video reference when audio is provided

### AI Catalog (`/api/v1/ai`)

- `GET /models`
- `GET /models?engine=openai`
- `GET /engines`
- `GET /engines-with-models`

Use these endpoints for model pickers, provider-aware client defaults, and admin tooling. A starter Bruno collection lives in `bruno/laravel-ai-engine`.

### Data Collectors

- Guided: `/api/v1/data-collector/*`
- Autonomous: `/api/v1/autonomous-collector/*`

Full contracts are documented in `reference/data-collector-api`.

## Node Federation Contract (`/api/ai-engine`)

- `GET /health` (public)
- `GET /manifest` (public)
- `POST /search` (protected, node-to-node)
- `POST /chat` (protected, node-to-node)
- `POST /tools/execute` (protected, node-to-node)

Use this contract for infrastructure, not frontend apps.

## Example: Strict Selected Set Reference Pack Request

`POST /api/v1/ai/generate/reference-pack`

```json
{
  "prompt": "Front-facing cinematic portrait of Mina, full body, clean background",
  "entity_type": "character",
  "name": "Mina Scene Pack",
  "save_as": "mina-scene-pack",
  "frame_count": 8,
  "look_mode": "strict_selected_set",
  "selected_looks": [
    {
      "id": "business-street-look",
      "name": "Commercial Street Business Look",
      "instruction": "Keep the charcoal business coat, white blouse, and practical day makeup locked.",
      "scene_context": "Scene 12 exterior day",
      "is_primary": true
    },
    {
      "id": "airport-disguise",
      "name": "Airport Security Disguise",
      "instruction": "Keep the cap, oversized trench, muted palette, and surveillance-avoidance styling locked.",
      "scene_context": "Scene 14 terminal surveillance"
    }
  ]
}
```

The package response metadata now distinguishes:

- `selected_looks`: what the app requested
- `looks`: the grouped looks that were actually generated
- `look_count`: how many stored looks were requested
- `frames_per_look`: the target view allocation per look

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/reference-pack \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt":"Front-facing cinematic portrait of Mina, full body, clean background",
    "entity_type":"character",
    "name":"Mina Production",
    "save_as":"mina-production",
    "frame_count":8,
    "look_id":"wardrobe_scene_12_a",
    "look_payload":{
      "label":"Scene 12A hero wardrobe",
      "instruction":"Keep the navy coat, silver jewelry, natural makeup, and pinned-back hair exactly consistent across all views."
    },
    "look_mode":"strict_stored"
  }'
```

## Standard Response Envelope

```json
{
  "success": true,
  "message": "Request completed.",
  "data": {
    "response": "..."
  },
  "error": null,
  "meta": {
    "status_code": 200,
    "schema": "ai-engine.v1"
  }
}
```

## Compatibility Flags

```env
AI_ENGINE_STANDARDIZE_API_RESPONSES=true
AI_ENGINE_API_RESPONSE_PRESERVE_LEGACY=true
```

When legacy-preserve is enabled, top-level legacy keys may still appear for backward compatibility.
