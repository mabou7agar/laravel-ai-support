## API Surface at a Glance

There are three route families:

- app-facing v1 APIs (`/api/v1/*`) for product features
- legacy app APIs (`/ai-demo/*`, `/api/ai-chat/*`) for compatibility
- node federation contract (`/api/ai-engine/*`) for master/child communication

## App-Facing v1 Endpoints

### RAG Chat (`/api/v1/rag`)

- `POST /chat`
- `GET /chat/history/{session_id}`
- `POST /chat/clear`
- `POST /analyze-file`
- `GET /collections`
- `GET /engines`
- `GET /health`
- `GET /conversations`

### Actions (`/api/v1/actions`)

- `POST /execute`
- `POST /execute-remote`
- `POST /execute-all`
- `POST /select-option`
- `GET /available`

### Modules (`/api/v1/modules`)

- `GET /discover`

### Direct Generation (`/api/v1/ai/generate`)

- `POST /text`
- `POST /image`
- `POST /transcribe`
- `POST /tts`

Authenticated calls to these endpoints are credit-enforced (including image/audio).

### Data Collectors

- Guided: `/api/v1/data-collector/*`
- Autonomous: `/api/v1/autonomous-collector/*`

Full contracts are documented in `reference/data-collector-api`.

## Node Federation Contract (`/api/ai-engine`)

- `GET /health` (public)
- `GET /manifest` (public)
- `POST /search` (protected, node-to-node)
- `POST /chat` (protected, node-to-node)
- `POST /tools/execute` (protected, node-to-node)

Use this contract for infrastructure, not frontend apps.

## Legacy Compatibility Routes

### AI Demo

- `POST /ai-demo/chat/send`
- `GET /ai-demo/chat/history/{sessionId}`
- `POST /ai-demo/chat/clear`
- `POST /ai-demo/chat/upload`
- `POST /ai-demo/chat/action`
- `POST /ai-demo/chat/execute-action`
- `GET /ai-demo/chat/engines`
- `GET /ai-demo/chat/actions`

### Legacy AI Chat

- `POST /api/ai-chat/send`
- `POST /api/ai-chat/action`
- `POST /api/ai-chat/execute-action`
- `GET /api/ai-chat/history/{session_id}`
- `DELETE /api/ai-chat/history/{session_id}`
- `GET /api/ai-chat/engines`

## Example: Chat Request

`POST /api/v1/rag/chat`

```json
{
  "message": "list invoices",
  "session_id": "session-123",
  "engine": "openai",
  "model": "gpt-4o-mini",
  "memory": true,
  "actions": true,
  "use_intelligent_rag": true
}
```

```bash
curl -X POST https://your-app.test/api/v1/rag/chat \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "message":"list invoices",
    "session_id":"api-ref-1001",
    "memory":true,
    "actions":true,
    "use_intelligent_rag":true
  }'
```

## Standard Response Envelope

```json
{
  "success": true,
  "message": "Request completed.",
  "data": {
    "response": "..."
  },
  "error": null,
  "meta": {
    "status_code": 200,
    "schema": "ai-engine.v1"
  }
}
```

## Compatibility Flags

```env
AI_ENGINE_STANDARDIZE_API_RESPONSES=true
AI_ENGINE_API_RESPONSE_PRESERVE_LEGACY=true
```

When legacy-preserve is enabled, top-level legacy keys may still appear for backward compatibility.
