## Step-by-Step

1. update package version
2. publish latest config and migrations
3. clear cache
4. run node/agent smoke tests

```bash
php artisan vendor:publish --tag=ai-engine-config --force
php artisan vendor:publish --tag=ai-engine-migrations --force
php artisan migrate
php artisan optimize:clear
php artisan ai-engine:test-package
php artisan ai-engine:test-nodes --quick
php artisan ai-engine:test-real-agent --script=minimal --json
```

## Breaking Changes

The legacy runtime pair below was removed:

- `LaravelAIEngine\Services\AIEngineManager`
- `LaravelAIEngine\Services\EngineBuilder`

Use the new runtime shape instead:

- `LaravelAIEngine\Services\UnifiedEngineManager` for fluent application-facing usage
- `LaravelAIEngine\Services\EngineProxy` for the object returned by `engine()` and `model()`
- `LaravelAIEngine\Services\AIEngineService` for direct typed execution with `AIRequest`

Common replacements:

- `app('ai-engine')` now resolves `UnifiedEngineManager`
- `AIEngine::engine('openai')->model('gpt-4o')...` still works and now returns `EngineProxy`
- constructor or service-container references to `AIEngineManager` should be replaced with `UnifiedEngineManager` or `AIEngineService`, depending on whether you need fluent orchestration or direct execution

## New Data Tables To Expect

Recent refactors introduce these package tables:

- `ai_entity_summaries`
- `ai_prompt_feedback_events`
- `ai_prompt_policy_versions`
- `ai_action_metrics`

If they are missing, re-run published migrations.

## If You Use Federation

```bash
php artisan ai-engine:nodes-sync --file=config/ai-engine-nodes.json
php artisan ai-engine:nodes-sync --file=config/ai-engine-nodes.json --autofix
php artisan ai-engine:nodes-sync --file=config/ai-engine-nodes.json --apply --prune --ping --force
php artisan ai-engine:node-list
```

## If You Have Stale Nodes

```bash
php artisan ai-engine:node-cleanup --status=error --days=0 --apply --force
```

## If You Are Moving To Central Neo4j GraphRAG

Recommended sequence:

1. enable graph publishing with graph reads still off
2. initialize and sync Neo4j
3. diagnose graph integrity
4. switch read preference to central graph
5. enable KB and schedule `neo4j-kb-build`

See [Qdrant to Neo4j Migration](/reference/qdrant-to-neo4j-migration).

## Backward Compatibility Notes

- `default_engine` remains as legacy alias
- canonical engine key is `ai-engine.default`
- canonical model key is `ai-engine.default_model`
