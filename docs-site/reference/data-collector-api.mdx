## Guided Base Path

`/api/v1/data-collector`

Middleware stack:

- `api`
- `SetRequestLocaleMiddleware`
- `StandardizeApiResponseMiddleware`

## `POST /start`

Start session using a pre-registered config.

Request:

```json
{
  "config_name": "invoice_create",
  "session_id": "dc-1001",
  "initial_data": {
    "customer_name": "ACME LLC"
  }
}
```

Success payload includes:

- `session_id`
- `message`
- `actions`
- `metadata`
- flattened shortcuts: `config_name`, `fields`, `current_field`, `collected_fields`, `remaining_fields`, `progress`, `data`, `config`, `field_definitions`

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/data-collector/start \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "config_name":"invoice_create",
    "session_id":"dc-ref-1001"
  }'
```

## `POST /start-custom`

Start session with inline config definition.

Key request fields:

- `title` (required)
- `fields` (required)
- `description`
- `session_id`
- `initial_data`
- `confirm_before_complete`
- `allow_enhancement`
- `allow_skip_optional`
- `language`
- `detect_locale`
- `system_prompt`
- `action_summary`, `action_summary_prompt`, `action_summary_prompt_config`
- `output_schema`, `output_prompt`, `output_config`
- `on_complete_action`
- `metadata`

`fields` accepts **both** formats:

1) associative map (recommended):

```json
{
  "customer_name": "Customer full name | required | min:2",
  "issue_date": {
    "type": "string",
    "description": "Issue date",
    "validation": "required|date"
  }
}
```

2) indexed list (useful for API docs "Try it"):

```json
[
  {
    "name": "customer_name",
    "description": "Customer full name",
    "validation": "required|min:2"
  },
  {
    "name": "issue_date",
    "description": "Issue date",
    "validation": "required|date"
  }
]
```

## `POST /message`

Continue an active session.

Request:

```json
{
  "session_id": "dc-1001",
  "message": "Issue date is 2026-03-09",
  "engine": "openai",
  "model": "gpt-4o"
}
```

Response includes collector state keys:

- `status`
- `is_complete`, `is_cancelled`
- `requires_confirmation`, `allows_enhancement`
- `current_field`, `fields`, `progress`
- `collected_fields`, `remaining_fields`
- `summary`, `action_summary`
- `generated_output` (when output schema generation is used)

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/data-collector/message \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "session_id":"dc-ref-1001",
    "message":"Customer is ACME and issue date is 2026-03-09"
  }'
```

## `GET /status/{sessionId}`

Returns raw session state snapshot:

- `status`
- `data`
- `current_field`
- `is_complete`, `is_cancelled`
- `validation_errors`

Returns `404` when session is missing.

## `POST /cancel`

Cancel active session.

Request:

```json
{
  "session_id": "dc-1001"
}
```

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/data-collector/cancel \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"session_id":"dc-ref-1001"}'
```

## `GET /data/{sessionId}`

Get only collected data for session.

Returns `404` when session is unknown.

## `POST /analyze-file`

Extract structured values from uploaded file.

Form fields:

- `file` (required; `pdf|txt|doc|docx`, max 10 MB)
- `session_id` (required)
- `fields` (JSON string)
- `field_config` (JSON string)
- `language`

## `POST /apply-extracted`

Apply extracted file data to existing session.

Request:

```json
{
  "session_id": "dc-1001",
  "extracted_data": {
    "customer_name": "ACME LLC",
    "issue_date": "2026-03-09"
  },
  "language": "en"
}
```

Returns updated assistant `message`, `actions`, `metadata`, and `requires_confirmation`.

## Autonomous Base Path

`/api/v1/autonomous-collector`

## `POST /start`

Start autonomous session from registered autonomous config.

Replace `config_name` with your actual registered autonomous collector key.

Request:

```json
{
  "config_name": "invoice_auto",
  "session_id": "ac-2001",
  "initial_message": "Create invoice for ACME with two keyboard items"
}
```

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/autonomous-collector/start \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "config_name":"invoice",
    "session_id":"ac-ref-2001",
    "initial_message":"Create invoice for ACME with 2 items"
  }'
```

## `POST /message`

Continue autonomous session.

```json
{
  "session_id": "ac-2001",
  "message": "Use customer id 88 and add shipping note"
}
```

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/autonomous-collector/message \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "session_id":"ac-ref-2001",
    "message":"yes, proceed"
  }'
```

## `GET /status/{sessionId}`

Returns current status if session exists.

## `POST /confirm`

Confirms and executes completion when status is `confirming`.

Copy-paste:

```bash
curl -X POST https://your-app.test/api/v1/autonomous-collector/confirm \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"session_id":"ac-ref-2001"}'
```

## `POST /cancel`

Cancels active autonomous session.

## `GET /data/{sessionId}`

Returns collected structured payload for current autonomous session.
