## Diagnostics

```bash
php artisan ai-engine:test-package
php artisan ai-engine:test-everything
php artisan ai-engine:test-everything --profile=graph
php artisan ai-engine:test-everything --profile=all --root-path=/path/to/root/app
php artisan ai-engine:test-orchestration
php artisan ai-engine:test-orchestration --no-phpunit --json
php artisan ai-engine:backend-status
php artisan ai-engine:model-status "App\\Models\\Project"
php artisan ai-engine:model-status "App\\Models\\Project" --id=15
php artisan ai-engine:test-engines
php artisan ai-engine:infra-health
php artisan ai-engine:system-health
```

Profiles:

- `safe`: package graph and chat unit/command slices, plus root mocked chat route when a host app is available
- `graph`: safe plus live Neo4j graph package checks
- `full`: graph plus root-app live graph/chat checks
- `all`: full plus billed live provider matrix

Use `--stop-on-failure` to halt after the first failed stage and `--json` for machine-readable summaries.

`ai-engine:test-orchestration` validates registered tools, sub-agents, skill links, and graph complexity. By default it also runs the focused package orchestration PHPUnit stage. Use `--max-complexity=120` to override the graph complexity limit and `--fail-on-warning` when warnings should fail CI.

Use `php artisan ai-engine:backend-status` to see whether the app is currently reading from `neo4j_graph` or falling back to `vector_qdrant`.
Use `php artisan ai-engine:model-status "App\\Models\\Project"` to inspect whether one model is actually ready for indexing, graph publishing, and chat retrieval. Add `--id=15` to inspect a real record instead of a blank model instance, and use `--json` if you want the raw diagnostic payload.

Safe workflow:

1. run `nodes-sync` without `--apply` (dry-run plan)
2. optionally run with `--autofix`
3. apply with `--apply --prune --ping --force` once plan is clean

## Vector and RAG

```bash
php artisan ai-engine:sync-models --provider=openai
php artisan ai-engine:sync-models --provider=fal_ai
php artisan ai-engine:sync-models --provider=all
php artisan ai-engine:list-models
php artisan ai-engine:analyze-model "App\\Models\\Invoice"
php artisan ai-engine:vector-index "App\\Models\\Invoice" --with-relationships
php artisan ai-engine:vector-search "overdue invoices"
php artisan ai-engine:vector-status
```

`ai-engine:analyze-model` is read-only. It inspects schema, model relationships, and approximate table size to suggest indexing config, but it does not insert, update, delete, or index records.

## Neo4j GraphRAG

```bash
php artisan ai-engine:neo4j-init
php artisan ai-engine:neo4j-sync --fresh
php artisan ai-engine:neo4j-stats
php artisan ai-engine:neo4j-diagnose
php artisan ai-engine:neo4j-repair --apply
php artisan ai-engine:neo4j-drift --repair --prune
php artisan ai-engine:neo4j-benchmark "who owns Apollo?" --iterations=5
php artisan ai-engine:neo4j-index-benchmark "App\\Models\\Project" --limit=10
php artisan ai-engine:neo4j-load-benchmark --profile=steady
php artisan ai-engine:neo4j-load-benchmark --mode=mixed --iterations=50 --concurrency=4
php artisan ai-engine:chat-benchmark "What changed for Apollo?" --iterations=3
php artisan ai-engine:benchmark-history --type=retrieval --limit=10
php artisan ai-engine:graph-ranking-feedback relationship
```

Typical rollout:

1. initialize schema with `neo4j-init`
2. bulk publish with `neo4j-sync --fresh`
3. inspect with `neo4j-stats` and `neo4j-diagnose`
4. repair if needed with `neo4j-repair --apply`
5. detect and repair graph drift with `neo4j-drift`
6. benchmark indexing, retrieval, chat, and load latency with `neo4j-index-benchmark`, `neo4j-benchmark`, `chat-benchmark`, and `neo4j-load-benchmark`
7. inspect recent benchmark trends with `benchmark-history` and ranking adaptation state with `graph-ranking-feedback`

## Shared Neo4j Notes

For shared Neo4j databases, prefer dedicated vector property/index names so one app does not collide with another app's embedding dimensions.

Example:

```bash
php artisan ai-engine:neo4j-init \
  --index=chunk_embedding_index \
  --property=embedding \
  --scope-node=billing_app
```

If `AI_ENGINE_NEO4J_SHARED_DEPLOYMENT=true`, the package defaults to node-scoped naming and derives names like:

- `chunk_embedding_index_billing_app`
- `embedding_billing_app`

`neo4j-load-benchmark` also supports reusable profiles:

- `smoke`
- `interactive`
- `steady`
- `burst`
- `stress`

## Reference Pack Workflows

Use `generate-character` when the entity is a reusable person/performer. Use `generate-reference-pack` for furniture, vehicles, props, products, creatures, or mixed non-character workflows.

Mode guidance:

- `--look-mode=strict_selected_set`: deterministic multi-look pack from an exact ordered stored look set
- `--look-mode=strict_stored`: production reference sheets from an app-approved look
- `--look-mode=guided`: start from an approved look, then allow exploratory vendor variants
- `--look-mode=vendor`: fully exploratory vendor-driven look progression
- `--strict-stored-looks`: boolean shorthand for strict production mode
- `--look-set='[...]'`: JSON array of exact stored looks for multi-look strict packs

Examples:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina-production \
  --frame-count=8 \
  --look-id=wardrobe_scene_12_a \
  --look-payload='{"label":"Scene 12A hero wardrobe","instruction":"Keep the navy coat, silver jewelry, natural makeup, and pinned-back hair exactly consistent across all views."}' \
  --look-mode=strict_stored
```

```bash
php artisan ai-engine:generate-reference-pack \
  "Modern curved couch, premium fabric, clean studio background" \
  --entity-type=furniture \
  --save-as=modern-couch-production \
  --frame-count=6 \
  --look-id=sku_4432_cream_boucle \
  --strict-stored-looks
```

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina-scene-pack \
  --frame-count=8 \
  --look-mode=strict_selected_set \
  --look-set='[
    {"id":"business-street-look","name":"Commercial Street Business Look","instruction":"Keep the commercial business wardrobe locked.","is_primary":true},
    {"id":"airport-disguise","name":"Airport Security Disguise","instruction":"Keep the airport disguise wardrobe locked."}
  ]'
```
