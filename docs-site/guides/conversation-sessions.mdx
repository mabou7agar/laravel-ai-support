Use the conversations endpoint when you need a chat inbox or history screen.

Endpoint:

- `GET /api/v1/rag/conversations`

This endpoint is authenticated and returns only the conversations that belong to the current authenticated user.

## What You Get

Each conversation item includes:

- `conversation_id`
- `title`
- `summary`
- `message_count`
- `last_message`
- `last_activity_at`
- `created_at`
- `settings`

The `summary` field is the right field for chat list screens.

It is resolved in this order:

1. stored summary in conversation metadata if available
2. latest assistant message
3. first user message
4. latest message fallback

## Long-Running Chat Context

Agent sessions support context compaction. When a conversation grows beyond the configured limits, the package keeps the newest messages in `conversation_history` and folds older turns into `metadata.conversation_summary`.

That compact summary is reused by:

- conversational responses
- intent routing
- autonomous RAG context building
- later compaction passes

This lets a user continue a workflow naturally without sending the whole historical transcript to the model. The compact summary is conversation memory, not business-record memory. Invoices, customers, products, and other domain data should still be read through deterministic tools, RAG indexes, or graph queries.

Tune the behavior with:

```env
AI_AGENT_CONTEXT_COMPACTION_ENABLED=true
AI_AGENT_CONTEXT_MAX_MESSAGES=12
AI_AGENT_CONTEXT_KEEP_RECENT_MESSAGES=6
AI_AGENT_CONTEXT_MAX_MESSAGE_CHARS=2000
AI_AGENT_CONTEXT_MAX_TOTAL_CHARS=12000
AI_AGENT_CONTEXT_MAX_SUMMARY_CHARS=4000
AI_AGENT_CONTEXT_SUMMARY_MESSAGE_CHARS=240
```

## Flutter Usage

```dart
final result = await client.getConversations();

for (final item in result.conversations) {
  print(item.title);
  print(item.summary);
}
```

## Mintlify API Base URL

The Mintlify `API` tab uses the `servers` block from:

- `openapi/ai-engine.openapi.yaml`

To point the interactive API docs at a different backend:

1. edit the `servers` list in the OpenAPI file
2. set your test or local base URL there
3. reload the Mintlify docs

Current test-friendly defaults are:

- `http://ai.test`
- `http://127.0.0.1:8000`

If your app runs somewhere else, replace those values with your real host.
