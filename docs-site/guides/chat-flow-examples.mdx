## Example 1: List + Follow-Up (No Re-list Loop)

### User

`list recent records`

### Expected system behavior

- tool executes the configured list/search query
- result gets cached in conversation context/session state

### User follow-up

`what is the status of the first one?`

### Expected behavior

- resolve from selected entity/context if available
- avoid re-running full list unless needed

## Example 2: Missing Fields -> Collector

### User

`create return request for Acme`

### Expected behavior

- identify missing required fields from the app's action schema
- start collector flow
- ask only for missing fields
- call the configured write tool after completion

## Example 3: Local or Remote Ownership

### User

`create chart of account`

### Expected behavior

- ownership resolver checks if local or child node owns domain
- if remote: route to owning node and return normalized response

## Example 4: Authenticated User Personal Context

### User

`what is my name`

### Expected behavior

- when `AI_ENGINE_INJECT_USER_CONTEXT=true` and request is authenticated, AI can answer from injected user context
- no custom hardcoded rule is required in app code
- response language follows resolved locale

## Example 5: Confirmation + Follow-Up Continuity

### User

`create return request for Alpha Laptop`

### Expected behavior

- system asks for missing fields and confirms before write action
- follow-up like `yes` should continue pending action, not restart from scratch
- if clarification is needed, continue in the same pending flow context
