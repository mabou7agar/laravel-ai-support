This page gives exact, separate requests for direct generation.

## Built-in HTTP endpoints

- `POST /api/v1/ai/generate/text`
- `POST /api/v1/ai/generate/image`
- `POST /api/v1/ai/generate/transcribe`
- `POST /api/v1/ai/generate/tts`

All return standard envelope keys: `success`, `message`, `data`, `error`, `meta`.

## Credit behavior

- authenticated requests use the same credit guard/deduction as chat and RAG
- applies to all direct endpoints: text, image, transcribe, and tts
- when balance is insufficient, API returns HTTP `402` with standard error envelope

## 1) Text generation (HTTP)

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/text \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt":"Write a concise follow-up email for unpaid invoice #124",
    "engine":"openai",
    "model":"gpt-4o-mini",
    "temperature":0.2,
    "max_tokens":300
  }'
```

## 2) Image generation (HTTP)

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/image \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt":"Clean SaaS dashboard hero image, blue palette",
    "engine":"openai",
    "model":"dall-e-3",
    "count":1,
    "size":"1024x1024",
    "quality":"standard"
  }'
```

## 3) Audio transcription (HTTP)

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/transcribe \
  -H "Accept: application/json" \
  -F "file=@/absolute/path/to/meeting.m4a" \
  -F "engine=openai" \
  -F "model=whisper-1"
```

## 4) Text-to-speech (HTTP)

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/tts \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "text":"Welcome to your dashboard.",
    "engine":"eleven_labs",
    "model":"eleven_multilingual_v2",
    "voice_id":"pNInz6obpgDQGcFmaJgB"
  }'
```

## 5) SDK text generation (`AIRequest`)

Use this when you want application code instead of HTTP.

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;

$request = new AIRequest(
    prompt: 'Summarize this support ticket in 3 bullets.',
    engine: 'openai',
    model: 'gpt-4o-mini',
    parameters: ['temperature' => 0.1]
);

$response = app(AIEngineService::class)->generate($request);

if (!$response->isSuccess()) {
    throw new RuntimeException($response->getError() ?? 'Generation failed');
}

$text = $response->getContent();
```

## 6) SDK image generation (`AIRequest`)

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;

$request = new AIRequest(
    prompt: 'Minimal product icon, blue background',
    engine: 'openai',
    model: 'dall-e-3',
    parameters: [
        'image_count' => 1,
        'size' => '1024x1024',
        'quality' => 'standard',
    ]
);

$response = app(AIEngineService::class)->generate($request);
$files = $response->getFiles();
```

## 7) SDK transcription (`AIRequest`)

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;

$request = new AIRequest(
    prompt: 'Transcribe this audio file.',
    engine: 'openai',
    model: 'whisper-1',
    files: ['/absolute/path/to/meeting.m4a']
);

$response = app(AIEngineService::class)->generate($request);
$transcript = $response->getContent();
```

## 8) SDK text-to-speech (`AIRequest`)

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;

$request = new AIRequest(
    prompt: 'Welcome to your dashboard.',
    engine: 'eleven_labs',
    model: 'eleven_multilingual_v2',
    parameters: [
        'voice_id' => 'pNInz6obpgDQGcFmaJgB',
        'audio_minutes' => 1.0,
    ]
);

$response = app(AIEngineService::class)->generate($request);
$audioFiles = $response->getFiles();
```

## 9) Optional app-specific wrapper endpoints

If you need custom auth/rate limits/tenant policies, wrap these calls in your app controller and proxy to package generation.

## 10) Common mistakes

- engine/model mismatch (example: `engine=openai` with non-OpenAI model slug)
- missing provider API key (you will get provider-specific error in envelope)
- using JSON for `/transcribe` instead of multipart upload
- expecting `/api/v1/rag/chat` to behave like direct multimodal generation API
