This page gives exact, separate requests for direct generation.

## Which PHP API To Use

- use `UnifiedEngineManager` / `Engine` facade when you want fluent builder-style calls
- use `AIEngineService` when you want direct execution from an `AIRequest`
- both paths resolve drivers through the same `DriverRegistry`

If you omit `engine` and pass only a `model`, direct requests now resolve the provider from the model registry plus configured provider availability. By default this prefers the model's native provider before OpenRouter-style fallbacks.

For text requests, you can also omit both `engine` and `model` and send a `preference` such as `cost`, `speed`, `performance`, or `quality`.

## Built-in HTTP endpoints

- `POST /api/v1/ai/generate/text`
- `POST /api/v1/ai/generate/image`
- `POST /api/v1/ai/generate/video`
- `GET /api/v1/ai/generate/video/jobs/{jobId}`
- `POST /api/v1/ai/generate/transcribe`
- `POST /api/v1/ai/generate/tts`

All return standard envelope keys: `success`, `message`, `data`, `error`, `meta`.

## Built-in Console Commands

- `php artisan ai-engine:generate-character`
- `php artisan ai-engine:generate-reference-pack`
- `php artisan ai-engine:test-ai-media`
- `php artisan ai-engine:test-fal-media`

## Credit behavior

- authenticated requests use the same credit guard/deduction as chat and RAG
- applies to all direct endpoints: text, image, video, transcribe, and tts
- when balance is insufficient, API returns HTTP `402` with standard error envelope
- console FAL commands skip credits when `--user-id` is omitted
- console FAL commands charge credits normally when `--user-id` is passed

## AIMedia storage

Generated media can now be persisted in the `ai_media` table with medialibrary-style records.

Run migrations:

```bash
php artisan migrate
```

Use your own cloud disk URLs by setting a storage disk such as `s3`:

```env
AI_ENGINE_MEDIA_ENABLED=true
AI_ENGINE_MEDIA_DISK=s3
AI_ENGINE_MEDIA_DIRECTORY=ai-generated
AI_ENGINE_MEDIA_VISIBILITY=public
```

When upload succeeds, media responses return your disk URL instead of the provider URL, and a matching `ai_media` row is created.

Quick verification:

```bash
php artisan ai-engine:test-ai-media --write-test --cleanup
```

## 6) Async video submission (HTTP)

Use this for long-running Kling or Seedance jobs so your app gets a local `job_id` immediately and polls later.

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/video \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt":"A cinematic chase through a flooded cyberpunk alley",
    "engine":"fal_ai",
    "model":"bytedance/seedance-2.0/text-to-video",
    "duration":"auto",
    "resolution":"720p",
    "generate_audio":true,
    "async":true
  }'
```

Response shape:

- `data.job_id`: local package job id
- `data.status`: local normalized status like `queued`, `processing`, or `completed`
- `data.job.metadata.provider.request_id`: provider request id on FAL

## 7) Async video status (HTTP)

```bash
curl -X GET "https://your-app.test/api/v1/ai/generate/video/jobs/local-job-id?refresh=1" \
  -H "Accept: application/json"
```

Use `refresh=1` when you want the package to poll FAL before returning the local status snapshot.

## 9) Text-to-speech (HTTP)

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/tts \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "text":"Welcome to your dashboard.",
    "engine":"eleven_labs",
    "model":"eleven_multilingual_v2",
    "voice_id":"pNInz6obpgDQGcFmaJgB"
  }'
```

If you already saved a reusable character with voice metadata, you can keep the same voice consistently by referencing the character alias instead of sending `voice_id` each time:

```bash
curl -X POST https://your-app.test/api/v1/ai/generate/tts \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "text":"Welcome back, Mina.",
    "engine":"eleven_labs",
    "model":"eleven_multilingual_v2",
    "use_character":"hero-mina"
  }'
```

Attach the voice when saving the character:

```bash
php artisan ai-engine:generate-character "Front-facing portrait of Mina" \
  --name="Mina" \
  --save-as="hero-mina" \
  --voice-id="pNInz6obpgDQGcFmaJgB" \
  --voice-stability="0.35" \
  --voice-similarity-boost="0.75" \
  --voice-style="0.10" \
  --voice-speaker-boost="true" \
  --sync
```

## 10) SDK text generation (`AIRequest`)

Use this when you want application code instead of HTTP.

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;

$request = new AIRequest(
    prompt: 'Summarize this support ticket in 3 bullets.',
    engine: 'openai',
    model: 'gpt-4o-mini',
    parameters: ['temperature' => 0.1]
);

$response = app(AIEngineService::class)->generate($request);

if (!$response->isSuccess()) {
    throw new RuntimeException($response->getError() ?? 'Generation failed');
}

$text = $response->getContent();
```

## 13) Console character generation

Generate a reusable Nano Banana character set and save it for later Kling/Seedance scenes.

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina \
  --frame-count=8 \
  --look-size=4 \
  --aspect-ratio=9:16 \
  --user-id=123
```

Character/reference-pack commands queue by default. Add `--wait` if you want the command to block until the queued job finishes, or `--sync` only when you explicitly want inline execution.

If you want to approve the identity first, generate only the preview/base image:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina-preview \
  --preview-only
```

Then expand that approved preview into the full reference set:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina \
  --from-character=mina-preview \
  --frame-count=16 \
  --look-size=4
```

`--frame-count` now builds a character sheet workflow:

- views are grouped into looks based on `--look-size`
- the first look keeps the same clothing, hair, makeup, and overall styling across all angles
- the next look keeps the same identity but intentionally changes hair and makeup styling
- each additional look continues introducing a new style variant while preserving the same identity

Reference-pack workflows now support explicit look modes:

- `vendor`: no stored/app-selected look is enforced; good for concept exploration and ideation
- `guided`: use the stored/app-selected look for the first look, then allow vendor-generated variants after that
- `strict_stored`: keep only the stored/app-selected look across all generated views; no vendor-invented beauty/fashion/material variants

Default behavior:

- no `look_id`: `vendor`
- `look_id` present with no mode: `guided`
- `strict_stored_looks=true`: shorthand for `look_mode=strict_stored`

Recommended usage:

- screenplay-driven characters, wardrobe, and makeup sheets: `strict_stored`
- art-direction exploration from a script-selected base look: `guided`
- moodboards, campaign discovery, and broad ideation: `vendor`

Production reference example with one script-derived look held constant across views:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina-production \
  --frame-count=8 \
  --look-id=wardrobe_scene_12_a \
  --look-payload='{"label":"Scene 12A hero wardrobe","instruction":"Keep the navy coat, silver jewelry, natural makeup, and pinned-back hair exactly consistent across all views."}' \
  --look-mode=strict_stored
```

Exploration example that starts from an approved look but still allows downstream vendor variants:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina-guided \
  --frame-count=8 \
  --look-size=4 \
  --look-id=wardrobe_scene_12_a \
  --look-payload='{"label":"Scene 12A hero wardrobe","instruction":"Start from the approved navy coat and natural makeup look."}' \
  --look-mode=guided
```

This stores:

- first generated image as `frontal_image_url`
- remaining generated images as `reference_image_urls`
- grouped look metadata in `character.metadata.looks`

Then reuse it later with `--use-character=mina` or `--use-last-character`.

## 13b) Console generic reference pack generation

The generic service/command can create reference packs for objects, furniture, vehicles, products, props, and creatures too.

```bash
php artisan ai-engine:generate-reference-pack \
  "Modern curved couch, premium fabric, clean studio background" \
  --entity-type=furniture \
  --name="Modern Couch" \
  --save-as=modern-couch \
  --frame-count=8 \
  --look-size=4
```

Preview-first flow also works for non-character entities:

```bash
php artisan ai-engine:generate-reference-pack \
  "Modern curved couch, premium fabric, clean studio background" \
  --entity-type=furniture \
  --save-as=modern-couch-preview \
  --preview-only
```

If you need to force inline execution during local debugging:

```bash
php artisan ai-engine:generate-reference-pack \
  "Modern curved couch, premium fabric, clean studio background" \
  --entity-type=furniture \
  --save-as=modern-couch-preview \
  --preview-only \
  --sync
```

Strict stored mode also works for generic reference packs when the host app already chose the approved look/material direction:

```bash
php artisan ai-engine:generate-reference-pack \
  "Modern curved couch, premium fabric, clean studio background" \
  --entity-type=furniture \
  --save-as=modern-couch-production \
  --frame-count=6 \
  --look-id=sku_4432_cream_boucle \
  --look-payload='{"label":"Cream boucle hero finish","instruction":"Keep the cream boucle upholstery, walnut legs, and approved silhouette exactly fixed across every view."}' \
  --strict-stored-looks
```

Queue a long character workflow instead of waiting in the terminal:

```bash
php artisan ai-engine:generate-character \
  "Front-facing cinematic portrait of Mina, full body, clean background" \
  --name=Mina \
  --save-as=mina \
  --frame-count=16 \
  --look-size=4 \
  --async
```

Check local queued workflow status:

```bash
php artisan ai-engine:generate-character --job-status=YOUR_JOB_ID
```

## 15) Console multi-character and multi-shot Kling scene

Repeat `--use-character`, `--character`, `--reference-image-url`, and `--shot`.

- first character becomes `@Element1`
- second character becomes `@Element2`
- first reference image becomes `@Image1`

```bash
php artisan ai-engine:test-fal-media \
  --use-character=mina \
  --use-character=ray \
  --reference-image-url=https://cdn.example.com/cafe-scene.png \
  --shot='{"prompt":"@Element1 enters beside @Element2. Match @Image1 lighting.","duration":3}' \
  --shot='{"prompt":"@Element1 sits across from @Element2 and looks toward camera.","duration":5}' \
  --user-id=123
```

## 16) Console async video submission

Use `--async` to enqueue a long-running FAL video job and print a local `job_id`.

```bash

  "A cinematic chase through a flooded cyberpunk alley" \
  --model=bytedance/seedance-2.0/text-to-video \
  --resolution=720p \
```

Wait locally by polling the job tracker:

```bash
php artisan ai-engine:test-fal-media \
  "A cinematic chase through a flooded cyberpunk alley" \
  --model=bytedance/seedance-2.0/text-to-video \
  --resolution=720p \
  --async \
  --wait \
  --timeout=300 \
  --poll-interval=5
```

Inspect an existing async job:

```bash
php artisan ai-engine:test-fal-media \
  --job-status=local-job-id \
  --refresh-status
```

If you want the same saved character voice over multiple calls, resolve the stored profile once and feed it into the request parameters:

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Services\AIEngineService;
use LaravelAIEngine\Support\Fal\FalCharacterStore;

$voice = app(FalCharacterStore::class)->voiceProfile('hero-mina');

$request = new AIRequest(
    prompt: 'Welcome back, Mina.',
    engine: 'eleven_labs',
    model: 'eleven_multilingual_v2',
    parameters: array_merge($voice ?? [], [
        'audio_minutes' => 1.0,
    ])
);

$response = app(AIEngineService::class)->generate($request);
$audioFiles = $response->getFiles();
```

## 21) Common mistakes

- engine/model mismatch (example: `engine=openai` with non-OpenAI model slug)
- missing provider API key (you will get provider-specific error in envelope)
- forgetting `start_image_url` or references for image-driven video models
- using synchronous video generation for jobs that can reasonably exceed request timeouts
- sending character source metadata without the required image URLs for character-consistent video
- forgetting `--user-id` in console flows when credits must be deducted from a real user
- assuming `frame_count` means video frames; here it means generated still images/views for Nano Banana
- using JSON for `/transcribe` instead of multipart upload
- expecting `/api/v1/rag/chat` to behave like direct multimodal generation API
