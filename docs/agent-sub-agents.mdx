# Agent Goals and Sub-Agents

The package includes a reusable goal-agent runtime for agentic workflows that need more than one step.

Use it when a request has a target and should be split across specialized handlers such as research, retrieval, validation, writing, auditing, or execution.

## Configure sub-agents

Register sub-agents in `config/ai-agent.php`:

```php
'sub_agents' => [
    'research' => [
        'name' => 'Research sub-agent',
        'description' => 'Finds relevant facts and source context.',
        'capabilities' => ['research', 'find', 'source'],
        'handler' => \App\AI\SubAgents\ResearchSubAgent::class,
    ],
    'writer' => [
        'name' => 'Writer sub-agent',
        'description' => 'Turns prior sub-agent results into a final response.',
        'capabilities' => ['write', 'summarize', 'synthesize'],
        'handler' => \App\AI\SubAgents\WriterSubAgent::class,
    ],
    'tool_researcher' => [
        'name' => 'Tool-backed researcher',
        'description' => 'Runs registered tools without a custom handler.',
        'capabilities' => ['research'],
        'tools' => ['web_search', 'summarize_result'],
    ],
],
```

Handlers implement `LaravelAIEngine\Contracts\SubAgentHandler`:

```php
use LaravelAIEngine\Contracts\SubAgentHandler;
use LaravelAIEngine\DTOs\SubAgentResult;
use LaravelAIEngine\DTOs\SubAgentTask;
use LaravelAIEngine\DTOs\UnifiedActionContext;

class ResearchSubAgent implements SubAgentHandler
{
    public function handle(
        SubAgentTask $task,
        UnifiedActionContext $context,
        array $previousResults = [],
        array $options = []
    ): SubAgentResult {
        return SubAgentResult::success(
            taskId: $task->id,
            agentId: $task->agentId,
            message: 'Research complete.',
            data: ['facts' => []]
        );
    }
}
```

## Call through chat

Send `agent_goal=true` to `POST /api/v1/agent/chat` with a `target`. You can let the planner match configured sub-agents by capabilities, or pass an explicit sub-agent plan.

Use `POST /api/v1/agent/chat` for new agentic flows.

```json
{
  "message": "Create a launch plan for the new product",
  "session_id": "launch-001",
  "agent_goal": true,
  "target": "Create a launch plan for the new product",
  "sub_agents": [
    {
      "id": "research_task",
      "agent_id": "research",
      "objective": "Collect launch facts and constraints"
    },
    {
      "id": "write_task",
      "agent_id": "writer",
      "objective": "Write the final launch plan",
      "depends_on": ["research_task"]
    }
  ]
}
```

The response uses the normal agent envelope. `metadata.goal_agent` contains the target, generated plan, and each sub-agent result.

## Safety model

Keep write operations inside app services. Sub-agent handlers should call repositories/services, enforce permissions, and return `SubAgentResult::needsUserInput()` when confirmation or missing data is required.

For destructive or high-risk actions, use the existing action framework and confirmation flow rather than writing directly from a sub-agent handler.

## Link agents, tools, and sub-agents

Sub-agents can be linked without writing another handler class:

- Add `tools` to a sub-agent definition to execute registered `AgentTool` implementations through the built-in tool-calling sub-agent handler.
- Add `sub_agents` or `delegates_to` to document delegation relationships and validate them.
- Use the built-in `run_sub_agent` tool when a tool-capable agent needs to delegate a target to a goal-agent plan.

```php
'sub_agents' => [
    'researcher' => [
        'name' => 'Researcher',
        'capabilities' => ['research'],
        'tools' => ['lookup_customer', 'search_options'],
    ],
    'writer' => [
        'name' => 'Writer',
        'capabilities' => ['write'],
        'sub_agents' => ['researcher'],
        'handler' => \App\AI\SubAgents\WriterSubAgent::class,
    ],
],
```

Validate the graph and run focused orchestration tests:

```bash
php artisan ai:test-orchestration
php artisan ai:test-orchestration --no-phpunit --json
php artisan ai:test-orchestration --max-complexity=120 --fail-on-warning
```

The command reports missing tool links, missing sub-agent links, and graph complexity before optionally running the focused agent orchestration PHPUnit stage.
