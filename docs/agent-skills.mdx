# Agent Skills

Skills are a lightweight orchestration layer above existing agent primitives.

A tool is an atomic callable operation. A skill is a user-facing ability that describes when to use tools, actions, workflows, required data, and confirmation rules.

## Structure

```php
use LaravelAIEngine\DTOs\AgentSkillDefinition;

new AgentSkillDefinition(
    id: 'create_invoice',
    name: 'Create Invoice',
    description: 'Create invoices through approved services and actions.',
    triggers: ['create invoice', 'new invoice'],
    requiredData: ['customer_id', 'items'],
    tools: ['find_customer', 'calculate_totals'],
    actions: ['invoices.create'],
    workflows: [App\AI\Workflows\CreateInvoiceWorkflow::class],
    capabilities: ['invoice', 'create'],
    requiresConfirmation: true,
    enabled: true,
);
```

Keep business writes in host application services and repositories. A skill should reference an action or workflow that validates, authorizes, confirms, and executes the write.

## Scaffold a skill

```bash
php artisan ai-engine:scaffold skill CreateInvoice
```

This creates an `App\AI\Skills\CreateInvoiceSkill` provider and registers it in the agent manifest under `skill_providers`.

Generated skills are disabled by default and marked as review-required.

## Scan project abilities

```bash
php artisan ai-engine:skills:discover
php artisan ai-engine:skills:discover --json
php artisan ai-engine:skills:discover --write
```

The scanner proposes draft skills from registered actions, workflows, autonomous collectors, and model discovery metadata.

Use `--write` to add draft definitions to `app/AI/agent-manifest.php`. Draft skills are disabled unless `--enable` is passed.

## Test a skill match

```bash
php artisan ai-engine:skills:test "create invoice for ACME"
php artisan ai-engine:skills:test "create invoice for ACME" --include-disabled --json
```

The command shows the matched skill, score, trigger, and compiled execution plan. Action-backed skills compile to the existing `update_action_draft` tool so validation, confirmation, and execution stay inside the existing action workflow.

## Diagnose the manifest

```bash
php artisan ai-engine:manifest:doctor
php artisan ai-engine:manifest:doctor --json
```

The doctor reports disabled draft skills, non-executable skills, and skills that reference missing actions, tools, or workflows.

Use the orchestration validator when skills also link to sub-agents through `metadata.sub_agents`:

```bash
php artisan ai-engine:test-orchestration
```

It validates skill-to-tool and skill-to-sub-agent links, then can run the focused orchestration tests.

## Manifest entries

```php
return [
    'skill_providers' => [
        'create_invoice' => App\AI\Skills\CreateInvoiceSkill::class,
    ],

    'skills' => [
        'create_invoice' => [
            'name' => 'Create Invoice',
            'description' => 'Create invoices through approved actions.',
            'triggers' => ['create invoice', 'new invoice'],
            'required_data' => ['customer_id', 'items'],
            'actions' => ['invoices.create'],
            'requires_confirmation' => true,
            'enabled' => false,
        ],
    ],
];
```

## Routing

When `ai-agent.skills.expose_as_capabilities` is enabled, skills are exposed as capability documents through the existing capability registry. This lets semantic routing discover complete abilities without exposing every low-level tool directly.

When `ai-agent.skills.prefer_deterministic_matches` is enabled, exact or strong trigger matches are compiled before the LLM router runs. This keeps common business requests deterministic while still using the existing tools, actions, workflows, and RAG fallback paths.
