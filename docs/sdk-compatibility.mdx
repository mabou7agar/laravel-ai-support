# SDK Compatibility Primitives

Laravel AI Engine keeps its orchestration, RAG, GraphRAG, credit, and node-federation features independent from the official `laravel/ai` package so it can continue supporting older Laravel applications.

The package now exposes compatible extension points for common SDK workflows without requiring PHP 8.3 or Laravel 12:

```php
use LaravelAIEngine\Facades\Engine;
use LaravelAIEngine\Tools\Provider\WebSearch;

$response = Engine::engine('openai')
    ->model('gpt-4o')
    ->withTools([
        (new WebSearch())->max(3)->allow(['laravel.com']),
    ])
    ->generate('Find the relevant Laravel AI documentation.');
```

## Reranking

```php
$results = Engine::rerank('invoice automation', [
    'Create invoices from approved orders.',
    'Reset user passwords.',
]);
```

The default reranker is deterministic and local. Applications can wrap or replace `LaravelAIEngine\Services\SDK\RerankingService` when they need a provider-backed reranker.

## Provider Tools

Provider-style tools are serializable and can be passed through `EngineProxy::withTools()`:

- `LaravelAIEngine\Tools\Provider\WebSearch`
- `LaravelAIEngine\Tools\Provider\WebFetch`
- `LaravelAIEngine\Tools\Provider\FileSearch`
- `LaravelAIEngine\Tools\Provider\CodeInterpreter`
- `LaravelAIEngine\Tools\Provider\ComputerUse`
- `LaravelAIEngine\Tools\Provider\McpServer`
- `LaravelAIEngine\Tools\Provider\GoogleMapsGrounding`
- `LaravelAIEngine\Tools\Provider\ImageGenerationTool`
- `LaravelAIEngine\Tools\Provider\ToolSearch`
- `LaravelAIEngine\Tools\Provider\HostedShell`
- `LaravelAIEngine\Tools\Provider\ApplyPatchTool`
- `LaravelAIEngine\Tools\Provider\ProviderSkill`
- `LaravelAIEngine\Tools\SimilaritySearch`

Drivers receive these as request functions and keep custom functions on the existing function-calling path.
OpenAI, Anthropic, and Gemini drivers map supported provider tools to native provider payloads where possible, while custom functions remain on the existing function-calling path.

OpenAI mappings include web search, file search, code interpreter, computer use, remote MCP, image generation, tool search, hosted shell, apply patch, and provider skills. Gemini mappings include Google Search grounding, URL context, Google Maps grounding, file retrieval metadata, and code execution through the existing `CodeInterpreter` provider tool. Anthropic mappings include web search, web fetch, code execution, computer use, and remote MCP with the required beta headers returned by the mapper.

Provider-hosted tool calls are tracked through the provider tool lifecycle tables. Computer use, MCP servers, and code execution require approval by default; web search and file search are audited without a blocking approval unless an app marks the tool with `requires_approval`.

Generated files, provider file IDs, containers, and citations are recorded as hosted artifacts. Downloadable generated files are linked to `ai_media` when the media library is enabled. Remote artifact persistence rejects private/local URLs by default and can be constrained with configured MIME type, extension, and size limits.

Apps can expose approvals, audit history, continuations, and artifacts through the built-in `/api/v1/ai/provider-tools` API group. The same group includes direct and queued generic FAL catalog execution for models synced into `ai_models`.

Provider tool versions that require fast-moving beta headers are configurable. Anthropic defaults preserve the package behavior, and host apps can override the type/header without waiting for a package release:

```php
'provider_tools' => [
    'versions' => [
        'anthropic' => [
            'code_execution' => [
                'type' => 'code_execution_20250825',
                'beta_header' => 'code-execution-2025-08-25',
            ],
            'computer_use' => [
                'type' => 'computer_20250124',
                'beta_header' => 'computer-use-2025-01-24',
            ],
            'mcp_server' => [
                'beta_header' => 'mcp-client-2025-04-04',
            ],
        ],
    ],
],
```

## Provider Options and Response State

Use `AIRequest::withProviderOptions()` when a provider adds request fields before the package has a first-class method for them. Generic options apply to every provider; provider-specific options override them for one driver:

```php
$response = Engine::engine('openai')
    ->model('gpt-4o')
    ->withProviderOptions(['store' => true])
    ->withProviderOptions([
        'background' => true,
        'include' => ['reasoning.encrypted_content'],
        'remember_response' => true,
    ], 'openai')
    ->generate('Continue the analysis');
```

OpenAI Responses state can be reused by setting a conversation id and opting in:

```php
$first = AIRequest::make('Analyze this file', 'openai', 'gpt-4o')
    ->setConversationId('support-thread-123')
    ->withMetadata(['openai_responses_api' => true])
    ->withProviderOptions(['remember_response' => true], 'openai');

$next = AIRequest::make('Use the previous result', 'openai', 'gpt-4o')
    ->setConversationId('support-thread-123')
    ->withMetadata(['openai_responses_api' => true])
    ->withProviderOptions(['use_previous_response' => true], 'openai');
```

The package stores the last provider response id in cache under the configured TTL and sends it as `previous_response_id` only when `use_previous_response` is enabled.

## Structured Output

```php
$response = Engine::engine('openai')
    ->withStructuredOutput([
        'type' => 'object',
        'properties' => [
            'answer' => ['type' => 'string'],
        ],
        'required' => ['answer'],
        'additionalProperties' => false,
    ], name: 'answer_payload')
    ->generate('Return a short answer.');
```

`withStructuredOutput()` accepts a JSON schema array or `LaravelAIEngine\DTOs\StructuredOutputSchema`. OpenAI-compatible drivers emit a native `response_format` JSON schema payload; Gemini emits `responseMimeType` and `responseSchema`.

## Documents and Vector Stores

```php
use LaravelAIEngine\Files\Document;
use LaravelAIEngine\Facades\Engine;

$store = Engine::vectorStores()->create('Knowledge Base');

Engine::vectorStores()->add(
    $store['id'],
    Document::fromStorage('manual.pdf', 'local')
);
```

When the package migrations are loaded, vector stores and document references are persisted in `ai_vector_stores` and `ai_vector_store_documents`. If those tables are unavailable, the service falls back to its in-memory store for lightweight tests and custom host adapters.

Hosted provider storage is available through drivers:

```php
$file = Engine::fileStores()
    ->provider('openai')
    ->upload(Document::fromStorage('manual.pdf', 'local'));

$store = Engine::vectorStores()
    ->provider('openai')
    ->create('Support Docs');

Engine::vectorStores()
    ->provider('openai')
    ->add($store['id'], Document::fromPath('/tmp/manual.pdf'));
```

Local storage remains the default. Hosted provider drivers are opt-in to avoid vendor lock-in and unexpected storage costs.

For local package-managed ingestion, use the shared extractor plus local vector-store registry:

```bash
php artisan ai:docs-index /path/to/docs --recursive --store-name="Support Docs"
```

`ai:docs-index` extracts text through `DocumentService`, records document metadata in `ai_vector_store_documents`, and keeps tenant/workspace scope in metadata when passed with repeated `--metadata=key=value` options.

## Realtime, Tracing, and Evaluations

```php
use LaravelAIEngine\DTOs\RealtimeSessionConfig;

$session = Engine::realtime()->create(
    RealtimeSessionConfig::make('openai', 'gpt-4o-realtime-preview')
        ->withVoice('alloy')
        ->withAudioFormats('pcm16', 'pcm16')
        ->withTurnDetection(['type' => 'server_vad'])
        ->withTemperature(0.2)
        ->withMaxResponseOutputTokens(512)
);

$span = Engine::traces()->start('support-agent.run');
Engine::traces()->end($span, metadata: ['tokens' => 420]);

Engine::evaluations()->assertPassed(
    'contains source',
    'Answer [1]',
    assertion: fn (string $answer) => str_contains($answer, '[')
);
```

Realtime support returns provider-specific session descriptors so host applications can attach their own WebSocket/WebRTC clients. OpenAI descriptors use snake_case fields expected by the Realtime session API. Gemini Live descriptors use Gemini-style camelCase fields such as `responseModalities`, `systemInstruction`, and `generationConfig`.

`LaravelAIEngine\Services\SDK\RealtimeToolBrokerService` dispatches provider realtime tool calls through the package `ToolRegistry` and returns a structured result payload for the host WebSocket/WebRTC client to send back to the provider. Tools that require confirmation return `approval_required` instead of executing.

`LaravelAIEngine\Services\SDK\McpAppToolAdapter` exports registered tools and enabled skills in an MCP/App-compatible schema, and can call local tools through the same registry.

The same bridge is exposed through versioned HTTP endpoints for browser clients, MCP/App adapters, and hosted realtime brokers:

```bash
curl -H "Accept: application/json" \
  http://localhost:8000/api/v1/ai/mcp/tools

curl -X POST http://localhost:8000/api/v1/ai/mcp/tools/run_skill/call \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "arguments": {"message": "Draft a reply"},
    "session_id": "support-thread-123",
    "user_id": "user-1"
  }'

curl -X POST http://localhost:8000/api/v1/ai/realtime/tools/dispatch \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "event": {"id": "call_1", "name": "run_skill", "arguments": {"message": "Draft a reply"}},
    "session_id": "support-thread-123",
    "user_id": "user-1",
    "approved": false
  }'
```

Use `withProviderOptions()` for provider-specific fields that the package should pass through without needing a package release. Tracing and evaluation services provide package-level recording primitives that can be exported to a provider or observability backend by registering `LaravelAIEngine\Contracts\ObservabilityExporter` classes in `ai-engine.observability.exporters`.

Built-in exporter classes are available for common backends:

```php
'observability' => [
    'exporters' => [
        LaravelAIEngine\Services\SDK\Exporters\HttpObservabilityExporter::class,
        LaravelAIEngine\Services\SDK\Exporters\OpenTelemetryObservabilityExporter::class,
        LaravelAIEngine\Services\SDK\Exporters\LangSmithObservabilityExporter::class,
        LaravelAIEngine\Services\SDK\Exporters\LogObservabilityExporter::class,
    ],
    'http' => [
        'endpoint' => env('AI_ENGINE_OBSERVABILITY_HTTP_ENDPOINT'),
        'headers' => ['Authorization' => 'Bearer '.env('OBSERVABILITY_TOKEN')],
    ],
    'opentelemetry' => [
        'endpoint' => env('AI_ENGINE_OTEL_EXPORTER_OTLP_ENDPOINT'),
        'service_name' => 'billing-api',
    ],
    'langsmith' => [
        'api_key' => env('LANGSMITH_API_KEY'),
        'project' => env('LANGSMITH_PROJECT', 'billing-api'),
    ],
],
```

## Structured Chat Collection

Use structured collection when a chat session should gather a target JSON object, ask for missing fields, confirm, close, and call back with the final payload. The package keeps the field keys stable and asks the AI to extract values and respond in the user's current language.

```php
use LaravelAIEngine\DTOs\StructuredCollectionDefinition;

$collection = StructuredCollectionDefinition::make('lead_capture')
    ->description('Collect lead details from chat.')
    ->addText('name', required: true, description: 'Lead name')
    ->addEmail('email', required: true)
    ->addSelect('level', [
        ['value' => 'beginner', 'labels' => ['en' => 'Beginner', 'ar' => 'مبتدئ']],
        ['value' => 'advanced', 'labels' => ['en' => 'Advanced', 'ar' => 'متقدم']],
    ])
    ->addTextarea('notes')
    ->confirmBeforeComplete()
    ->closeOnComplete()
    ->callbackUrl('https://app.test/api/ai/lead-callback', headers: [
        'Authorization' => 'Bearer '.config('services.ai_callback.token'),
    ]);
```

`addField()` remains the generic method for custom JSON-schema properties. Convenience methods such as `addText()`, `addTextarea()`, `addEmail()`, `addUrl()`, `addPhone()`, `addNumber()`, `addInteger()`, `addBoolean()`, `addDate()`, `addDateTime()`, `addTime()`, `addSelect()`, `addMultiSelect()`, `addRadio()`, `addCheckboxes()`, `addFile()`, `addHidden()`, `addJson()`, `addArray()`, and `addObject()` add schema plus UI metadata.

Select-like fields store stable canonical values while the API response returns localized labels in `collection.fields`:

```json
{
  "collection": {
    "data": {"level": "beginner"},
    "fields": [
      {
        "name": "level",
        "type": "string",
        "ui": "select",
        "required": false,
        "options": [
          {"value": "beginner", "label": "مبتدئ"},
          {"value": "advanced", "label": "متقدم"}
        ]
      }
    ]
  }
}
```

Send the definition with chat:

```php
$response = app(\LaravelAIEngine\Services\ChatService::class)->processMessage(
    message: 'اسمي أحمد',
    sessionId: 'lead-session-123',
    userId: 'user-uuid',
    extraOptions: [
        'collection' => $collection->toArray(),
    ],
);
```

Or pass the same payload to `/api/v1/agent/chat`:

```json
{
  "message": "اسمي أحمد",
  "session_id": "lead-session-123",
  "collection": {
    "name": "lead_capture",
    "schema": {
      "type": "object",
      "required": ["name", "email"],
      "properties": {
        "name": {"type": "string"},
        "email": {"type": "string", "format": "email"},
        "level": {
          "type": "string",
          "enum": ["beginner", "advanced"],
          "metadata": {
            "ui": "select",
            "options": [
              {"value": "beginner", "labels": {"en": "Beginner", "ar": "مبتدئ"}},
              {"value": "advanced", "labels": {"en": "Advanced", "ar": "متقدم"}}
            ]
          }
        },
        "notes": {"type": "string"}
      }
    },
    "confirm_before_complete": true,
    "close_on_complete": true,
    "callback": {
      "type": "url",
      "url": "https://app.test/api/ai/lead-callback",
      "method": "POST"
    }
  }
}
```

When the required fields are complete and the user confirms, the callback receives:

```json
{
  "session_id": "lead-session-123",
  "user_id": "user-uuid",
  "status": "completed",
  "collection": "lead_capture",
  "data": {
    "name": "Ahmed",
    "email": "ahmed@example.com"
  },
  "metadata": {
    "language": "ar",
    "confirmed": true,
    "completed_at": "2026-05-18T01:30:00+03:00"
  }
}
```

Every completion also fires `LaravelAIEngine\Events\AgentStructuredCollectionCompleted`, so apps can use Laravel listeners instead of HTTP callbacks by setting `callback.type` to `event`.

## Scaffolding

```bash
php artisan ai:make-agent Invoice --model="App\\Models\\Invoice"
php artisan ai:make-tool LookupCustomer
```

These commands reuse the generic `ai:scaffold` implementation and register generated classes in the agent manifest unless `--no-register` is passed.

## Testing

```php
$fake = Engine::fake(['Done']);

Engine::generatePrompt('Summarize this');

$fake->assertPrompted(fn (array $request) => $request['prompt'] === 'Summarize this');
```

The fake records prompt, request, and message calls and returns queued `AIResponse` objects, arrays, or strings.
