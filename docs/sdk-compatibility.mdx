# SDK Compatibility

Laravel AI Engine keeps orchestration, RAG, GraphRAG, credits, provider routing, and node federation independent from the official `laravel/ai` package while exposing compatible primitives for modern SDK workflows.

Use this page as the compatibility index. The implementation guides now live in focused pages instead of one long catch-all document.

## Basic SDK Shape

```php
use LaravelAIEngine\Facades\Engine;
use LaravelAIEngine\Tools\Provider\WebSearch;

$response = Engine::openai()
    ->model('gpt-4o')
    ->withTools([
        (new WebSearch())->max(3)->allow(['laravel.com']),
    ])
    ->generate('Find the relevant Laravel AI documentation.');
```

Provider shortcuts are thin aliases over `Engine::engine('provider')`. Built-in shortcuts include `openai()`, `anthropic()`, `gemini()`, `openrouter()`, `fal()` / `falAi()`, `elevenLabs()`, `googleTts()`, `replicate()`, `huggingFace()`, `cloudflareWorkersAi()`, and the other `EngineEnum` providers. Use `engine($slug)` when provider names are computed at runtime.

## Reranking

```php
$results = Engine::rerank('invoice automation', [
    'Create invoices from approved orders.',
    'Reset user passwords.',
]);
```

The default reranker is deterministic and local. Applications can wrap or replace `LaravelAIEngine\Services\SDK\RerankingService` when they need a provider-backed reranker.

## Upgrade Rule

When a provider adds a new field before this package has a first-class method, use provider options instead of waiting for a package release:

```php
$response = Engine::openai()
    ->model('gpt-4o')
    ->withProviderOptions(['store' => true])
    ->withProviderOptions([
        'background' => true,
        'include' => ['reasoning.encrypted_content'],
        'remember_response' => true,
    ], 'openai')
    ->generate('Continue the analysis');
```

Keep stable package APIs for workflows you use often; use provider options as the escape hatch for fast-moving provider fields.
