# Provider Credentials

You do not need every provider token to use Laravel AI Engine. Start with the providers your application actually needs, then add optional providers for cost, fallback, local execution, or specialized media.

## Recommended Starting Set

| Capability | Provider | Env |
| --- | --- | --- |
| Chat, agents, tool calling, structured output, hosted files | OpenAI | `OPENAI_API_KEY` |
| Image, video, and audio generation catalog | FAL | `FAL_AI_API_KEY` |
| Semantic memory, RAG, nearest tool and skill retrieval | Qdrant | `QDRANT_API_KEY` |
| Entity relationships, permissions, and graph memory | Neo4j | `NEO4J_URI`, `NEO4J_USERNAME`, `NEO4J_PASSWORD` |
| Local/private development | Ollama, ComfyUI, Local Audio | `OLLAMA_BASE_URL`, `COMFYUI_BASE_URL`, `LOCAL_AUDIO_BASE_URL` |

## Provider Matrix

| Provider | When to use | Required env |
| --- | --- | --- |
| OpenAI | Main production chat, agents, tools, vision, structured JSON, hosted files, TTS, transcription, and vector workflows. | `OPENAI_API_KEY` |
| Anthropic | Reasoning-heavy tasks, long document analysis, planning, or fallback from OpenAI. | `ANTHROPIC_API_KEY` |
| Gemini | Cost-sensitive multimodal work, large context, Imagen, Veo, native TTS, and Lyria media fallback. | `GEMINI_API_KEY` |
| FAL | Production image, video, audio generation, async media jobs, and dynamic catalog execution. | `FAL_AI_API_KEY` |
| Pexels | Stock photo search, curated image discovery, and fallback when generated images are not needed. | `PEXELS_API_KEY` |
| Cloudflare Workers AI | Low-cost edge inference, lightweight image/audio/transcription, and Cloudflare-hosted apps. | `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID` |
| Hugging Face | Open model experimentation, specialized image/audio models, and provider marketplace routing. | `HUGGINGFACE_API_KEY` or `HF_TOKEN` |
| Replicate | Public image/video/audio model endpoints and quick access to community models. | `REPLICATE_API_TOKEN` |
| ComfyUI | Local or self-hosted image/video workflows, custom node graphs, and low provider cost. | `COMFYUI_BASE_URL` |
| Stability AI | Stability image models when you need that provider directly. | `STABILITY_API_KEY` |
| ElevenLabs | High-quality text-to-speech, voices, narration, and agent audio. | `ELEVENLABS_API_KEY` |
| Google Text-to-Speech | Google Cloud TTS voices when you need a dedicated Google speech provider. | `GOOGLE_TTS_API_KEY` or `GOOGLE_TTS_ACCESS_TOKEN` |
| OpenRouter | Router across many LLMs with one API key, model comparison, and fallback. | `OPENROUTER_API_KEY` |
| NVIDIA NIM | Enterprise or GPU-hosted model endpoints. | `NVIDIA_NIM_API_KEY` or `NVIDIA_API_KEY` |
| Ollama | Local/offline chat and embedding development. | `OLLAMA_BASE_URL` |
| Local Audio | Local STT/TTS through OpenAI-compatible Whisper/Kokoro/LocalAI servers or command adapters. See [LocalAI local voice setup](localai-local-voice.mdx) for Docker on macOS and Linux. | `LOCAL_AUDIO_BASE_URL` or published command config |
| LiveKit | Optional realtime WebRTC/SIP room transport for browser voice and LiveKit Agents sidecars. | `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` |
| Qdrant | Semantic search, scoped RAG, memory, and nearest tool or skill discovery. | `QDRANT_API_KEY` |
| Pinecone | Managed vector database alternative when your application already uses Pinecone. | `PINECONE_API_KEY` |
| Neo4j | Graph memory, linked entities, tenant/user/workspace relationships, and explainable links. | `NEO4J_URI`, `NEO4J_USERNAME`, `NEO4J_PASSWORD` |
| LangGraph | Optional durable agent graph runtime for resumable workflows and human approval loops. | `AI_AGENT_LANGGRAPH_API_TOKEN` |

## Cost And Markup

Provider rate multipliers let you charge more than the raw provider cost without changing driver code:

```env
AI_OPENAI_RATE=2.0
AI_ANTHROPIC_RATE=1.5
AI_GEMINI_RATE=1.2
AI_FAL_AI_RATE=1.3
AI_CLOUDFLARE_WORKERS_AI_RATE=1.2
AI_HUGGINGFACE_RATE=1.2
AI_REPLICATE_RATE=1.25
AI_COMFYUI_RATE=1.0
AI_GOOGLE_TTS_RATE=1.2
AI_ELEVENLABS_RATE=1.4
AI_OPENROUTER_RATE=1.25
AI_NVIDIA_NIM_RATE=1.2
```

Media routing estimates live in `ai-engine.media_routing.providers`. These estimates are used for app-side routing and credit policy only; provider invoices remain the source of truth.

## Practical Defaults

For most applications, use OpenAI for chat and tool use, Qdrant for semantic retrieval, Neo4j for linked business entities, and FAL or Gemini for hosted media. Use Gemini native TTS when you already run Gemini media and want one Google AI Studio key; use Google Cloud Text-to-Speech when you need the dedicated Cloud TTS voice catalog. Add Cloudflare, Hugging Face, Replicate, Ollama, or ComfyUI when you need lower cost, local execution, or model variety.

## Realtime Voice Defaults

Realtime voice uses normal provider keys from config. OpenAI WebRTC uses `OPENAI_API_KEY` on the server to mint `/realtime/client_secrets` or proxy `/realtime/calls`; do not expose the standard key to browser clients. Gemini Live uses `GEMINI_API_KEY` for server-side WebSocket connections or provider ephemeral tokens in production client-to-server flows. LiveKit uses `LIVEKIT_API_KEY` and `LIVEKIT_API_SECRET` only on the server to mint room tokens for browser clients.

```env
AI_ENGINE_REALTIME_PROVIDER=openai
AI_ENGINE_REALTIME_MODEL=gpt-realtime
AI_ENGINE_REALTIME_TRANSPORT=webrtc
AI_ENGINE_REALTIME_VOICE=marin
AI_ENGINE_OPENAI_REALTIME_TRANSCRIPTION_MODEL=gpt-realtime-whisper
```

When a provider has no native realtime transport, the package returns a fallback descriptor using existing STT, chat/agent, and TTS endpoints:

```env
AI_ENGINE_REALTIME_FALLBACK_STT_PROVIDER=openai
AI_ENGINE_REALTIME_FALLBACK_STT_MODEL=gpt-4o-transcribe
AI_ENGINE_REALTIME_FALLBACK_CHAT_PROVIDER=openai
AI_ENGINE_REALTIME_FALLBACK_CHAT_MODEL=gpt-4o-mini
AI_ENGINE_REALTIME_FALLBACK_TTS_PROVIDER=openai
AI_ENGINE_REALTIME_FALLBACK_TTS_MODEL=gpt-4o-mini-tts
```

For fully local voice with Ollama:

```env
LOCAL_AUDIO_BASE_URL=http://127.0.0.1:8880/v1
LOCAL_AUDIO_STT_MODEL=whisper-small
LOCAL_AUDIO_STT_LANGUAGE=
LOCAL_AUDIO_STT_PROMPT=
AI_ENGINE_TRANSCRIPTION_NORMALIZATION_ENABLED=false
AI_ENGINE_TRANSCRIPTION_NORMALIZATION_ENGINE=openai
AI_ENGINE_TRANSCRIPTION_NORMALIZATION_MODEL=gpt-4o-mini
LOCAL_AUDIO_TTS_MODEL=local-tts
AI_ENGINE_REALTIME_FALLBACK_STT_PROVIDER=local_audio
AI_ENGINE_REALTIME_FALLBACK_STT_MODEL=local-whisper
AI_ENGINE_REALTIME_FALLBACK_CHAT_PROVIDER=ollama
AI_ENGINE_REALTIME_FALLBACK_CHAT_MODEL=gemma3:4b
AI_ENGINE_REALTIME_FALLBACK_TTS_PROVIDER=local_audio
AI_ENGINE_REALTIME_FALLBACK_TTS_MODEL=local-tts
```

Enable LiveKit when you need long-lived browser voice rooms, SIP, or a separate Python/Node voice worker:

```env
LIVEKIT_URL=wss://your-livekit-host
LIVEKIT_API_KEY=...
LIVEKIT_API_SECRET=...
LIVEKIT_AGENT_NAME=laravel-ai-engine
```

## Dynamic Catalogs

Do not add every provider model to `EntityEnum`. Keep enums for stable package aliases and use the database catalog for fast-moving providers:

```bash
php artisan ai:sync-models --provider=media
php artisan ai:sync-models --provider=fal_ai
```

Synced rows in `ai_models` are resolved by `DynamicModelResolver`, so applications can run discovered provider models without a package release for every catalog change.
