# API Reference

This page summarizes the package HTTP surfaces most apps use directly. Route registration is versioned under `/api/v1/...`.

## Agent Chat

```http
POST /api/v1/agent/chat
Accept: application/json
Content-Type: application/json
```

```json
{
  "message": "Create an invoice for Ahmed",
  "session_id": "thread-123",
  "engine": "openai",
  "model": "gpt-4o-mini",
  "memory": true,
  "actions": true,
  "use_rag": true,
  "response_points_format": "array",
  "response_suggestions": true,
  "response_suggestion_limit": 5
}
```

Response:

```json
{
  "success": true,
  "data": {
    "response": "What email should I use for Ahmed?",
    "rag_enabled": false,
    "sources": [],
    "response_points": [],
    "suggestions": [],
    "collection": null,
    "actions": [],
    "usage": [],
    "session_id": "thread-123"
  },
  "meta": {
    "status_code": 200,
    "schema": "ai-engine.v1"
  }
}
```

Sync chat responses also include full route metadata under `data.metadata`:

| Field | Meaning |
| --- | --- |
| `metadata.agent_runtime` | Runtime that handled the turn, such as `laravel`. |
| `metadata.routing_decision` | Selected routing decision passed to the dispatcher. |
| `metadata.routing_trace` | Ordered decisions and abstentions from the routing pipeline. |
| `metadata.route_explanation` | Compact route summary with `decision_path`, source, reason, and skipped stages. |
| `metadata.rag_enabled` | Whether the response used RAG. Also exposed as `data.rag_enabled`. |
| `metadata.context_count` | Number of retrieved context items. Also exposed as `data.context_count`. |
| `metadata.sources` | Normalized RAG sources. Also exposed as `data.sources`. |

Use the trace fields for debugging and observability. Use top-level `data.response`, `data.rag_enabled`, `data.sources`, `data.response_points`, `data.suggestions`, `data.collection`, and `data.actions` for normal UI rendering. See [ChatFlow Trace](chat-flow.mdx) for the full flow.

Async durable response:

```json
{
  "message": "Create an invoice and generate the preview",
  "session_id": "thread-123",
  "execution_mode": "auto",
  "memory": true,
  "actions": true,
  "use_rag": false
}
```

`execution_mode` accepts `sync`, `async`, or `auto`. Auto keeps simple chat synchronous and queues durable work such as goal/sub-agent runs, streaming requests, structured collection callbacks, and matched skills. The legacy `async=true` flag is still supported and is equivalent to `execution_mode=async`.

When the resolved mode is async, the endpoint returns `202 Accepted`, creates an `ai_agent_runs` record, dispatches `RunAgentJob`, and returns run inspection URLs:

```json
{
  "success": true,
  "data": {
    "queued": true,
    "agent_run_id": "run-uuid",
    "status_url": "/api/v1/ai/agent-runs/run-uuid",
    "trace_url": "/api/v1/ai/agent-runs/run-uuid/trace",
    "stream_url": "/api/v1/ai/agent-runs/run-uuid/stream",
    "execution_mode": "async",
    "execution_mode_reason": "matched_skill",
    "broadcast": {
      "enabled": false,
      "channel": "private-agent-run.run-uuid"
    }
  }
}
```

## Structured Output

```php
$response = Engine::openai()
    ->withStructuredOutput([
        'type' => 'object',
        'properties' => [
            'answer' => ['type' => 'string'],
        ],
        'required' => ['answer'],
        'additionalProperties' => false,
    ], name: 'answer_payload')
    ->generate('Return a short answer.');
```

`withStructuredOutput()` accepts a JSON schema array or `LaravelAIEngine\DTOs\StructuredOutputSchema`. OpenAI-compatible drivers emit a native `response_format` JSON schema payload; Gemini emits `responseMimeType` and `responseSchema`.

## Engine Facade Shortcuts

```php
$text = Engine::openai()
    ->model('gpt-4o-mini')
    ->generate('Write a concise summary.');

$image = Engine::fal()
    ->model('fal-ai/flux-pro')
    ->generateImage('A neutral product photo');

$response = Engine::engine($providerSlug)
    ->model($model)
    ->generate($prompt);
```

Shortcuts return the same `EngineProxy` as `Engine::engine('provider')`. Use shortcuts for static provider choices and `engine($slug)` when tenant, workspace, or runtime policy selects the provider dynamically.

## Structured Collection Payload

```json
{
  "message": "اسمي أحمد ومستواي مبتدئ",
  "session_id": "lead-session-123",
  "collection": {
    "name": "lead_capture",
    "schema": {
      "type": "object",
      "required": ["name", "email", "level"],
      "properties": {
        "name": {"type": "string", "metadata": {"ui": "text"}},
        "email": {"type": "string", "format": "email", "metadata": {"ui": "email"}},
        "level": {
          "type": "string",
          "enum": ["beginner", "advanced"],
          "metadata": {
            "ui": "select",
            "options": [
              {"value": "beginner", "labels": {"en": "Beginner", "ar": "مبتدئ"}},
              {"value": "advanced", "labels": {"en": "Advanced", "ar": "متقدم"}}
            ]
          }
        }
      }
    },
    "presentation": {"preview": true, "mode": "html"},
    "confirm_before_complete": true,
    "close_on_complete": true,
    "callback": {"type": "event"}
  }
}
```

## MCP And Realtime

```http
GET /api/v1/ai/mcp/tools
POST /api/v1/ai/mcp/tools/{tool}/call
POST /api/v1/ai/realtime/sessions
POST /api/v1/ai/realtime/sdp
POST /api/v1/ai/realtime/tools/dispatch
```

`POST /api/v1/ai/realtime/sessions` accepts native providers such as `openai` and `gemini`, fallback providers such as `openrouter`, and `livekit` for room-based realtime voice. Use `transport=livekit` with `metadata.room` and `metadata.participant_identity` to receive a LiveKit connection descriptor and optional room token.

## Agent Runs

```http
GET /api/v1/ai/agent-runs
GET /api/v1/ai/agent-runs/{run}
GET /api/v1/ai/agent-runs/{run}/trace
GET /api/v1/ai/agent-runs/{run}/stream?timeout=30&poll=500
POST /api/v1/ai/agent-runs/{run}/resume
POST /api/v1/ai/agent-runs/{run}/cancel
```

The stream endpoint returns Server-Sent Events from persisted run events. Use it for browser progress UIs without requiring Reverb or Pusher.

## Pricing Preview

```http
POST /api/v1/ai/pricing/preview
Content-Type: application/json
```

```json
{
  "engine": "fal_ai",
  "model": "fal-ai/kling-video/o3/standard/image-to-video",
  "prompt": "Animate this product",
  "parameters": {
    "image_url": "https://example.test/product.png"
  }
}
```

The preview endpoint never calls providers and never deducts credits.

## Health

```http
GET /api/v1/ai/health
```

Use this for load balancer readiness and deployment health checks.
