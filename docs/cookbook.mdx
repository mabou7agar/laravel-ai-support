## Creating Tools

Custom tools extend the agent's capabilities to perform actions, query data, or integrate with external systems. The Laravel AI Engine provides a base class and registry system for building and registering tools that the agent can discover and invoke.

### Basic Tool Structure

Every tool extends `LaravelAIEngine\Services\Agent\Tools\AgentTool` and implements five abstract methods:

```php
<?php

namespace App\AI\Tools;

use LaravelAIEngine\Services\Agent\Tools\AgentTool;
use LaravelAIEngine\DTOs\UnifiedActionContext;
use LaravelAIEngine\DTOs\ActionResult;

class SendEmailTool extends AgentTool
{
    public function getName(): string
    {
        return 'send_email';
    }

    public function getDescription(): string
    {
        return 'Send an email to a recipient with a subject and body.';
    }

    public function getParameters(): array
    {
        return [
            'to' => [
                'type' => 'string',
                'description' => 'Email address of the recipient',
                'required' => true,
            ],
            'subject' => [
                'type' => 'string',
                'description' => 'Email subject line',
                'required' => true,
            ],
            'body' => [
                'type' => 'string',
                'description' => 'Email body text',
                'required' => true,
            ],
        ];
    }

    public function execute(array $parameters, UnifiedActionContext $context): ActionResult
    {
        // Validate and execute the tool
        if (!filter_var($parameters['to'], FILTER_VALIDATE_EMAIL)) {
            return ActionResult::failure('Invalid email address.');
        }

        // Perform the actual work
        Mail::raw($parameters['body'], function ($message) use ($parameters) {
            $message->to($parameters['to'])->subject($parameters['subject']);
        });

        return ActionResult::success('Email sent successfully.');
    }
}
```

### SimpleAgentTool for Less Boilerplate

For simple tools, extend `SimpleAgentTool` instead — it provides default implementations for many methods and derives the tool name automatically from your class name (removing the `Tool` suffix and converting to snake_case):

```php
<?php

namespace App\AI\Tools;

use LaravelAIEngine\Services\Agent\Tools\SimpleAgentTool;
use LaravelAIEngine\DTOs\UnifiedActionContext;
use LaravelAIEngine\DTOs\ActionResult;

class GetWeatherTool extends SimpleAgentTool
{
    public string $description = 'Fetch current weather for a city.';

    public array $parameters = [
        'city' => [
            'type' => 'string',
            'description' => 'City name',
            'required' => true,
        ],
    ];

    protected function handle(array $parameters, UnifiedActionContext $context): ActionResult
    {
        $weather = WeatherService::getForCity($parameters['city']);
        return ActionResult::success('Weather retrieved.', $weather);
    }
}
// Tool is automatically named: get_weather
```

### Registering Tools via Configuration

Register tools in `config/ai-agent.php` under the `tools` key:

```php
'tools' => [
    'send_email' => \App\AI\Tools\SendEmailTool::class,
    'get_weather' => \App\AI\Tools\GetWeatherTool::class,
],
```

The framework will auto-resolve these through Laravel's service container when the `ToolRegistry` is initialized.

### Confirmation and Validation

Tools can require user confirmation before execution and define custom validation:

```php
public function requiresConfirmation(): bool
{
    return true;
}

public function getConfirmationMessage(): ?string
{
    return 'This will send an email. Proceed?';
}

public function validate(array $parameters): array
{
    $errors = [];
    if (empty($parameters['to'] ?? null)) {
        $errors[] = 'Recipient email is required.';
    }
    return $errors;
}
```

### Advanced: Model-Backed Tools with AiResource

For Eloquent model operations (search, create, update), use `AiResource` — a zero-code builder that generates `find_<name>`, `create_<name>`, and `show_<name>` tools without writing tool classes:

```php
// In a service provider or bootstrap file:

use LaravelAIEngine\Services\Agent\Tools\AiResource;
use App\Models\Customer;

AiResource::for(Customer::class)
    ->search(['name', 'email'])              // Lookup columns
    ->writable(['name', 'email', 'company']) // Create/update fields
    ->identity(['email'])                     // Find-or-create key
    ->required(['name', 'email'])             // Mandatory fields
    ->detail()                                 // Register a show_customer tool
    ->with(['invoices'])                      // Include related records
    ->register();
```

Or declare it once in `config/ai-agent.php` under `resources`:

```php
'resources' => [
    'customer' => [
        'model' => \App\Models\Customer::class,
        'search' => ['name', 'email'],
        'writable' => ['name', 'email', 'company'],
        'identity' => ['email'],
        'required' => ['name', 'email'],
        'with' => ['invoices'],
    ],
],
```

#### AiResource Options

- **`search(array $columns)`** — Columns the `find_<name>` tool searches by (defaults to `['name']`).
- **`returns(array $columns)`** — Columns returned in results (all columns if empty).
- **`writable(array $fields)`** — Fields the `create_<name>` tool can set; omit to make the resource read-only.
- **`identity(array $fields)`** — Keys used for find-or-create logic (e.g., email for upserts).
- **`required(array $fields)`** — Fields that must be provided to create.
- **`defaults(array|Closure $values)`** — Server-set columns that cannot be overridden by the agent.
- **`scopedBy(Closure $fn)`** — Tenant/workspace isolation: `fn($context, $params) => ['workspace_id' => $context->metadata['workspace_id']]`.
- **`lookupOnly()`** — Register only the `find_<name>` tool (no create).
- **`detail()`** — Generate a `show_<name>` tool for detailed record + relations.
- **`with(array $relations)`** — Relations to eager-load in detail view; implies `->detail()`.
- **`confirmationMessage(string $msg)`** — Custom confirmation prompt for writes.

### Built-In Tools

The registry auto-registers several tools controlled by config flags in `ai-engine.php` and `ai-agent.php`:

| Tool | Config Flag | Purpose |
|------|---|---|
| `data_query` | `ai-engine.data_query.enabled` (default: true) | Answer count, list, and filter questions over registered Eloquent models with exact database results. |
| `aggregate_data` | `ai-engine.data_query.aggregate_enabled` (default: true) | Compute sum, average, min, max, and group-by aggregations for analytics questions. |
| `analyze_file` | `ai-engine.file_analysis.enabled` (default: false) | Extract text from uploads and suggest creation actions based on keyword patterns. Requires `keyword_suggestions` config. |
| `search_knowledge` | `ai-agent.ai_native.knowledge_tool_enabled` (default: true) | Query the vector/document RAG store for grounded knowledge-base answers. |
| `find_tools` | Tool disclosure = `'progressive'` | Meta-tool that loads full schemas on demand instead of injecting all tools upfront; reduces initial context. |
| `search_learned_context` | Always registered | Search learned examples, design rules, and workflow guidance. |
| `run_skill` | Always registered | Execute a named skill within the agent runtime. |
| `run_sub_agent` | `ai-agent.goal_agent.register_sub_agent_tool` (default: true) | Delegate work to a named sub-agent. |

Inject the registry to register tools dynamically or query available tools:

```php
use LaravelAIEngine\Services\Agent\Tools\ToolRegistry;

$registry = app(ToolRegistry::class);

// Retrieve a tool

// Check if a tool is registered
if ($registry->has('send_email')) {
    // ...

// Get all tools
$allTools = $registry->all();

// Export tool definitions for the planner
$definitions = $registry->getToolDefinitions();
```

Tools expose metadata about their capabilities and return schema:

```php
public function getCapabilities(): array
{
    return ['read', 'write'];
}

public function getToolKind(): ?string
{
    return 'action'; // or 'query', 'integration', etc.
}

public function getEntityType(): ?string
{
    return 'customer'; // or 'invoice', 'document', etc.
}

public function getRelations(): array
{
    // Describe relations this tool can resolve:
    return [
        ['name' => 'customer', 'type' => 'belongsTo', 'model' => Customer::class],
    ];
}

public function getResultSchema(): array
{
    return [
        'id' => ['type' => 'integer'],
        'name' => ['type' => 'string'],
        'total' => ['type' => 'number'],
    ];
}
```

### Preview and Confirmation

Optionally implement `previewConfirmation()` to show a summary before the tool executes:

```php
public function previewConfirmation(array $parameters, UnifiedActionContext $context): ?ActionResult
{
    // Return a preview; if null, the confirmation uses default formatting
    return ActionResult::success('Preview', [
        'recipient' => $parameters['to'],
        'subject' => $parameters['subject'],
        'body_preview' => substr($parameters['body'], 0, 100) . '...',
    ]);
}
```

### ActionResult Details

Tools return `ActionResult` to signal success or failure:

```php
// Success with data
return ActionResult::success('Operation completed.', ['id' => 123]);

// Success with metadata (e.g., for tracing)
return ActionResult::success(
    'Email sent.',
    ['message_id' => 'abc123'],
    ['provider' => 'sendgrid', 'latency_ms' => 45]
);

// Failure
return ActionResult::failure(
    'Could not send email.',
    null,
    ['validation_errors' => $errors]
);
```

## Creating Skills

Skills are high-level business capabilities that group tools, triggers, required data, actions, and confirmation policies. A skill encapsulates a complete workflow (e.g., "create invoice", "send email", "approve request") and orchestrates the tools needed to accomplish it under the AiNative runtime. Skills are matched to user intents deterministically (by trigger keywords) or semantically (by AI understanding), then executed through the `run_skill` tool.

### The AgentSkill Base Class

All skills inherit from `LaravelAIEngine\Services\Agent\Skills\AgentSkill` and declare their capability as properties. The skill definition is built from the `configure()` method, which receives a `SkillBuilder`:

```php
<?php

namespace App\AI\Skills;

use LaravelAIEngine\Services\Agent\Skills\AgentSkill;
use LaravelAIEngine\Services\Agent\Skills\SkillBuilder;
use App\AI\Tools\CreateInvoiceTool;
use App\AI\Tools\SearchCustomerTool;

class CreateInvoiceSkill extends AgentSkill
{
    public string $id = 'create_invoice';
    public string $name = 'Create Invoice';
    public string $description = 'Create a new invoice for a customer with line items.';
    
    public array $triggers = [
        'create invoice',
        'new invoice',
        'make invoice',
        'invoice customer',
    ];
    
    public array $requiredData = [
        'customer',
        'line items',
    ];
    
    public array $actions = [
        'create',
        'finalize',
    ];
    
    public array $capabilities = [
        'invoicing',
        'billing',
    ];
    
    public bool $requiresConfirmation = true;
    public bool $enabled = true;

    public function configure(SkillBuilder $skill): void
    {
        $skill
            ->use(SearchCustomerTool::class)
            ->use(CreateInvoiceTool::class)
            ->final(CreateInvoiceTool::class)
                ->confirmTerms(['create', 'submit', 'save'])
            ->prompt('You are creating an invoice. First find the customer, then ask for line items if not provided, then create the invoice.');
    }
}
```

#### Key Properties

- **`$id`** (string) — Unique skill identifier, defaults to snake_case of class name minus "Skill" suffix if left empty.
- **`$name`** (string) — Human-readable name; defaults to title-cased `$id` if empty.
- **`$description`** (string) — One or two sentences describing what the skill does; defaults to "Handle {name} requests using declared tools."
- **`$triggers`** (array of strings) — Keywords that deterministically match this skill (exact match = 100 score, word boundary = 75 score).
- **`$requiredData`** (array of strings) — Human-readable list of data the skill needs (e.g., "customer", "dates").
- **`$actions`** (array of strings) — CRUD/domain verbs the skill performs (e.g., "create", "update", "delete", "send").
- **`$capabilities`** (array of strings) — Business domains the skill covers; defaults to `[$id]` if empty. Used for semantic routing and documentation.
- **`$requiresConfirmation`** (bool) — Whether the skill's final tool requires user approval before execution (default: `true`).
- **`$enabled`** (bool) — Whether this skill is available to the agent (default: `true`).

### The SkillBuilder & Tool Registration

Inside `configure()`, use `SkillBuilder` to declare and chain tools:

```php
public function configure(SkillBuilder $skill): void
{
    $skill
        ->use(SearchCustomerTool::class)          // Register a lookup tool
        ->use(ValidateLineItemsTool::class)       // Register a validator
        ->final(CreateInvoiceTool::class)         // Designate final (action) tool
            ->confirmTerms(['create', 'submit'])  // Confirmation keywords
        ->prompt('Create the invoice after confirming all details.');
}
```

#### SkillBuilder Methods

- **`use(tool)`** — Register a tool class or string name; returns `SkillToolUseBuilder` for chaining.
- **`final(tool)`** — Register a tool as the skill's final (write/action) tool; this tool's confirmation policy is enforced.
  - `->confirmTerms([...])` — Array of keywords that indicate user approval (e.g., 'confirm', 'yes', 'do it').
- **`prompt(string)` / `->instructions(string)`** — Add a skill-specific prompt to guide the agent's decision-making during execution.
- **`target()`** — Returns `SkillTargetBuilder` for declaring the JSON schema of the skill's expected input data.
- **`relation(name)`** — Declare a relation field and its lookup/create tools (see below).

### Target JSON & Relations

Use `SkillTargetBuilder` to declare the shape of data the skill expects:

```php
public function configure(SkillBuilder $skill): void
{
    $skill->target()
        ->type('object')
        ->prop('customer_id', 'integer', required: true)
        ->prop('line_items', 'array', required: true)
            ->items()
                ->type('object')
                ->prop('product_id', 'integer', required: true)
                ->prop('quantity', 'integer', required: true);
    
    $skill
        ->use(SearchCustomerTool::class)
        ->use(SearchProductTool::class)
        ->use(CreateInvoiceTool::class)
        ->final(CreateInvoiceTool::class);
}
```

### Relations: Lookup & Create Tools

Relations connect target fields to lookup or create tools. The `SkillBuilder::relation()` method pairs a field with its lookup and/or create tool:

```php
public function configure(SkillBuilder $skill): void
{
    $skill
        ->use(SearchCustomerTool::class)
        ->use(CreateCustomerTool::class)
        ->use(CreateInvoiceTool::class)
        ->final(CreateInvoiceTool::class)
        ->relation('customer')              // Relation name
            ->field('customer_id')          // Target field to populate
            ->lookup(SearchCustomerTool::class)
            ->create(CreateCustomerTool::class)
            ->lookupFields(['email', 'customer_name'])
            ->createRequired(['name', 'email'])
            ->safeCreate(true);             // Allow auto-creation if lookup fails
}
```

#### SkillRelationBuilder Methods

- **`field(name)`** — Target field name (e.g., `customer_id`).
- **`lookup(tool)`** — Tool to find/search for the entity.
- **`create(tool)`** — Tool to create the entity if not found.
- **`lookupFields([])`** — Fields to search on when doing a lookup (e.g., `['email']`).
- **`createRequired([])`** — Required fields when creating a new entity.
- **`safeCreate(bool)`** — If `true`, the agent may auto-create if lookup fails; if `false`, always ask the user.

### Skill Matching & Execution

#### Skill Registry

The `AgentSkillRegistry` retrieves all registered skills and filters by enable/disable state:

```php
$registry = app(AgentSkillRegistry::class);
$allSkills = $registry->skills();
$enabledSkills = $registry->skills(includeDisabled: false);
$onlyInvoicing = $registry->skills(only: ['create_invoice', 'send_invoice']);
```

#### Skill Matcher

The `AgentSkillMatcher` scores skills against user messages using deterministic trigger matching or semantic AI intent matching:

```php
$matcher = app(AgentSkillMatcher::class);

// Deterministic (trigger-based) matching
$match = $matcher->match('create invoice for Acme');
// Returns: ['skill' => AgentSkillDefinition, 'score' => 100, 'trigger' => 'create invoice', 'reason' => '...']

// Semantic matching (AI intent, enabled by config)
$match = $matcher->matchIntent('I want to bill the customer', $context);
```

Matching scores (deterministic only):
- **100** — Exact trigger match
- **90** — Trigger as first words
- **75** — Trigger found as word boundary
- **0** — No match

Semantic matching uses AI to understand user intent when triggers don't match. Configure intent matching in `config/ai-agent.php`:

```php
'skills' => [
    'intent_matching' => [
        'enabled' => env('AI_AGENT_SKILL_INTENT_MATCHING_ENABLED', true),
        'engine' => env('AI_AGENT_SKILL_INTENT_MATCHING_ENGINE', 'openai'),
        'model' => 'gpt-4o-mini',
        'min_confidence' => 72,  // Confidence threshold (0–100)
    ],
    'continuation_terms' => [
        'do it', 'create it', 'proceed', 'finalize',
    ],
    'intent_aliases' => [
        'billing' => ['invoice', 'bill', 'charge'],
    ],
],
```

### The run_skill Tool

The `run_skill` tool is registered automatically and orchestrates skill execution under the AiNative runtime:

```php
$tool = RunSkillTool::class; // LaravelAIEngine\Services\Agent\Tools\RunSkillTool

// In agent plans, the model invokes:
[
    'name' => 'run_skill',
    'arguments' => [
        'skill_id' => 'create_invoice',
        'message' => 'Create invoice for Acme Corp.',
        'reset' => false,
        'fresh_start' => false,
    ]
]
```

#### run_skill Parameters

- **`skill_id`** (string, required) — The skill ID to execute.
- **`message`** (string, optional) — User message for the skill; defaults to the current turn's message.
- **`reset`** (boolean, optional) — Clear skill state and start fresh.
- **`fresh_start`** (boolean, optional) — Begin skill execution with a clean context.

When invoked, `run_skill` delegates to `AiNativeRuntime`, which:
1. Scopes the agent to the skill's declared tools only.
2. Runs a sub-loop of tool calls under the skill's prompt/constraints.
3. Returns the result (success, failure, or needs input) to the orchestrating agent.

### Complete Example: Multi-Step Skill

```php
<?php

namespace App\AI\Skills;

use LaravelAIEngine\Services\Agent\Skills\AgentSkill;
use LaravelAIEngine\Services\Agent\Skills\SkillBuilder;
use App\AI\Tools\SearchCustomerTool;
use App\AI\Tools\CalculateTotalTool;
use App\AI\Tools\CreateInvoiceTool;

class CreateInvoiceSkill extends AgentSkill
{
    public string $id = 'create_invoice';
    public string $name = 'Create Invoice';
    public string $description = 'Create and send an invoice to a customer with line items, discounts, and payment terms.';
    
    public array $triggers = [
        'create invoice',
        'new invoice',
        'make an invoice',
        'invoice the customer',
    ];
    
    public array $requiredData = [
        'customer',
        'line items or products',
    ];
    
    public array $actions = [
        'create',
        'finalize',
    ];
    
    public array $capabilities = [
        'invoicing',
        'billing',
        'financial',
    ];
    
    public bool $requiresConfirmation = true;

    public function configure(SkillBuilder $skill): void
    {
        $skill
            ->target()
                ->type('object')
                ->prop('customer_id', 'integer', required: true)
                ->prop('line_items', 'array', required: true)
                    ->items()
                        ->type('object')
                        ->prop('product_id', 'integer', required: true)
                        ->prop('quantity', 'integer', required: true)
                ->prop('discount_percent', 'number', required: false)
                ->prop('notes', 'string', required: false);
        
        $skill
            ->use(SearchCustomerTool::class)
            ->use(SearchProductTool::class)
            ->use(CalculateTotalTool::class)
            ->final(CreateInvoiceTool::class)
                ->confirmTerms(['create', 'confirm', 'submit']);
        
        $skill
            ->relation('customer')
                ->field('customer_id')
                ->lookup(SearchCustomerTool::class)
                ->lookupFields(['email', 'name', 'phone']);
        
        $skill
            ->prompt(implode("\n", [
                'You are creating an invoice.',
                'First, find the customer by email or name.',
                'Then, ask for line items or search for products if needed.',
                'Calculate the total and ask for confirmation before creating.',
                'If the user provides a discount, apply it.',
            ]));
    }

    public function targetJson(): array
    {
        return [
            'customer_id' => 42,
            'line_items' => [
                ['product_id' => 1, 'quantity' => 2],
            ],
            'discount_percent' => 10,
            'notes' => 'Net 30',
        ];
    }
}
```

This skill:
- Declares customer and line items as required data.
- Uses three tools: two lookups and one final action.
- Declares a relation: customer_id is found/matched via the SearchCustomerTool.
- Requires confirmation before the CreateInvoiceTool runs.
- Provides a domain-specific prompt to guide the agent's reasoning.

## Using Any Model & Engine

The Laravel AI Engine provides a unified interface for working with multiple AI providers and their models. You can switch between engines (OpenAI, Anthropic, Gemini, etc.) and models either globally in configuration or per-request, with built-in automatic failover to backup engines when errors occur.

### Configuration Setup

Configure available engines and credentials in `config/ai-engine-defaults.php` or `config/ai-engine.php` (published):

```php
// config/ai-engine.php
```

### Automatic Failover to Backup Engines

The `generate()` method includes built-in failover: if the primary engine fails, it automatically tries configured fallback engines in order, skipping engines with missing credentials:

```php
// config/ai-engine-defaults.php (excerpt)
'error_handling' => [
    'fallback_engines' => [
        'openai' => ['openrouter', 'anthropic', 'gemini'],
        'anthropic' => ['openrouter', 'openai', 'gemini'],
        'gemini' => ['openrouter', 'openai', 'anthropic'],
        'stable_diffusion' => ['cloudflare_workers_ai', 'huggingface', 'replicate', 'comfyui'],
    ],
],
```

When you call `$service->generate($request)` with OpenAI:
1. Tries OpenAI first
2. If it fails, tries OpenRouter (if `OPENROUTER_API_KEY` is set)
3. Then Anthropic, then Gemini
4. Stops at the first successful response
5. Returns error only if all engines are exhausted

Fallback engines without configured API keys are automatically skipped. To disable failover and get direct errors, use `generateDirect()`:

```php
// No failover—immediately fails if openai has an error
$response = $service->generateDirect($request);
```

## Relating Records

When an agent creates or updates a record with foreign keys, it often needs to find or create related records first — a customer, project, or account must exist before an invoice references it. The `ResolvesAgentRelations` trait and `RelationResolver` let you declare those dependencies and resolve them deterministically, using a source-of-truth identity (e.g., email) to find existing records or create missing ones in a single operation.

### Using ResolvesAgentRelations in Write Tools

Write tools (those that create/update records) can declare and resolve relations before persisting. The `ResolvesAgentRelations` trait provides a `relations()` method to specify what to resolve and a `resolveRelations()` call to execute:

```php
<?php

use LaravelAIEngine\Services\Agent\Tools\AgentTool;
use LaravelAIEngine\Services\Agent\Tools\Concerns\ResolvesAgentRelations;
use LaravelAIEngine\DTOs\ActionResult;
use LaravelAIEngine\DTOs\UnifiedActionContext;

class CreateInvoiceTool extends AgentTool
{
    use ResolvesAgentRelations;

    public function getName(): string
    {
        return 'create_invoice';
    }

    public function getDescription(): string
    {
        return 'Create a new invoice for a customer.';
    }

    public function getParameters(): array
    {
        return [
            'invoice_number' => ['type' => 'string', 'required' => true, 'description' => 'Invoice number.'],
            'customer_email' => ['type' => 'string', 'required' => true, 'description' => 'Customer email address.'],
            'customer_name' => ['type' => 'string', 'required' => true, 'description' => 'Customer name.'],
            'total' => ['type' => 'number', 'required' => true, 'description' => 'Invoice total.'],
        ];
    }

    /**
     * Declare relations: find or create a customer by email before saving the invoice.
     */
    protected function relations(): array
    {
        return [
            [
                'field'    => 'customer_id',            // Foreign key column to populate
                'model'    => \App\Models\Customer::class,
                'identity' => ['email'],                 // Find by email (source of truth)
                'map'      => [
                    'email' => 'customer_email',         // email column ← customer_email param
                    'name' => 'customer_name',           // name column ← customer_name param
                ],
                'create'   => ['email', 'name'],         // Required to create a new customer
            ],
        ];
    }

    public function execute(array $parameters, UnifiedActionContext $context): ActionResult
    {
        // Resolve relations first: find or create the customer by email.
        // If the customer exists, $parameters['customer_id'] is set to their ID.
        // If missing, a new customer is created with the provided name and email.
        $parameters = $this->resolveRelations($parameters, $context);

        // Now customer_id is guaranteed to be set (or creation failed).
        if (empty($parameters['customer_id'])) {
            return ActionResult::needsUserInput(
                'Unable to resolve or create a customer. Please provide a customer name and email.'
            );
        }

        $invoice = \App\Models\Invoice::create([
            'number'      => $parameters['invoice_number'],
            'customer_id' => $parameters['customer_id'],
            'total'       => $parameters['total'],
        ]);

        return ActionResult::success('Invoice created.', ['invoice_id' => $invoice->id]);
    }
}
```

### Relation Specification Keys

Each relation in the `relations()` array uses these keys:

- **`field`** (required) — The foreign-key column to populate on the payload (e.g., `customer_id`).
- **`model`** (required) — The Eloquent model class to find or create (e.g., `\App\Models\Customer::class`).
- **`identity`** (required) — Array of column names used to uniquely identify a record (e.g., `['email']`). These are the "source of truth" — if the payload already has a foreign key whose identity no longer matches (e.g., the name changed), the ID is dropped and re-resolved.
- **`map`** (optional) — Maps identity/create columns to payload keys. Without a map, column names and parameter names must match exactly. Example: `['email' => 'customer_email', 'name' => 'customer_name']` tells the resolver to look for `customer_email` and `customer_name` in the payload.
- **`create`** (optional) — Array of columns required to create a new record. If any are missing from the payload, creation is skipped and the relation remains unresolved (allowing the tool to re-prompt for the missing value).
- **`defaults`** (optional) — Create-only columns added to new records but NOT used to match existing ones. Useful for generated passwords, source flags, or creation timestamps: `['defaults' => ['source' => 'ai']]`.
- **`normalize`** (optional) — Array of columns to lowercase before matching or creating (default: the identity columns). Use when matching should be case-insensitive: `['normalize' => ['email']]`.
- **`scope`** (optional via `relationScope()`) — Additional where conditions for lookup and create (e.g., `tenant_id`).

### Constraining Relations to a Tenant

Override the `relationScope()` method to add tenant/owner filters that apply to both lookup and creation:

```php
protected function relationScope(UnifiedActionContext $context, array $relation): array
{
    // All customer lookups and creations are scoped to the current workspace.
    return [
        'workspace_id' => $context->workspaceId(),
    ];
}
```

Now when the resolver finds a customer by email, it only searches within the workspace. If the customer doesn't exist, the new record is created with `workspace_id` set automatically.

### Resolving Multiple Relations

A single tool can declare and resolve multiple relations — each is handled in order:

```php
protected function relations(): array
{
    return [
        [
            'field'  => 'customer_id',
            'model'  => \App\Models\Customer::class,
            'identity' => ['email'],
            'map'    => ['email' => 'customer_email', 'name' => 'customer_name'],
            'create' => ['email', 'name'],
        ],
        [
            'field'  => 'project_id',
            'model'  => \App\Models\Project::class,
            'identity' => ['code'],
            'map'    => ['code' => 'project_code', 'name' => 'project_name'],
            'create' => ['code', 'name'],
        ],
    ];
}
```

### Relation Resolution via AiResource (Config-Driven)

The `AiResource` class generates write tools automatically, but for relations you must use the `ResolvesAgentRelations` trait directly. However, you can create a custom upsert tool that uses both patterns:

```php
use LaravelAIEngine\Services\Agent\Tools\ModelBackedUpsertTool;
use LaravelAIEngine\Services\Agent\Tools\Concerns\ResolvesAgentRelations;

class InvoiceUpsertTool extends ModelBackedUpsertTool
{
    use ResolvesAgentRelations;

    protected array $identity = ['number'];
    protected array $write = ['number', 'customer_id', 'total'];
    protected array $required = ['number', 'customer_id', 'total'];

    protected function relations(): array
    {
        return [
            [
                'field'  => 'customer_id',
                'model'  => \App\Models\Customer::class,
                'identity' => ['email'],
                'map'    => ['email' => 'customer_email', 'name' => 'customer_name'],
                'create' => ['email', 'name'],
            ],
        ];
    }

    public function execute(array $parameters, UnifiedActionContext $context): ActionResult
    {
        // Resolve relations before calling parent.
        $parameters = $this->resolveRelations($parameters, $context);
        return parent::execute($parameters, $context);
    }
}
```

### Showing Related Records with GenericModelDetailTool

The `GenericModelDetailTool` (created by `AiResource->with()->detail()`) eager-loads and returns related records:

```php
AiResource::for(Invoice::class)
    ->name('invoice')
    ->search(['number'])
    ->writable(['customer_id', 'total'])
    ->identity(['number'])
    ->required(['customer_id', 'total'])
    // Include line items in the detail tool (show_invoice)
    ->with([
        'items' => ['id', 'product_id', 'quantity', 'unit_price'],  // Show these columns
    ])
    ->detail()  // Enable the show_invoice tool
    ->register();
```

Now calling `show_invoice` returns the full invoice plus its related items:

```json
{
  "found": true,
  "record": {
    "id": 1,
    "number": "INV-001",
    "customer_id": 10,
    "total": 150.00
  },
  "relations": {
    "items": [
      { "id": 1, "product_id": 5, "quantity": 2, "unit_price": 75.00 },
      { "id": 2, "product_id": 8, "quantity": 1, "unit_price": 0.00 }
    ]
  }
}
```

The `with()` method accepts either a simple relation name (`['items']`) to return all columns, or `relation => [columns]` to select specific columns.

### Email as the Source of Truth

A common pattern is using email as the unique identity for customers or users:

```php
[
    'field'    => 'customer_id',
    'model'    => \App\Models\Customer::class,
    'identity' => ['email'],           // Email is the source of truth
    'map'      => ['email' => 'customer_email', 'name' => 'customer_name'],
    'create'   => ['email', 'name'],
    'normalize' => ['email'],          // Case-insensitive matching
    'defaults' => ['status' => 'active'],  // Auto-activate new customers
]
```

If the payload arrives with `customer_id: 10` but that customer's email no longer matches the provided `customer_email`, the resolver drops the ID and re-resolves by email. This ensures the identity parameter always takes precedence — if the user says "email john@example.com" and that maps to a different customer, the resolver respects the email over any pre-set ID.

### Configuration Keys and Defaults

All relation resolution is done in PHP at runtime; there is no separate config file. The `ResolvesAgentRelations` trait uses `app(RelationResolver::class)` internally, which is a singleton managed by the service container.

Key classes and namespaces:
- `LaravelAIEngine\Services\Agent\Tools\Concerns\ResolvesAgentRelations` — trait for tools
- `LaravelAIEngine\Services\Agent\Tools\RelationResolver` — core resolution logic (invoked by `resolveRelations()`)
- `LaravelAIEngine\Services\Agent\Tools\GenericModelDetailTool` — detail tool with relation eager-loading
- `LaravelAIEngine\Services\Agent\Tools\AiResource` — fluent API for `with()` and `detail()`

There are no opt-in config flags; relation resolution is always available when you use the trait.

## Embeddings & RAG

Embeddings transform text into vector representations for semantic search, and RAG (Retrieval-Augmented Generation) augments AI responses with retrieved context from your application data. The Laravel AI Engine provides a batteries-included system for both, using the `Vectorizable` trait and a pluggable vector driver architecture.

### Quick Start: Make a Model Searchable

The simplest path is to add the `Vectorizable` trait to any Eloquent model:

```php
<?php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use LaravelAIEngine\Traits\Vectorizable;

class Document extends Model
{
    use Vectorizable;
    
    protected $fillable = ['title', 'content', 'user_id'];
}
```

The trait automatically:
- Extracts and embeds text from your model's fields via `getVectorContent()` (uses `$fillable` by default, or override with a `protected array $vectorizable` property)
- Stores embeddings in your configured vector database (Qdrant, Pinecone, etc.)
- Provides `vectorSearch()` for semantic queries and `intelligentChat()` for RAG conversations

### EmbeddingService: Core Embeddings API

The `EmbeddingService` handles text-to-vector conversion with automatic provider fallback and caching.

**Namespace:** `LaravelAIEngine\Services\Vector\EmbeddingService`

```php
use LaravelAIEngine\Services\Vector\EmbeddingService;

$embeddingService = app(EmbeddingService::class);

// Single embedding
$embedding = $embeddingService->embed("Hello world");
// Returns: float[] (default 3072 dimensions for text-embedding-3-large)

// Batch embedding (more efficient)
$embeddings = $embeddingService->embedBatch([
    "Document one",
    "Document two",
    "Document three"
]);

// Cosine similarity (for comparing embeddings)
$similarity = $embeddingService->cosineSimilarity($embedding1, $embedding2);
// Returns: float (0.0 to 1.0, where 1.0 = identical direction)

// Get model info
$model = $embeddingService->getModel(); // e.g., "text-embedding-3-large"
$dims = $embeddingService->getDimensions(); // e.g., 3072
```

**Key config keys** (in `config/ai-engine.php` or `ai-engine-defaults.php`):
- `ai-engine.vector.embedding_model` (default: `text-embedding-3-large`)
- `ai-engine.vector.embedding_dimensions` (default: 3072; capped to model max)
- `ai-engine.vector.cache_embeddings` (default: `true`)
- `ai-engine.vector.cache_ttl` (default: 86400 seconds / 24 hours)
- `ai-engine.vector.testing.use_fake_embeddings` (default: `false`)

**Testing mode:** Set `AI_ENGINE_USE_FAKE_EMBEDDINGS=true` to use deterministic local embeddings without API calls. Useful for tests and offline development.

### VectorDriverManager: Pluggable Vector Stores

Swap between vector database backends (Qdrant, Pinecone, etc.) at runtime.

**Namespace:** `LaravelAIEngine\Services\Vector\VectorDriverManager`

```php
use LaravelAIEngine\Services\Vector\VectorDriverManager;

$driverManager = app(VectorDriverManager::class);

// Get the default driver (configured in config/ai-engine.php)
$driver = $driverManager->driver();

// Or request a specific driver by name
$qdrant = $driverManager->driver('qdrant');
$pinecone = $driverManager->driver('pinecone');

// Register a custom driver
$driverManager->extend('my_custom_db', fn() => new MyCustomDriver());

// Change default
$driverManager->setDefaultDriver('pinecone');
```

**Supported drivers:**
- `qdrant` – Qdrant vector database (default)
- `pinecone` – Pinecone serverless

**Config structure** (in `ai-engine.vector.drivers`):

```php
'vector' => [
    'default_driver' => env('VECTOR_DB_DRIVER', 'qdrant'),
    'drivers' => [
        'qdrant' => [
            'host' => env('QDRANT_HOST', 'http://localhost:6333'),
            'api_key' => env('QDRANT_API_KEY'),
            'timeout' => env('QDRANT_TIMEOUT', 30),
        ],
        'pinecone' => [
            'api_key' => env('PINECONE_API_KEY'),
            'environment' => env('PINECONE_ENVIRONMENT', 'us-west1-gcp'),
            'timeout' => env('PINECONE_TIMEOUT', 30),
        ],
    ],
    'collection_prefix' => env('VECTOR_COLLECTION_PREFIX', 'vec_'),
    // ... more options
],
```

### Vectorizable Trait: Content & Metadata

The `Vectorizable` trait (included in your model) provides methods to control what gets embedded.

**Namespace:** `LaravelAIEngine\Traits\Vectorizable`

#### Controlling What Gets Indexed

```php
class Article extends Model
{
    use Vectorizable;
    
    // Option 1: Auto-detect from $fillable (simplest, recommended)
    protected $fillable = ['title', 'body', 'summary', 'user_id', 'status'];
    
    // Option 2: Explicit fields to vectorize
    protected array $vectorizable = ['title', 'body', 'summary'];
    
    // Option 3: Override to customize
    public function getVectorContent(): string
    {
        return "{$this->title}\n\n{$this->body}";
    }
}
```

#### Custom Metadata for Filtering

Override `getVectorMetadata()` to add fields for filtering search results:

```php
class Article extends Model
{
    use Vectorizable;
    
    public function getVectorMetadata(): array
    {
        $metadata = parent::getVectorMetadata(); // Gets user_id, tenant_id, etc. automatically
        
        return array_merge($metadata, [
            'status' => $this->status, // 'draft', 'published', 'archived'
            'category_id' => $this->category_id,
            'author_id' => $this->user_id, // Alias for clarity
            'is_featured' => $this->is_featured ? 1 : 0,
        ]);
    }
}
```

#### Indexing Control

```php
class Draft extends Model
{
    use Vectorizable;
    
    // Don't index drafts or soft-deleted records
    public function shouldBeIndexed(): bool
    {
        return !$this->trashed() && $this->status !== 'draft';
    }
}
```

For fast metadata filtering in Qdrant, declare which fields to index:

```php
public function getQdrantIndexes(): array
{
    return [
        'user_id' => 'integer',
        'status' => 'keyword',
        'is_public' => 'bool',
    ];
}
```

### RAG: Intelligent Context-Aware Chat

RAG lets the AI automatically retrieve relevant context before answering, grounding responses in your app's data.

#### Intelligent Chat (AI Decides When to Search)

```php
use App\Models\Document;

$response = Document::intelligentChat(
    query: "What are the latest company policies?",
    sessionId: auth()->id(),
    options: [
        'search_all' => true, // Search all RAG collections (default: false)
        'restrict_to_model' => false, // false = broader context
    ]
);

echo $response->getContent(); // AI's answer with retrieved context
echo json_encode($response->getMetadata()['sources'] ?? []); // Source citations
```

#### Vector Chat (Always Searches)

Simpler method that always retrieves context for the current model:

```php
use App\Models\EmailMessage;

$result = EmailMessage::vectorChat(
    query: "What did John say about the project deadline?",
    userId: auth()->id(),
    options: [
        'engine' => 'openai',
        'model' => 'gpt-4o',
    ]
);

echo $result['response']; // Answer
echo count($result['sources']); // Number of retrieved items
```

#### Streaming RAG Chat

Stream responses token-by-token while using RAG context:

```php
$result = Document::vectorChatStream(
    query: "Summarize recent updates",
    callback: function(string $chunk, bool $isFirst, bool $isDone) {
        if ($isFirst) echo "Thinking...\n";
        if (!$isDone) echo $chunk;
        if ($isDone) echo "\n✓ Done\n";
    },
    userId: auth()->id(),
    options: [
        'max_tokens' => 2000,
    ]
);

echo "Retrieved {$result['context_count']} items";
```

#### Instance Methods

Ask questions about a specific model instance:

```php
$doc = Document::find(1);

// Ask about this document
$answer = $doc->ask("What is the main point?");

// Get a summary
$summary = $doc->summarize(maxWords: 100);

// Generate tags
$tags = $doc->generateTags(maxTags: 5); // Returns array
```

### RAG Configuration

**Namespace:** config key `ai-engine.rag`

Key settings:


### Chunking Large Content

When content is too large for a single embedding, it's automatically split into chunks:

```php
'vectorization' => [
    'strategy' => env('AI_ENGINE_VECTORIZATION_STRATEGY', 'split'), // 'split' or 'truncate'
    'chunk_size' => env('AI_ENGINE_CHUNK_SIZE', null), // Auto-calculated
    'chunk_overlap' => env('AI_ENGINE_CHUNK_OVERLAP', 200), // Chars of overlap
    'max_content_length' => env('AI_ENGINE_MAX_CONTENT_LENGTH', null), // null = auto
],
```

- **`split`** (recommended): Large documents are split into overlapping chunks, each embedded separately. Better for RAG.
- **`truncate`**: Truncates to max size. Faster but loses information.

## Generating & Understanding Images

The Laravel AI Engine supports image generation via multiple specialized providers (Stable Diffusion, fal_ai, OpenAI's DALL-E, Clipdrop) and image understanding through GPT-4o vision capabilities.

### Generating Images

Image generation is performed via `AIRequest` with the appropriate engine and model. The response includes stored URLs pointing to the generated images.

**Example: DALL-E 3 Image Generation**

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Enums\EngineEnum;
use LaravelAIEngine\Enums\EntityEnum;

$request = new AIRequest(
    prompt: 'A serene mountain landscape at sunset',
    engine: EngineEnum::OpenAI,
    model: EntityEnum::DALL_E_3,
    parameters: [
        'image_count' => 1,
        'size' => '1024x1024',
        'quality' => 'standard', // 'standard' or 'hd'
    ]
);

$response = app(LaravelAIEngine\Services\AIEngine::class)->generate($request);

if ($response->isSuccessful()) {
    $imageUrls = $response->getFiles(); // Array of stored image URLs
    dd($imageUrls);
}
```

**Example: Stable Diffusion (via Stability AI)**

```php
$request = new AIRequest(
    prompt: 'A futuristic cityscape with neon lights',
    engine: EngineEnum::StableDiffusion,
    model: EntityEnum::SD3_LARGE, // or SD3_MEDIUM, SDXL_1024
    parameters: [
        'image_count' => 1,
        'width' => 1024,
        'height' => 1024,
        'steps' => 30,
        'cfg_scale' => 7.0,
        'negative_prompt' => 'blurry, low quality', // Optional
        'seed' => 12345, // Optional: for reproducible results
    ]
);

$response = app(LaravelAIEngine\Services\AIEngine::class)->generate($request);
$imageUrls = $response->getFiles();
```

**Example: FAL AI (FLUX, Stable Diffusion 3, or Video)**

```php
$request = new AIRequest(
    prompt: 'A golden retriever playing in a sunny garden',
    engine: EngineEnum::FalAI,
    model: EntityEnum::FAL_FLUX_PRO, // FAL_FLUX_DEV, FAL_FLUX_SCHNELL, FAL_SDXL, FAL_SD3_MEDIUM
    parameters: [
        'image_count' => 1,
        'steps' => 50,
        'guidance_scale' => 7.5,
        'negative_prompt' => 'distorted, unrealistic',
        'seed' => 42,
    ]
);

$response = app(LaravelAIEngine\Services\AIEngine::class)->generate($request);
$imageUrls = $response->getFiles();
```

**Configuration**: Ensure the driver's API key is set in your `.env`:
- OpenAI: `OPENAI_API_KEY`
- Stable Diffusion: Configure via `config('ai-engine.engines.stable-diffusion.api_key')` (Stability AI API)
- FAL AI: `FAL_AI_API_KEY`

### Image Editing & Operations

Use `ImageOperationService` for editing existing images or apply operations via Clipdrop (background removal, upscaling, cleanup, etc.).

**Example: Clipdrop Image Operations**

```php
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Enums\EngineEnum;
use LaravelAIEngine\Enums\EntityEnum;

// Background removal
$request = new AIRequest(
    prompt: '', // Not used for image operations
    engine: EngineEnum::Clipdrop,
    model: EntityEnum::CLIPDROP_IMAGE_EDIT,
    parameters: [
        'operation' => 'background_removal',
        'image' => '/path/to/image.png', // Local path, data: URI, or base64
    ]
);

$response = app(LaravelAIEngine\Services\AIEngine::class)->generate($request);
$editedImageUrl = $response->getFiles()[0] ?? null;
```

Supported Clipdrop operations: `background_removal`, `cleanup`, `upscale`, `sketch_to_image`, `reimagine`, `remove_text`. Pass the operation name in `parameters['operation']` and provide the image via `parameters['image']`. For `cleanup` and `upscale`, add `parameters['mask']` (cleanup) or `parameters['target_width']` / `parameters['target_height']` (upscale).

### Understanding Images: Vision Analysis

Extract text and analyze images using GPT-4o vision capabilities through the `FileAnalysisService`.

**Example: Extract Text from an Image**

```php

```

**Example: Analyze an Uploaded Image File**

```php
use LaravelAIEngine\Services\FileAnalysisService;

$service = app(FileAnalysisService::class);

$uploadedImage = request()->file('image'); // Illuminate\Http\UploadedFile

$analysis = $service->analyzeImage(
    file: $uploadedImage,

// Returns array with keys:
// - 'content': AI-extracted analysis
```

```php
],
```

The Laravel AI Engine provides a comprehensive video generation system powered by FAL AI, supporting text-to-video, image-to-video, and reference-based video generation workflows with both synchronous and asynchronous processing modes.

### Synchronous Video Generation

For straightforward video generation, use the `GenerateVideoService` with a direct synchronous request:

```php
use LaravelAIEngine\Services\AIEngineService;
use LaravelAIEngine\DTOs\AIRequest;
use LaravelAIEngine\Enums\EngineEnum;
use LaravelAIEngine\Enums\EntityEnum;

$ai = app(AIEngineService::class);

$response = $ai->generateDirect(new AIRequest(
    prompt: 'A serene landscape with mountains and a flowing river at sunset',
    engine: EngineEnum::FalAI,
    model: EntityEnum::FAL_SEEDANCE_2_TEXT_TO_VIDEO,
    parameters: [
        'duration' => '5',
        'aspect_ratio' => '16:9',
        'seed' => 42,
    ],
    userId: auth()->id(),
));

if ($response->isSuccessful()) {
    $videoUrl = $response->getFiles()[0] ?? null;
    // Use the video URL...
}
```

### Asynchronous Video Generation (Recommended for Long-Running Tasks)

For production use, leverage asynchronous processing with job tracking and webhooks:

```php
use LaravelAIEngine\Services\Media\GenerateVideoService;

$videoService = app(GenerateVideoService::class);

$submission = $videoService->generate([
    'async' => true,
    'prompt' => 'A dancer performing under neon lights in an urban environment',

    'model' => 'bytedance/seedance-2.0/text-to-video',
    'duration' => '6',
    'aspect_ratio' => '9:16',
    'use_webhook' => true, // Enable webhook notifications
]);

// Response contains job_id for tracking
$jobId = $submission['job_id'];
```

### Checking Job Status

Poll or wait for async job completion:

```php
$videoService = app(GenerateVideoService::class);

$status = $videoService->status($jobId, refresh: true);

if ($status['status'] === 'completed') {
    // Job finished successfully
    $videoUrl = $status['data']['metadata']['response']['files'][0] ?? null;

    // Handle failure
    $error = $status['data']['metadata']['provider']['error'] ?? 'Unknown error';
}
```

## Characters, Personas & Sub-agents

Laravel AI Engine supports delegating work to specialized sub-agents through a goal-oriented architecture. Each sub-agent can have a distinct personality or role (name, description, capabilities) and handle tasks either through tool execution, conversational reasoning, or custom callables.

### Registering and Configuring Sub-agents

Sub-agents are registered in `config/ai-agent.php` under the `sub_agents` array. Each sub-agent definition includes a handler (how it executes) and optional tools/capabilities:

```php
// config/ai-agent.php
'sub_agents' => [
    'researcher' => [
        'name' => 'Research Specialist',
        'description' => 'Gathers information, analyzes sources, and summarizes findings.',
        'capabilities' => ['research', 'analysis', 'summarization'],
        'keywords' => ['find', 'analyze', 'research'],
        'handler' => \LaravelAIEngine\Services\Agent\SubAgents\ConversationalSubAgentHandler::class,
    ],
    'writer' => [
        'name' => 'Content Writer',
        'description' => 'Drafts clear, compelling written content.',
        'capabilities' => ['writing', 'content', 'editing'],
        'handler' => \LaravelAIEngine\Services\Agent\SubAgents\ConversationalSubAgentHandler::class,
    ],
    'validator' => [
        'name' => 'Quality Validator',
        'description' => 'Reviews and validates work for accuracy and completeness.',
        'capabilities' => ['validation', 'review', 'quality'],
        'tools' => ['validate_field', 'suggest_value'],
        'handler' => \LaravelAIEngine\Services\Agent\SubAgents\ToolCallingSubAgentHandler::class,
    ],
],
```

### Sub-agent Handler Types

The `handler` field determines how a sub-agent executes. Three built-in handlers are available:

**ConversationalSubAgentHandler** — The sub-agent conducts multi-turn reasoning via the LLM:
```php
'handler' => \LaravelAIEngine\Services\Agent\SubAgents\ConversationalSubAgentHandler::class,
```

**ToolCallingSubAgentHandler** — The sub-agent runs a sequence of registered tools (deterministic):
```php
'tools' => ['search_knowledge', 'find_customer'],
'handler' => \LaravelAIEngine\Services\Agent\SubAgents\ToolCallingSubAgentHandler::class,
```

**CallableSubAgentHandler** — Use a custom callable (closure or invokable class):
```php
'handler' => function (SubAgentTask $task, UnifiedActionContext $context, array $previousResults, array $options) {
    return SubAgentResult::success(
        $task->id,
        $task->agentId,
        'Custom logic executed.'
    );
},
```

### Running Sub-agents via Goal Agent

The `run_sub_agent` tool (enabled by default via `goal_agent.register_sub_agent_tool`) allows the agent to delegate targets to sub-agents at runtime:

```php
$response = app(\LaravelAIEngine\Services\Agent\GoalAgentService::class)->execute(
    target: 'Research the impact of AI on healthcare and draft a summary.',
    context: $unifiedActionContext,
    options: [
        'max_sub_agents' => 2,
        'stop_on_failure' => true,
    ]
);

// Response contains plan and per-sub-agent results
$plan = $response->data['plan']; // Tasks matched to sub-agents
$results = $response->data['results']; // SubAgentResult for each task
```

Internally, the planner matches the target against registered sub-agents' `capabilities` and `keywords`, then executes matched tasks in order.

### Inline Sub-agent Requests

Pass sub-agents directly in the request's `options` array without relying on registered definitions:

```php
$response = app(\LaravelAIEngine\Services\Agent\GoalAgentService::class)->execute(
    target: 'Prepare a presentation outline.',
    context: $context,
    options: [
        'sub_agents' => [
            [
                'agent_id' => 'structure_lead',
                'name' => 'Structure Specialist',
                'objective' => 'Design the outline structure.',
                'input' => ['style' => 'business'],
            ],
            [
                'agent_id' => 'content_lead',
                'name' => 'Content Specialist',
                'objective' => 'Fill in talking points.',
                'input' => ['source' => 'research_docs'],
                'depends_on' => ['structure_lead'],
            ],
        ],
    ]
);
```

Each inline sub-agent can reference previous results via `depends_on`.

### Customizing Sub-agent Prompts and Personas

For conversational sub-agents, the prompt is built from the sub-agent's name, description, objective, and any input. To inject a custom system prompt or persona, use the sub-agent's `metadata` field or task input:

```php
'sub_agents' => [
    'brand_voice' => [
        'name' => 'Brand Voice Specialist',
        'description' => 'Ensures all content matches our brand tone and values.',
        'capabilities' => ['branding', 'voice', 'tone'],
        'handler' => ConversationalSubAgentHandler::class,
        // metadata is passed into each task and available to the handler
        'metadata' => [
            'system_prompt' => 'You are a brand strategist. Speak in a professional yet approachable tone. Emphasize values of innovation and trust.',
            'tone_guide' => 'Friendly, clear, authoritative',
        ],
    ],
],
```

When a task is created for this sub-agent, its metadata flows through the `SubAgentTask` DTO and can be accessed by the handler to inject custom instructions or context.

### Task Dependencies

Sub-agents can depend on results from previous sub-agents:

```php
'sub_agents' => [
    [
        'agent_id' => 'data_gatherer',
        'name' => 'Data Gatherer',
        'objective' => 'Collect all relevant information.',
    ],
    [
        'agent_id' => 'analyst',
        'name' => 'Analyst',
        'objective' => 'Analyze the collected data.',
        'depends_on' => ['data_gatherer'],
    ],
    [
        'agent_id' => 'presenter',
        'name' => 'Presenter',
        'objective' => 'Create a summary for stakeholders.',
        'depends_on' => ['analyst'],
    ],
],
```

The executor validates all dependencies before running and injects previous results into the input of dependent tasks.

### Configuration Options

Key settings for sub-agent execution (in `config/ai-agent.php`):

```php
'goal_agent' => [
    'enabled' => env('AI_AGENT_GOAL_ENABLED', true),
    'max_sub_agents' => env('AI_AGENT_GOAL_MAX_SUB_AGENTS', 5),    // Max agents matched per target
    'stop_on_failure' => env('AI_AGENT_GOAL_STOP_ON_FAILURE', true), // Stop on critical failure
```

## File Intake (upload → extract → suggest → pre-fill)

The file intake workflow—upload a document, extract its text, detect what entity to create from it, and optionally pre-fill the creation form—is handled by the `analyze_file` agent tool, backed by `FileAnalysisService` and `StructuredFileExtractor`. This is generic and configuration-driven: suggest creating an invoice, a customer, a product, or any entity your app defines, all driven by simple regex patterns in config.

### Enable File Analysis

File analysis is **opt-in**. In `config/ai-engine.php`, set:

```php
'file_analysis' => [
    'enabled' => env('AI_ENGINE_FILE_ANALYSIS', false),
    'base_path' => env('AI_ENGINE_FILE_ANALYSIS_PATH', storage_path('app/uploads')),
    'allowed_extensions' => ['pdf', 'doc', 'docx', 'txt', 'csv', 'xlsx', 'jpg', 'jpeg', 'png'],
    'max_bytes' => 10 * 1024 * 1024,  // 10 MB
    'validate_actions' => true,  // Only suggest actions that exist as registered tools
    'prefill' => env('AI_ENGINE_FILE_ANALYSIS_PREFILL', false),  // Extract fields for pre-fill
    'vision_model' => 'gpt-4o',  // For OCR on images
],
```

Or set via environment:

```bash
AI_ENGINE_FILE_ANALYSIS=true
AI_ENGINE_FILE_ANALYSIS_PATH=/storage/uploads
AI_ENGINE_FILE_ANALYSIS_PREFILL=false
```

### Configure Keyword Suggestions

Define what documents suggest which create actions. For example, suggest `create_invoice` when an uploaded file mentions "invoice":

```php
'file_analysis' => [
    // ...
    'keyword_suggestions' => [
        [
            'pattern' => '/\binvoice\b/i',
            'action_id' => 'create_invoice',
            'action_label' => 'Create an invoice from this file',
            'confidence' => 80,
            'detected_fields' => ['invoice_number', 'total', 'due_date'],
        ],
        [
            'pattern' => '/\bcustomer|contact|account\b/i',
            'action_id' => 'create_customer',
            'action_label' => 'Create a customer from this file',
            'confidence' => 70,
            'detected_fields' => ['name', 'email', 'phone'],
        ],
        [
            'pattern' => '/\bproduct|item|sku\b/i',
            'action_id' => 'create_product',
            'action_label' => 'Add a product from this file',
            'confidence' => 75,
            'detected_fields' => ['name', 'sku', 'price', 'category'],
        ],
    ],
],
```

Each pattern is checked against the extracted file content. If it matches and the `action_id` is a registered agent tool (when `validate_actions => true`), the suggestion is included in the response.

### Basic Usage: Extract & Suggest

Use the `analyze_file` tool in an agent prompt or call the service directly:

```php
use LaravelAIEngine\Services\FileAnalysisService;

$service = app(FileAnalysisService::class);

// Extract text + suggest actions from a stored file path.
// (The analyze_file agent tool uses this internally.)
$result = $service->extractAndSuggestFromPath(
    path: 'path/to/file.pdf',
    extension: 'pdf',
    name: 'invoice.pdf'
);

// Returns:
// [
//     'content' => 'INVOICE #123...',
//     'suggestions' => [
//         [
//             'action_id' => 'create_invoice',
//             'action_label' => 'Create an invoice from this file',
//             'confidence' => 80,
//             'reason' => 'File matched configured keywords.',
//             'detected_fields' => ['invoice_number', 'total'],
//         ],
//     ],
// ]
```

### Image & Vision Model Support

For images (JPG, PNG, WebP), the service uses GPT-4o's vision to transcribe text:

```php
// Extract text from a scanned/photographed invoice:
$text = $service->extractImageText('/path/to/scan.jpg');
// Uses config('ai-engine.file_analysis.vision_model', 'gpt-4o')
```

```php
'file_analysis' => [

    'vision_max_tokens' => 2000,
```

### Pre-fill Form Fields (Phase 2)

When `prefill => true`, the agent extracts each suggested create action's fields from the file content, so the creation form can be pre-filled. This runs one model call per suggested action (at model `gpt-4o-mini` by default, per `orchestration_model`):

```php
'file_analysis' => [
    'prefill' => true,  // Enable extraction
    'max_content_chars' => 12000,  // Truncate file for extraction
    'extraction_max_tokens' => 1500,
],
```

When enabled, suggestions include a `prefill` key:

```php
// Result with prefill enabled:
[
    'suggestions' => [
        [
            'action_id' => 'create_invoice',
            'action_label' => 'Create an invoice from this file',
            'confidence' => 80,
            'prefill' => [
                'invoice_number' => 'INV-2024-001',
                'customer_name' => 'Acme Corp',
                'total' => '1200.00',
                'due_date' => '2024-12-31',
            ],
        ],
    ],
]
```

The extracted fields are determined automatically from the create action's parameter list; no extra config needed.

### Security & Sandboxing

File analysis is sandboxed:

- **Base Path**: Only files under `ai-engine.file_analysis.base_path` (default: `storage('app/uploads')`) are readable.
- **Extension Allowlist**: Only extensions in `allowed_extensions` are accepted (default: `pdf`, `doc`, `docx`, `txt`, `csv`, `xlsx`, `jpg`, `jpeg`, `png`, `webp`, `gif`).
- **Size Cap**: Files larger than `max_bytes` (default: 10 MB) are rejected.
- **Path Traversal**: `../../etc/hosts` and absolute paths outside the sandbox are rejected.
- **Action Validation**: When `validate_actions => true`, only suggestions matching registered agent tools are returned, preventing undefined actions.

### Direct Service Usage

All extraction methods on `FileAnalysisService`:

```php
$service = app(FileAnalysisService::class);

// Extract text from a path (no suggestions):

// Extract text + suggestions (preferred):
$result = $service->extractAndSuggestFromPath('/path/to/file.pdf', 'pdf', 'invoice.pdf');
// ['content' => '...', 'suggestions' => [...]]

// OCR an image:
$text = $service->extractImageText('/path/to/scan.jpg');

// Suggest actions for any content string:
$suggestions = $service->suggestActions('this is an invoice...', 'invoice.pdf');
```

The `StructuredFileExtractor` is called by `analyze_file` when `prefill => true`. It extracts specific fields:

```php
    content: 'INVOICE #123\nTotal: $500\nDue: 2024-12-31',
```


### Namespace & Classes

All core classes reside in the `LaravelAIEngine\` namespace:

### Example Workflow

1. **User uploads a file** → it's stored in `storage/app/uploads/inv-123.pdf`.
2. **Agent calls `analyze_file`** with `path: 'inv-123.pdf'`.
3. **Service extracts text** from the PDF and detects "invoice" keyword.
4. **Suggests `create_invoice`** (if registered and `validate_actions => true`).
5. **If `prefill => true`**: Extracts invoice fields (number, customer, total, due date) and includes them in the suggestion.
6. **Agent uses the suggestion** to call `create_invoice` with pre-filled data or ask the user to confirm.

## Scaling Tool Context

As your tool registry grows (through AiResource, custom tools, or MCP integrations), every tool's full JSON schema is injected into the planner's prompt, inflating context and costs. The Laravel AI Engine provides configurable **tool selection strategies** and **progressive disclosure** to keep the prompt efficient while ensuring the right tools are available.

### Tool Selection Strategies

Configure which tools the planner sees each turn under `ai-agent.ai_native.tool_selection.strategy`:

**All (default)** — No filtering; expose every registered tool:
```php
// config/ai-agent.php
'ai_native' => [
    'tool_selection' => [
        'strategy' => 'all',
        'always' => ['search_knowledge', 'data_query'],
    ],
],
```

**Skill-scoped** — When a skill is active, expose only that skill's declared tools plus a core set:
```php
'tool_selection' => [
    'strategy' => 'skill_scoped',
    'always' => ['search_knowledge', 'data_query'],
],
```
This is the lowest-risk reduction—the active skill is re-matched every turn, so topic changes restore the full set.

**Keyword** — Lexical top-K selection: score tools by keyword overlap with the turn message and expose the best N:
```php
'tool_selection' => [
    'strategy' => 'keyword',
    'limit' => 12,  // Expose top 12 matching tools per turn
    'always' => ['search_knowledge', 'data_query'],
],
```
Cheap and deterministic (no embeddings). Falls back to all tools if no lexical signal is detected.

**Semantic** — Embedding-based top-K selection: embed the message and each tool's name+description, rank by cosine similarity:
```php
'tool_selection' => [
    'strategy' => 'semantic',
    'limit' => 12,
    'always' => ['search_knowledge', 'data_query'],
],
```
Most expensive (one embedding call per turn), but most accurate for large, unrelated tool registries. Tool embeddings are cached (by name and description), so only the message is embedded each turn. Fails open: if embedding is unavailable, all tools are exposed.

### Progressive Disclosure

When you have many tools, progressive disclosure keeps the base prompt small by listing tools by name + one-line summary only. The planner calls the `find_tools` meta-tool to load a tool's full parameter schema on demand, just-in-time:

```php
'tool_selection' => [
    'disclosure' => 'progressive',  // 'full' (default) or 'progressive'
],
```

With `disclosure => 'progressive'`:
- The prompt lists each tool as `{ "name": "create_invoice", "description": "Create a new invoice" }` instead of the full schema.
- The `find_tools` tool is always available and takes a natural-language query (e.g., "how do I create an invoice?") or a tool name.
- The planner calls `find_tools` to retrieve the full parameter schema for tools it needs.
- Tool embeddings for `find_tools` matches are cached.

Example: a planner seeing 50 tools listed by summary alone, then calling `find_tools("find invoices by customer")` to load just the schemas for `find_invoice` and `search_invoices` before using them.

### Combining Strategies and Disclosure

You can layer both together for maximum efficiency:

```php
'ai_native' => [
    'tool_selection' => [
        'strategy' => 'semantic',      // Rank by embedding similarity
        'limit' => 8,                  // Expose top 8 per turn
        'disclosure' => 'progressive', // List by name + summary only
        'always' => ['search_knowledge', 'data_query', 'find_tools'],
    ],
],
```

This combination:
1. **Selects** the top 8 semantically-relevant tools per turn (semantic + limit).
2. **Discloses** each as name + summary in the prompt (progressive).
3. **Exposes** `find_tools` unconditionally so the planner can load additional schemas on demand (always).

### Configuration Reference

All options in `config/ai-agent.php`:
```php
'tool_selection' => [
    /**
     * 'all' (default) — expose every tool
     * 'skill_scoped' — when a skill is active, expose its tools + always core
     * 'keyword' — lexical top-K by message overlap
     * 'semantic' — embedding-based top-K (cached tool embeddings, one message embedding/turn)
     */
    'strategy' => env('AI_AGENT_TOOL_SELECTION', 'all'),
    
    /**
     * Tools always exposed, regardless of strategy.
     * Default: ['search_knowledge', 'data_query']
     */
    'always' => ['search_knowledge', 'data_query'],
    
    /**
     * For keyword and semantic strategies: max tools to expose per turn (default 12)
     */
    'limit' => (int) env('AI_AGENT_TOOL_SELECTION_LIMIT', 12),
    
    /**
     * 'full' (default) — inject each selected tool's complete JSON schema
     * 'progressive' — list tools by name + summary; planner calls find_tools to load full schemas on demand
     */
    'disclosure' => env('AI_AGENT_TOOL_DISCLOSURE', 'full'),
],
```

### Behind the Scenes

The selection logic lives in `\LaravelAIEngine\Services\Agent\Tools\Selectors\`:
- **ToolSelectorContract** — Interface that all selectors implement.
- **AllToolSelector** — Default; returns unfiltered tools.
- **SkillScopedToolSelector** — Reads the active skill's `tools`, `final_tool`, and relation lookup/create tools from the skill registry, plus the `always` core.
- **KeywordToolSelector** — Tokenizes the message, scores tools by name/description overlap (3 points for exact token match in name, 2 for substring, 1 for description), sorts by score, returns top N plus always core.
- **SemanticToolSelector** — Embeds the message and each tool's `name + description`, caches tool embeddings per (name, description) pair, ranks by cosine similarity, returns top N plus always core.

The `\LaravelAIEngine\Services\Agent\Tools\FindToolsTool` implements the `find_tools` meta-tool, which searches all registered tools by keyword (same scoring as KeywordToolSelector) and returns the full schemas of matches.

Progressive disclosure is applied in `\LaravelAIEngine\Services\Agent\AiNative\AiNativePromptBuilder::toolDocuments()`: when enabled, each tool (except `find_tools`) is reduced to `{ "name": "...", "description": "..." }` before being injected into the prompt.

## Structured Data & Analytics

The Laravel AI Engine provides two powerful tools for exact data queries and analytics: `data_query` (for counts, lists, and filtering) and `aggregate_data` (for sums, averages, rankings, and per-group breakdowns). These tools run in SQL rather than estimating from samples, ensuring accuracy for analytics questions.

### Overview

**`data_query` tool** answers "how many", "list", and "filter" questions by returning exact counts and paginated record lists from your Eloquent models. It auto-detects status, date ranges ("this month", "last 7 days"), and applies exact field filters via the `filters` parameter.

**`aggregate_data` tool** computes SQL aggregations (sum/avg/min/max), returns top/bottom ranked records, and groups by dimensions—answering "total revenue", "which customer spent the most", "average order value" with real numbers, not estimates.

Both tools are **fail-closed by default**: queries fail if no access scope (user_id, workspace_id, or tenant_id) applies, unless the model is marked `public => true`. This prevents accidental data leaks in multi-tenant scenarios.

### Configuration

Enable and configure data querying in `config/ai-engine.php`:

```php
'data_query' => [
    'enabled' => env('AI_ENGINE_DATA_QUERY_ENABLED', true),
    // Explicit model registry: controls keywords, filters, and aggregation columns
    'models' => [
        'invoice' => [
            'class' => App\Models\Invoice::class,
            'aliases' => ['invoice', 'invoices'],
            'list' => ['id', 'invoice_number', 'total', 'status', 'due_date'],
            'status_column' => 'status',
            'statuses' => ['paid', 'overdue', 'draft'],
            'date_column' => 'created_at',
            'filterable' => ['status', 'customer_id'],
            'aggregatable' => ['total', 'tax', 'discount'],
            'groupable' => ['customer_id', 'status'],
            'metric_aliases' => ['revenue' => 'total', 'taxes' => 'tax'],
            'public' => false,  // Fail-closed: require user/workspace/tenant scope
        ],
    ],
    // When 'models' is empty, auto-discover queryable models from your RAG setup
    'use_discovery' => env('AI_ENGINE_DATA_QUERY_DISCOVERY', true),
    // Scope columns checked in order; applied if present on the table
    'scope_columns' => ['user_id', 'workspace_id', 'tenant_id'],
    // When true (default), queries fail if no access scope can be applied
    'require_scope' => env('AI_ENGINE_DATA_QUERY_REQUIRE_SCOPE', true),
    // Maximum rows returned per list/top/bottom query
    'max_limit' => env('AI_ENGINE_DATA_QUERY_MAX_LIMIT', 50),
],
```

### Using `data_query` for Counts and Lists

The agent automatically routes structured queries to `data_query` when it detects counting or listing questions.

```php
// In your agent chat, the user might ask:
// "how many invoices do we have?"
// "list active customers"
// "show me draft invoices for Apollo Labs"

// The agent calls the tool like this:
$tool = new \LaravelAIEngine\Services\Agent\Tools\DataQueryTool();

$result = $tool->execute([
    'query' => 'how many invoices do we have',
], $context);

// Result: ActionResult with operation='count', count=127
// Respects the user's scope (user_id, workspace_id, etc.)
```

**Filters**: Pass exact field values in the `filters` object to narrow results. Only columns listed in `filterable` or `groupable` are applied.

```php
$result = $tool->execute([
    'query' => 'list invoices',
    'filters' => ['customer_id' => 'cust_123', 'status' => 'overdue'],
    'limit' => 20,
], $context);

// Returns: ['operation' => 'list', 'rows' => [...], 'count' => 5]
```

**Automatic Status & Date Filters**: The tool infers status filters from configured values and date ranges from phrases like "today", "this week", "last 30 days":

```php
// User: "how many paid invoices were created this month?"
// The tool detects:
//   - status = 'paid' (from config statuses)
//   - created_at BETWEEN [month start] AND [month end]
$result->data['status'];  // 'paid'
$result->data['date'];    // 'this month'
$result->data['count'];   // Exact count for that subset
```

**Fail-Closed Scoping**: By default, queries without an applicable scope are refused to prevent data leaks:

```php
// If the model has user_id but no user is authenticated:
$context = new UnifiedActionContext('dq', userId: null);
$result = $tool->execute(['query' => 'how many invoices'], $context);
// Result: failure - "Access blocked: no scope could be applied"

// Allow intentionally-unscoped models (e.g., public product catalog):
'product' => [
    'class' => App\Models\Product::class,
    'public' => true,  // Bypass require_scope check
    // ...
],
```

### Using `aggregate_data` for Analytics

The agent routes analytics questions (totals, averages, rankings) to `aggregate_data` for exact calculations.

```php
$tool = new \LaravelAIEngine\Services\Agent\Tools\AggregateQueryTool();

// Example 1: Total revenue across all invoices
$result = $tool->execute([
    'query' => 'what is total revenue for all invoices',
], $context);
// Result: operation='sum', metric='total', value=125000.50

// Example 2: Average invoice value
$result = $tool->execute([
    'query' => 'average invoice amount',
], $context);
// Result: operation='avg', metric='total', value=1234.56

// Example 3: The 5 most expensive invoices
$result = $tool->execute([
    'query' => 'list the top invoices by amount',
    'limit' => 5,
], $context);
// Result: operation='top', rows=[...full rows...], metric='total', direction='desc'

// Example 4: Per-customer breakdown
$result = $tool->execute([
    'query' => 'how much has each customer spent',
    'operation' => 'sum',
    'metric' => 'total',
    'group_by' => 'customer_id',
], $context);
// Result: groups=[
//   ['key' => 'apollo_labs', 'value' => 50000, 'count' => 5],
//   ['key' => 'mozilla', 'value' => 45000, 'count' => 3],
// ]
```

**Operations Supported**:
- **Scalar**: `sum`, `avg`, `min`, `max`, `count` → returns a single number
- **Ranked**: `top`, `bottom` → returns actual records ranked by a metric
- **Count Distinct**: `count` with `group_by` → distinct dimension values

**Metric Aliases**: Use friendly names in natural language; they resolve to database columns:

```php
'metric_aliases' => [
    'revenue' => 'total',
    'taxes' => 'tax',
    'profit' => 'net_amount',
],

// User: "what is our total revenue for paid invoices this month?"
// Tool resolves: metric='total' (from alias 'revenue')
```

**Filters & Status**: Just like `data_query`, filters on allowlisted columns and auto-detects status and date ranges:

```php
// User: "which paid customer has spent the most this quarter?"
$result = $tool->execute([
    'query' => 'which paid customer has spent the most this quarter',
], $context);

// Applies:
//   status = 'paid'
//   created_at in current quarter
//   operation = 'sum' (from "spent")
//   metric = 'total' (from 'revenue' alias or first aggregatable column)
//   group_by = 'customer_id' (inferred from "which customer")
```

**Structured-Only Calls**: The planner can call with structured parameters alone (no `query`):

```php
$result = $tool->execute([
    'entity' => 'invoice',
    'operation' => 'sum',
    'metric' => 'total',
    'filters' => ['status' => 'paid'],
    'group_by' => 'status',
], $context);
```

### Configuration Reference

| Key | Type | Default | Purpose |
|-----|------|---------|---------|
| `data_query.enabled` | bool | `true` | Enable the `data_query` and `aggregate_data` tools |
| `data_query.models` | array | `[]` | Model registry (keywords, columns, aggregation config) |
| `data_query.use_discovery` | bool | `true` | Auto-discover models from RAG setup if `models` is empty |
| `data_query.scope_columns` | array | `['user_id', 'workspace_id', 'tenant_id']` | Columns checked for access scoping |
| `data_query.require_scope` | bool | `true` | Fail-closed: refuse queries with no applicable scope |
| `data_query.max_limit` | int | `50` | Cap on rows returned per query |

### Model Configuration Schema

Each entry in `ai-engine.data_query.models` accepts:

```php
'invoice' => [
    'class' => App\Models\Invoice::class,              // Eloquent model class (required)
    'aliases' => ['invoice', 'invoices', 'inv'],       // Keywords to match in queries
    'label' => 'invoices',                             // Display name (auto from class if omitted)
    'public' => false,                                 // true: allow unscoped access
    'list' => ['id', 'number', 'total', 'status'],     // Columns shown in list/top queries
    'status_column' => 'status',                       // Column for status filtering
    'statuses' => ['paid', 'draft', 'overdue'],        // Known status values (auto-detected if empty)
    'date_column' => 'created_at',                     // Column for "today"/"this month" filters
    'filterable' => ['status', 'customer_id'],         // Columns allowed in filters param
    'aggregatable' => ['total', 'tax', 'discount'],    // Numeric columns for sum/avg/min/max
    'groupable' => ['customer_id', 'status', 'month'], // Dimension columns for group_by
    'metric_aliases' => [
        'revenue' => 'total',
        'taxes' => 'tax',
    ],
],
```

### Security & Best Practices

1. **Scope is fail-closed**: Unscoped queries are refused unless `public: true`. Explicitly trust multi-tenant models.
2. **Allowlists, not wildcards**: Only columns in `filterable`, `groupable`, or `aggregatable` can be queried or grouped—no arbitrary SELECT.
3. **Use exact filters for subset queries**: To answer "how much has customer X spent", pass `filters: {customer_id: 'x'}` so the total is for that subset, not the whole table.
4. **Date and status detection**: Built-in inference of "this month", "paid status" from natural language—no special setup needed if you name columns and values conventionally.

## Memory & Conversations

The laravel-ai-engine package provides two-layer conversation memory: transcript history for full chat logs, and durable scoped memories for key facts and preferences extracted during context compaction. Conversations are stored in the `ai_conversations` table with associated messages, while persistent memories live in `ai_conversation_memories` with optional semantic indexing for retrieval.

### Basic Conversation Setup

Create and manage conversations using the `ConversationManager`:

```php
use LaravelAIEngine\Services\ConversationManager;

$manager = app(ConversationManager::class);

// Create a conversation with optional system prompt
$conversation = $manager->createConversation(
    userId: auth()->id(),
    title: 'Customer Support Chat',
    systemPrompt: 'You are a helpful support agent. Always be professional.',
    settings: [
        'max_messages' => 50,
        'temperature' => 0.7,
        'auto_title' => true,
    ]
);

// Add messages (returns Message model)
$manager->addUserMessage($conversation->conversation_id, 'What are your hours?');

// Retrieve context (includes system prompt + recent messages formatted for the model)
$context = $manager->getConversationContext($conversation->conversation_id);

// Get stats on the conversation
$stats = $manager->getConversationStats($conversation->conversation_id);
// Returns: total_messages, user_messages, assistant_messages, total_tokens, total_credits, etc.
```

### Memory Drivers

The `MemoryManager` supports pluggable storage backends configured via `ai-engine.memory.default_driver`:

```php
use LaravelAIEngine\Services\Memory\MemoryManager;

$memory = app(MemoryManager::class);

// Use the default driver (usually 'database')
$memory->addMessage($conversationId, 'user', 'Hello!', ['custom_field' => 'value']);

// Explicitly use a different driver
$memory->addMessage($conversationId, 'user', 'Hello!', driver: 'redis');

// Get messages and context
$messages = $memory->getMessages($conversationId);
$context = $memory->getContext($conversationId);

// Create conversation (returns conversation_id string)
$convId = $memory->createConversation(
    userId: 'user123',
    title: 'Support Ticket #456',
    systemPrompt: 'You are helpful.',
    settings: ['engine' => 'openai', 'model' => 'gpt-4o']
);

// Clear history or delete entirely
$memory->clearConversation($conversationId);  // Delete all messages
$memory->deleteConversation($conversationId); // Mark inactive
```

Available drivers: `database` (default), `redis`, `file`, `mongodb` (if MongoDB library is installed).

### MemoryProxy Fluent Interface

For chained operations, use the `MemoryProxy`:

```php
use LaravelAIEngine\Services\MemoryProxy;

$proxy = app(MemoryProxy::class)
    ->conversation($conversationId)
    ->driver('redis'); // optional; defaults to config

// Add messages fluently
$proxy
    ->addUserMessage('What is your name?')

    ->addUserMessage('Nice to meet you!');
```

### Durable Conversation Memory (Scoped Facts)

Across compacted turns, the package extracts small facts and preferences into `ai_conversation_memories` for stable context without bloating the prompt:

```php
use LaravelAIEngine\Repositories\ConversationMemoryRepository;

$repo = app(ConversationMemoryRepository::class);

// Store or update a memory fact
$repo->upsert(
    namespace: 'preferences',
    key: 'language',
    value: 'Spanish',
    summary: 'User prefers Spanish language responses',
    scopeType: 'user',
    scopeId: auth()->id(),
    sessionId: $conversationId,
    confidence: 0.95
);

// Retrieve memories for a conversation/scope
$memories = $repo->search(
    namespace: 'preferences',
    scopeType: 'user',
    scopeId: auth()->id(),
    sessionId: $conversationId,
    limit: 6
);

// Each memory is an AIConversationMemory model with:
// - memory_id, namespace, key, value, summary, metadata
// - scope_type, scope_id, session_id, confidence
// - last_seen_at, expires_at (TTL)
```

### Configuration

Memory behavior is controlled via environment variables and `config/ai-agent.php`:

```env
# Enable/disable durable conversation memory extraction
AI_AGENT_CONVERSATION_MEMORY_ENABLED=true

# Storage driver for conversation transcripts
AI_AGENT_CONVERSATION_MEMORY_DRIVER=database

# Extract facts when context is compacted (before each agent turn)
AI_AGENT_MEMORY_EXTRACT_ON_COMPACTION=true

# Extraction method: 'ai' (model-backed, multilingual), 'none' (disabled), or custom class
AI_AGENT_MEMORY_EXTRACTOR=ai

# Memory retrieval limits per turn
AI_AGENT_MEMORY_MAX_PER_TURN=6
AI_AGENT_MEMORY_MAX_PROMPT_CHARS=1200

# Relevance filtering (0.0–1.0)
AI_AGENT_MEMORY_MIN_SCORE=0.45

# Auto-expire memories after N days
AI_AGENT_MEMORY_TTL_DAYS=180

# Context compaction (keeps recent turns verbatim, compacts older ones)
AI_AGENT_CONTEXT_COMPACTION_ENABLED=true
AI_AGENT_CONTEXT_MAX_MESSAGES=12
AI_AGENT_CONTEXT_KEEP_RECENT_MESSAGES=6
AI_AGENT_CONTEXT_MAX_TOTAL_CHARS=12000
```

### Semantic Indexing (Optional Vector Search)

For faster retrieval of relevant memories, opt-in to semantic (vector) indexing:

```env
# Enable vector-backed memory search
AI_AGENT_MEMORY_SEMANTIC_ENABLED=true

# Vector driver and collection name
AI_AGENT_MEMORY_SEMANTIC_DRIVER=qdrant
AI_AGENT_MEMORY_SEMANTIC_COLLECTION=ai_conversation_memories

# Index memories on every write (vs. only on compaction)
AI_AGENT_MEMORY_SEMANTIC_INDEX_ON_WRITE=false

# Scope fields to include in vector metadata (for filtering)
AI_AGENT_MEMORY_VECTOR_SCOPE_FIELDS=scope_type,scope_id,session_id,namespace
```

The package falls back to lexical (SQL `LIKE`) search if the vector driver is unavailable.

### Memory Policy

The `ConversationMemoryPolicy` class centralizes configuration checks. Use it in custom extractors or memory handlers:

```php
use LaravelAIEngine\Services\Agent\Memory\ConversationMemoryPolicy;

    $maxMemories = $policy->maxMemoriesPerTurn();      // 6 (default)
```

## Streaming & Realtime

Stream agent run events live via Server-Sent Events (SSE), or establish realtime voice/tool sessions with OpenAI, Google Gemini, or LiveKit while keeping provider keys and execution policy server-side in Laravel.

### SSE Streaming for Agent Runs

When an agent run is created asynchronously (with `async=true`), you can stream its lifecycle events to the browser in real-time. The `AgentRunSseStreamService` polls the run's persisted event stream and emits new events:

```php
use LaravelAIEngine\Services\Agent\AgentRunSseStreamService;

$sseService = app(AgentRunSseStreamService::class);

// Returns a StreamedResponse (SSE 200 with text/event-stream headers)
return $sseService->response($runId, [
    'timeout' => 60,           // Max seconds to hold connection
    'poll' => 500,             // Poll interval in milliseconds
    'auth_user_id' => auth()->id(),  // Authorization check
]);
```

In your route (already registered at `GET /api/v1/ai/agent-runs/{run}/stream`):

```php
// JavaScript client
const events = new EventSource('/api/v1/ai/agent-runs/abc123/stream?timeout=60&poll=500');

events.addEventListener('run.started', (event) => {
    const payload = JSON.parse(event.data);
    console.log('Run started:', payload.payload);
});

events.addEventListener('tool.completed', (event) => {
    const payload = JSON.parse(event.data);
    console.log('Tool completed:', payload.payload);
});

events.addEventListener('run.completed', (event) => {
    console.log('Run finished');
    events.close();
});
```

Events include: `run.started`, `agent.reasoning`, `tool.started`, `tool.completed`, `tool.failed`, `rag.started`, `rag.sources_found`, `rag.completed`, `approval.required`, `final_response.token_streamed`, `final_response.stream_completed`, `run.completed`, `run.failed`, `run.cancelled`, `run.expired`.

**Config:**
- `ai-agent.event_stream.sse.enabled` (default: `true`)
- `ai-agent.event_stream.sse.max_seconds` (default: `30`)
- `ai-agent.event_stream.sse.poll_milliseconds` (default: `500`)
- `ai-agent.event_stream.sse.heartbeat_seconds` (default: `10`)
- `ai-agent.event_stream.sse.authorize_owned_runs` (default: `true`) — only owners can stream their runs
- `ai-agent.event_stream.sse.allow_anonymous_runs` (default: `false`)
- `ai-agent.event_stream.sse.authorizer` — custom callable `fn($run, $options): bool`

### Token-Level Streaming (Final Responses)

For conversational agent runs, you can stream the final response token-by-token into the event stream so the browser "types out" the reply live:

```env
AI_AGENT_FINAL_RESPONSE_STREAMING_ENABLED=true
```

The `AgentFinalResponseStreamingService` yields each token while emitting `final_response.token_streamed` and `final_response.stream_completed` events:

```php
use LaravelAIEngine\Services\Agent\AgentFinalResponseStreamingService;
use LaravelAIEngine\DTOs\AIRequest;

$streaming = app(AgentFinalResponseStreamingService::class);

// Returns a Generator yielding each token
foreach ($streaming->stream(
    AIRequest::text('Summarize this article'),
    $run,
    $step,
    ['engine' => 'openai', 'model' => 'gpt-4o-mini']
) as $token) {
    echo $token;
}
```

### Realtime Sessions (WebRTC/LiveKit/Fallback)

Create a provider-agnostic realtime session descriptor—useful for browser voice chat, transcription, or tool-driven conversations:

```php
use LaravelAIEngine\DTOs\RealtimeSessionConfig;
use LaravelAIEngine\Services\SDK\RealtimeSessionService;

$sessionService = app(RealtimeSessionService::class);

$session = $sessionService->create(
    RealtimeSessionConfig::make('openai', 'gpt-realtime')
        ->voiceChat()
        ->withVoice('marin')
        ->withInputAudioTranscription(['model' => 'gpt-realtime-whisper'])
        ->withTurnDetection(['type' => 'server_vad'])
        ->withMaxResponseOutputTokens(512)
);
// Returns: ['provider' => 'openai', 'connect' => [...], 'capabilities' => [...], 'session' => [...], ...]
```

**Route:**
- `POST /api/v1/ai/realtime/sessions` — create a session descriptor

**Config defaults:**
- `ai-engine.realtime.default_provider` (default: `openai`)
- `ai-engine.realtime.default_model` (default: `gpt-realtime`)
- `ai-engine.realtime.default_transport` (default: `webrtc`)
- `ai-engine.realtime.default_voice` (default: `marin`)
- `ai-engine.realtime.input_audio_format` (default: `pcm16`)
- `ai-engine.realtime.output_audio_format` (default: `pcm16`)
- `ai-engine.realtime.turn_detection` (default: `['type' => 'server_vad']`)

#### WebRTC (OpenAI/Gemini)

For browser-native WebRTC, use `transport: 'webrtc'` and exchange SDP offers:

```bash
# Create session descriptor
curl -X POST http://localhost:8000/api/v1/ai/realtime/sessions \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "openai",
    "model": "gpt-realtime-mini",
    "transport": "webrtc",
    "voice": "alloy",
    "modalities": ["audio"],
    "tools": [{"type": "function", "name": "agent_chat", "description": "Send user turns to Laravel AgentChat"}]
  }'
```

Then POST the browser WebRTC offer SDP to `POST /api/v1/ai/realtime/sdp`:

```bash
curl -X POST http://localhost:8000/api/v1/ai/realtime/sdp \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "openai",
    "model": "gpt-realtime-mini",
    "transport": "webrtc",
    "voice": "alloy",
    "sdp": "v=0\r\no=..."
  }'
```

The response includes `sdp.answer` — set it as the remote description on the peer connection.

#### LiveKit (Multi-party Rooms)

For multi-party voice rooms with STT/TTS + chat pipelines, use `transport: 'livekit'`:

```php
$session = $sessionService->create(
    RealtimeSessionConfig::make('livekit', 'voice-pipeline')
        ->withTransport('livekit')
        ->withMetadata([
            'room' => 'support-room-123',
            'participant_identity' => 'user-1',
            'participant_name' => 'Alice',
            'agent_name' => 'support-bot',
        ])
        ->withFallbackPipeline([
            'stt' => ['provider' => 'openai', 'model' => 'whisper-1'],
            'chat' => ['provider' => 'openai', 'model' => 'gpt-4o-mini'],
            'tts' => ['provider' => 'elevenlabs', 'model' => 'eleven_multilingual_v2'],
        ])
);
// Returns: ['provider' => 'livekit', 'connect' => [...], 'pipeline' => [...]]
```

The descriptor includes a short-lived room token (if `LIVEKIT_API_KEY` and `LIVEKIT_API_SECRET` are set). Connect the browser to the WebRTC room, then run a LiveKit Agents Python or Node sidecar that joins the same room and uses the returned pipeline config.

**LiveKit config:**
- `ai-engine.realtime.livekit.url` — WebRTC server URL
- `ai-engine.realtime.livekit.api_key` / `api_secret` — for token generation
- `ai-engine.realtime.livekit.default_room` (default: `ai-engine-voice`)
- `ai-engine.realtime.livekit.token_ttl` (default: `3600`)
- `ai-engine.realtime.livekit.allowed_rooms` — optional allow-list (empty = all rooms allowed)
- `ai-engine.realtime.livekit.room_resolver` — optional closure/class method to override requested room (fail-closed when resolver returns a value)
- `ai-engine.realtime.livekit.identity_resolver` — optional closure to override participant identity (prevents client from minting tokens for other users)

#### Fallback Pipeline (Non-Native Providers)

Providers without native realtime voice (e.g., Anthropic, Ollama) return a fallback descriptor that chains your existing `/api/v1/ai/generate/transcribe`, `/api/v1/agent/chat`, and `/api/v1/ai/generate/tts` endpoints:

```php
$session = $sessionService->create(
    RealtimeSessionConfig::make('anthropic', 'claude-3-5-sonnet-20241022')
        ->withFallbackPipeline([
            'stt' => ['provider' => 'openai', 'model' => 'whisper-1'],
            'chat' => ['provider' => 'anthropic', 'model' => 'claude-3-5-sonnet-20241022'],
            'tts' => ['provider' => 'google_tts', 'voice' => 'en-US-Neural2-C'],
        ])
);
```

### Realtime Tool Dispatch

When the realtime provider calls a tool (e.g., `agent_chat`, `run_skill`, or a custom registered tool), POST the event to `POST /api/v1/ai/realtime/tools/dispatch`:

```bash
curl -X POST http://localhost:8000/api/v1/ai/realtime/tools/dispatch \
  -H "Content-Type: application/json" \
  -d '{
    "event": {
      "id": "call_abc123",
      "name": "agent_chat",
      "arguments": {"message": "What is our invoice policy?"}
    },
    "session_id": "support-session-123",
    "user_id": "user-456",
    "approved": false,
    "metadata": {}
  }'
```

The `RealtimeToolBrokerService` returns:

```json
{
  "success": true,
  "status": "completed",
  "tool_call_id": "call_abc123",
  "tool_name": "agent_chat",
  "message": "Our invoice policy requires...",
  "output": {
    "success": true,
    "message": "...",
    "metadata": { "needs_user_input": false },
    "response": { ... }
  }
}
```

**Possible statuses:**
- `completed` — tool ran successfully
- `needs_user_input` — tool is collecting missing data; ask the user for more input
- `approval_required` — tool requires confirmation (set `approved: true` to execute)
- `policy_blocked` — tool is blocked by org execution policy
- `tool_not_found` — tool is not registered
- `failed` — exception during tool execution

**Security:**
- By default, the broker uses the authenticated context from Laravel (user_id, session_id); client-supplied values are ignored.
- Set `ai-engine.realtime.trust_client_identity: true` to allow the tool request to override user_id/session_id (OPT-IN for backend-to-backend callers that intentionally relay identity).
- Execution policy (org deny-list) is enforced; check `ai-agent.restrictions` to deny specific tools.

**Route:**
- `POST /api/v1/ai/realtime/tools/dispatch` — handle tool calls from provider

## Observability & Tracing

Monitor AI Engine operations with built-in tracing, structured events, and pluggable observability exporters. Trace individual API calls, agent runs, and custom spans, then export metrics to OpenTelemetry, LangSmith, HTTP webhooks, or application logs.

### Record Custom Traces

Use `Engine::traces()` to manually instrument your code with timing and metadata:

```php
use LaravelAIEngine\Facades\Engine;

// Record a single operation (start + end in one call)
$span = Engine::traces()->record('fetch_customer_data', [
    'customer_id' => 123,
    'data_source' => 'database',
]);

// Or control timing manually
$spanId = Engine::traces()->start('custom_processing', [
    'batch_size' => 50,
    'operation' => 'analytics',
]);

// ... do work ...

$completed = Engine::traces()->end($spanId, 'ok', [
    'records_processed' => 47,
    'errors' => 3,
]);
```

Each span records an `id`, `name`, `metadata`, `started_at`, `ended_at`, `duration_ms`, and `status`. Metadata is merged across start and end calls, so you can add context progressively.

### Listen to Request Lifecycle Events

The package fires three key events during every API call. Listen to them to build custom observability:

```php
// In a service provider or listener class
use LaravelAIEngine\Events\AIRequestStarted;
use LaravelAIEngine\Events\AIRequestCompleted;
use LaravelAIEngine\Events\AIFailoverTriggered;

// AIRequestStarted: fired when request begins
\Illuminate\Support\Facades\Event::listen(AIRequestStarted::class, function (AIRequestStarted $event) {
    \Illuminate\Support\Facades\Log::info('AI request started', $event->getEventData());
    // Available: $event->request, $event->requestId, $event->userId, $event->engine, $event->model, $event->metadata
});

// AIRequestCompleted: fired when response is received
\Illuminate\Support\Facades\Event::listen(AIRequestCompleted::class, function (AIRequestCompleted $event) {
    $data = [
        'request_id' => $event->requestId,
        'success' => $event->wasSuccessful(),
        'engine' => $event->engine,
        'model' => $event->model,
        'tokens_used' => $event->totalTokens,
        'execution_time_ms' => $event->responseTime * 1000,
        'cost' => $event->cost,
    ];
    metrics('ai.request.completed', $event->totalTokens, tags: ['model' => $event->model]);
});

// AIFailoverTriggered: fired when primary engine fails and backup is attempted
\Illuminate\Support\Facades\Event::listen(AIFailoverTriggered::class, function (AIFailoverTriggered $event) {
    \Illuminate\Support\Facades\Log::warning('AI failover triggered', [
        'request_id' => $event->requestId,
        'from_engine' => $event->primaryEngine,
        'to_engine' => $event->fallbackEngine,
        'reason' => $event->reason,
    ]);
});
```

### Enable Observability Exporters

Register exporters in `config/ai-engine.php` to automatically push traces and evaluations to external systems. Exporters are **best-effort**—failures do not stop your application.

```php

        'exporters' => [
            LaravelAIEngine\Services\SDK\Exporters\LogObservabilityExporter::class,

            LaravelAIEngine\Services\SDK\Exporters\LangSmithObservabilityExporter::class,
        ],
        
        // HTTP exporter: POST spans to a webhook endpoint
        'http' => [
            'endpoint' => env('AI_ENGINE_OBSERVABILITY_HTTP_ENDPOINT'),
            'timeout' => (int) env('AI_ENGINE_OBSERVABILITY_HTTP_TIMEOUT', 10),
            'headers' => [
                'Authorization' => 'Bearer ' . env('OBSERVABILITY_TOKEN'),
            ],
        ],
        
        // OpenTelemetry exporter: OTLP endpoint
        'opentelemetry' => [
            'endpoint' => env('AI_ENGINE_OTEL_EXPORTER_OTLP_ENDPOINT'),
            'timeout' => (int) env('AI_ENGINE_OTEL_EXPORTER_TIMEOUT', 10),
            'headers' => [],
            'service_name' => env('AI_ENGINE_OTEL_SERVICE_NAME', 'laravel-ai-engine'),
        ],
        
        // LangSmith exporter: integrates with LangSmith debugging platform
        'langsmith' => [
            'endpoint' => env('LANGSMITH_ENDPOINT', 'https://api.smith.langchain.com/runs'),
            'api_key' => env('LANGSMITH_API_KEY'),
            'project' => env('LANGSMITH_PROJECT', env('APP_NAME', 'laravel-ai-engine')),
            'timeout' => (int) env('LANGSMITH_TIMEOUT', 10),
        ],
        
        // Log exporter: writes to ai-engine log channel

            'channel' => env('AI_ENGINE_OBSERVABILITY_LOG_CHANNEL', 'ai-engine'),
        ],
```


### Define and Run Evaluations

The `EvaluationService` lets you assert conditions on responses and export results:

```php
use LaravelAIEngine\Facades\Engine;

$response = Engine::anthropic()->text('Summarize: ' . $content);

// Simple assertion: actual === expected
Engine::evaluations()->evaluate(
    'summary_length',
    strlen($response->content),
    500  // expected
);

// Custom assertion with callable
Engine::evaluations()->assertPassed(
    'contains_key_points',
    $response->content,
    assertion: fn (string $summary) => str_contains($summary, '[1]') && str_contains($summary, '[2]')
);

// Retrieve all evaluations from this request
$allEvals = Engine::evaluations()->runs();
// Each run is: ['name' => '...', 'passed' => true|false, 'actual' => ..., 'expected' => ..., 'created_at' => '...']
```

Evaluations are exported via the same exporters configured in `observability.exporters`, so they appear in LangSmith, OpenTelemetry dashboards, and webhooks alongside traces.

### Custom Observability Exporter

Implement the `ObservabilityExporter` contract to push traces and evaluations anywhere:

```php
<?php

namespace App\Observability;

use LaravelAIEngine\Contracts\ObservabilityExporter;

class CustomExporter implements ObservabilityExporter
{
    public function export(string $type, array $payload): void
    {
        // $type is 'trace' or 'evaluation'
        
        if ($type === 'trace') {
            $this->sendToAnalytics($payload);
        } elseif ($type === 'evaluation') {
            $this->trackEvaluation($payload);
        }
    }
    
    private function sendToAnalytics(array $span): void
    {
        // Example: write to time-series DB or send to APM vendor
        redis()->lpush('ai_spans', json_encode($span));
    }
    
    private function trackEvaluation(array $evaluation): void
    {
        // Example: track pass rate in your metrics system
        metrics('ai.evaluation', $evaluation['passed'] ? 1 : 0);
    }
}
```

Register it in config:

```php
'observability' => [

        App\Observability\CustomExporter::class,
    ],
],
```

## Billing, Credits & Pricing

The laravel-ai-engine package includes a flexible credit system for tracking and billing AI usage. Credits are calculated based on model `credit_index` (cost per unit), engine rates (markup), and usage, with optional entity-scoped ledgers, custom lifecycle handlers, and async credit reservations.

### Enable Credits

Credits are enabled by default (`AI_ENGINE_CREDITS_ENABLED=true`). Disable them with:

```php
// config/ai-engine.php
'credits' => [
    'enabled' => env('AI_ENGINE_CREDITS_ENABLED', true),
    'default_balance' => env('AI_DEFAULT_CREDITS', 100.0),
    'currency' => env('AI_CREDITS_CURRENCY', 'MyCredits'),
]
```

### Credit Calculation

Credits are calculated as:
```
final_credits = (base_engine_credits + additional_input_engine_credits) × engine_rate
```

- **base_engine_credits**: From the model's `credit_index` × request unit count (words for text, images for images, minutes for video, etc.)
- **additional_input_engine_credits**: Input media costs (FAL reference images, edit inputs, etc.)
- **engine_rate**: Configured markup per engine (e.g., OpenAI at 2.0× means "100 app credits = 50 provider credits")

### Default Engine Rates

```php
'engine_rates' => [
    'openai' => 2.0,              // 2 app credits per provider credit
    'anthropic' => 3.0,           // 3 app credits per provider credit
    'gemini' => 1.2,              // 20% markup
    'fal_ai' => 1.3,              // 30% markup
    'openrouter' => 2.5,
    'nvidia_nim' => 1.5,
]
```

Override via environment:

```bash
AI_OPENAI_RATE=2.5
AI_ANTHROPIC_RATE=2.8
```

### Check & Deduct Credits

Use the `CreditManager` service to check and deduct credits before AI calls:

```php
use LaravelAIEngine\Services\CreditManager;
use LaravelAIEngine\DTOs\AIRequest;

$creditManager = app(CreditManager::class);
$userId = auth()->id();

$request = new AIRequest(
    prompt: 'Explain quantum computing',
    engine: 'openai',
    model: 'gpt-4o',
    userId: $userId,
);

// Check if user has sufficient credits
if (!$creditManager->hasCredits($userId, $request)) {
    throw new \LaravelAIEngine\Exceptions\InsufficientCreditsException(
        'Insufficient credits for this request'
    );
}

// Calculate required credits (dry-run)
$requiredCredits = $creditManager->calculateCredits($request);
echo "This request costs {$requiredCredits} credits.";

// Get detailed breakdown for UI/auditing
$breakdown = $creditManager->calculateCreditBreakdown($request);
echo "Engine credits: {$breakdown['total_engine_credits']}, "
   . "Engine rate: {$breakdown['engine_rate']}, "
   . "Final app credits: {$breakdown['final_credits']}";

// Deduct credits after successful provider call
$creditManager->deductCredits($userId, $request, $actualCreditsUsed = null);
```

### User Credit Ownership Models

By default, credits are stored on the `User` model via `my_credits` (balance) and `has_unlimited_credits` (flag). Change the owner model for multi-tenant setups:

```php
'credits' => [
    'owner_model' => env('AI_CREDITS_OWNER_MODEL', 'App\\Models\\User'),
    'owner_id_column' => env('AI_CREDITS_OWNER_ID_COLUMN', 'id'),
]
```

For tenants/organizations:

```bash
AI_CREDITS_OWNER_MODEL=App\\Models\\Tenant
AI_CREDITS_OWNER_ID_COLUMN=tenant_id
```

### Entity-Scoped Credit Ledgers

For per-engine or per-model credit budgets, add an `entity_credits` JSON column to your owner model:

```php

    $table->json('entity_credits')->nullable(); // stores {'openai': {'gpt-4o': {...}}}
});
```

The `CreditManager` will auto-detect and use this ledger:

```php

// Returns: ['balance' => 500.0, 'is_unlimited' => false, 'currency' => 'MyCredits']

// Set per-engine balance

// Set engine-specific unlimited
```

### Manage Credit Balances

Add, set, and check balances programmatically:

```php

// Grant unlimited credits
$creditManager->setUnlimitedCredits($userId, true);

    // Low balance threshold is configurable
    // config('ai-engine.credits.low_balance_threshold', 10.0)

// Get breakdown by engine

// Returns engine-specific conversion rates
```

### Request-Level Credit Accumulation

Track cumulative credits across multiple AI calls in a single request:

```php
use LaravelAIEngine\Services\CreditManager;

// Start accumulating

// Make AI calls (each internally calls accumulate())

$response2 = $this->generateText($prompt2);

// Get total and stop

```

### Credit Reservations (Async Jobs)

For long-running async jobs (FAL video generation), reserve credits upfront and finalize/refund later:

```php
// Reserve credits before async submission
$reservation = $creditManager->reserveCredits(
    userId: $userId,
    request: $videoRequest,
    creditsToReserve: null, // Auto-calculate
    metadata: ['job_id' => 'abc-123'],
    idempotencyKey: 'job-abc-123'
);

if ($reservation) {
    // Submit async job; if it fails:
    $creditManager->refundCreditReservation($reservation->uuid);
    
    // If it succeeds:
    $creditManager->finalizeCreditReservation($reservation->uuid);
}
```

Reserved credits are stored in the `ai_credit_reservations` table. Publish migrations if not present:

```bash

```

### Custom Lifecycle Handlers

Implement custom credit logic (expiration, priority deduction, subscription-based limits) via `CreditLifecycleInterface`:

```php
namespace App\Services\AI;

use Illuminate\Database\Eloquent\Model;
use LaravelAIEngine\Contracts\CreditLifecycleInterface;
use LaravelAIEngine\DTOs\AIRequest;

class ExpiringCreditsHandler implements CreditLifecycleInterface
{
    public function hasCredits(Model $owner, AIRequest $request): bool
    {
        $expiration = $owner->credits_expire_at;
        if ($expiration && $expiration->isPast()) {
            return false;
        }
        return ((float) ($owner->my_credits ?? 0)) > 0;
    }

    public function deductCredits(Model $owner, AIRequest $request, float $creditsToDeduct): bool
    {
        // Custom expiration/priority logic
        $owner->my_credits -= $creditsToDeduct;
        return $owner->save();
    }

    public function addCredits(Model $owner, float $credits, array $metadata = []): bool
    {
        $owner->my_credits += $credits;
        $owner->credits_expire_at = now()->addDays(30);
        return $owner->save();
    }

    public function getAvailableCredits(Model $owner): float
    {
        if ($owner->credits_expire_at && $owner->credits_expire_at->isPast()) {
            return 0.0;
        }
        return (float) ($owner->my_credits ?? 0);
    }

    public function hasLowCredits(Model $owner): bool
    {
        return $this->getAvailableCredits($owner) < 10.0;
    }
}
```

Register via config:

```php
'credits' => [
    'lifecycle_handler' => env('AI_LIFECYCLE_HANDLER', 'App\\Services\\AI\\ExpiringCreditsHandler'),
]
```

Or set at runtime:

```php
\LaravelAIEngine\Services\CreditManager::setLifecycleHandler(
    'App\\Services\\AI\\ExpiringCreditsHandler'
);
```

## Graph & Hybrid (Neo4j) RAG

Laravel AI Engine can store entities, chunks, and relationships in Neo4j alongside your vector store (Qdrant) for hybrid retrieval that combines semantic search with relationship expansion. Entities publish their scoped access metadata and graph relationships; retrieval then enforces per-user `CAN_ACCESS` edges so results are tenant-safe.

### Enable Hybrid Graph + Vector Retrieval

In your `.env`:

```env
AI_ENGINE_HYBRID_RAG_ENABLED=true
AI_ENGINE_HYBRID_RAG_STRATEGY=vector_then_graph
AI_ENGINE_HYBRID_RAG_VECTOR_DRIVER=qdrant
AI_ENGINE_HYBRID_RAG_GRAPH_DRIVER=neo4j

AI_ENGINE_GRAPH_ENABLED=true
AI_ENGINE_GRAPH_BACKEND=neo4j
AI_ENGINE_GRAPH_SYNC_ON_INDEX=true
AI_ENGINE_GRAPH_REQUIRE_ACCESS_SCOPE=false

AI_ENGINE_NEO4J_URL=http://localhost:7474
AI_ENGINE_NEO4J_DATABASE=neo4j
AI_ENGINE_NEO4J_USERNAME=neo4j
AI_ENGINE_NEO4J_PASSWORD=secret
```

Supported hybrid strategies:
- `vector_then_graph`: search Qdrant first, seed Neo4j graph with the vector hits, then merge related graph context.
- `graph_then_vector`: query Neo4j first, then backfill semantic matches from Qdrant.
- `reciprocal_rank`: query both stores and merge with reciprocal rank fusion (RRF).

### Prepare Your Models

Make your models graph-aware by implementing the optional model contracts (overrides add them to the published config):

```php
use LaravelAIEngine\Traits\Vectorizable;
use LaravelAIEngine\Contracts\GraphObjectInterface;
use LaravelAIEngine\Contracts\AccessScopeProviderInterface;
use LaravelAIEngine\Contracts\GraphRelationProviderInterface;

class Invoice extends Model
{
    use Vectorizable;

    /**
     * Return a sanitized object safe to expose in retrieval responses.
     * The graph stores this in the entity's object_json field.
     */
    public function toGraphObject(): array
    {
        return [
            'id' => $this->id,
            'invoice_number' => $this->invoice_number,
            'customer_name' => $this->customer?->name,
            'total' => (float) $this->total,
            'status' => $this->status,
            'issued_at' => $this->created_at?->toIso8601String(),
        ];
    }

    /**
     * Return access and ownership metadata for graph scoping.
     * Used by Neo4j to build CAN_ACCESS edges for per-user retrieval.
     */
    public function getAccessScope(): array
    {
        return [
            'canonical_user_id' => $this->user?->id,
            'user_email_normalized' => strtolower(trim($this->user?->email ?? '')),
            'scope_type' => 'workspace',
            'scope_id' => $this->workspace_id,
            'scope_label' => $this->workspace?->name,
        ];
    }

    /**
     * Return relationship descriptors for graph publishing.
     * Each relation becomes a directed edge in Neo4j.
     */
    public function getGraphRelations(): array
    {
        return [
            [
                'relation_type' => 'BELONGS_TO',
                'target_model_class' => Customer::class,
                'target_model_id' => $this->customer_id,
                'properties' => ['created_at' => now()->toIso8601String()],
            ],
            [
                'relation_type' => 'IN_WORKSPACE',
                'target_model_class' => Workspace::class,
                'target_model_id' => $this->workspace_id,
            ],
        ];
    }

    /**
     * Return compact summary for RAG list views.
     */
    public function toRAGSummary(): string
    {
        return "Invoice #{$this->invoice_number} for {$this->customer?->name}: \${$this->total} ({$this->status})";
    }

    /**
     * Return detailed content for RAG context.
     */
    public function toRAGDetail(): string
    {
        return "Invoice #{$this->invoice_number} issued to {$this->customer?->name} "
            . "on {$this->created_at?->toDateString()}. "
            . "Total: \${$this->total}. Status: {$this->status}. "
            . "Items: " . implode(', ', $this->lineItems->pluck('description')->all());
    }

    /**
     * Optional: specify the source node (app/system) that owns this entity.
     * Defaults to 'local' if not provided.
     */
    public function getSourceNode(): string
    {
        return 'local';
    }
}
```

### Initialize the Neo4j Graph

Before syncing models, initialize the graph schema:

```bash
php artisan ai:neo4j-init
```

Then bulk-sync your models:

```bash
php artisan ai:neo4j-sync --fresh
```

When `AI_ENGINE_GRAPH_SYNC_ON_INDEX=true`, every vector index operation also publishes the model into Neo4j as an `Entity` node with `HAS_CHUNK` edges to its indexed chunks.

### Multi-Tenant Access Scoping (Fail-Closed)

By default, unscoped queries (no resolved `canonical_user_id` / `user_email_normalized`) match the entire graph. For multi-tenant deployments, enable strict scoping:

```env
AI_ENGINE_GRAPH_REQUIRE_ACCESS_SCOPE=true
```

With this flag, an unscoped retrieval returns no results instead of leaking across tenants. Always pass an authenticated `userId` or an explicit `options['access_scope']`:

```php
use LaravelAIEngine\Services\RAG\HybridGraphVectorSearchService;

$results = app(HybridGraphVectorSearchService::class)->retrieveRelevantContext(
    searchQueries: ['invoices from Q4'],
    collections: [Invoice::class],
    maxResults: 5,
    threshold: 0.3,
    options: ['hybrid_strategy' => 'vector_then_graph'],
    userId: auth()->id()  // Pass authenticated user
);
```

The returned objects include:
- `vector_metadata.hybrid_score`
- `vector_metadata.hybrid_sources` (array of 'vector' and/or 'graph')
- `vector_metadata.vector_score`
- `vector_metadata.graph_score`
- `entity_ref` (model class, id, source node, scope metadata)
- `graph_object` (the sanitized `toGraphObject()` payload)
- `graph_neighbors` (related entities from 1-hop graph expansion)

### Cross-Store Links

Qdrant points now include stable graph references in their payloads:

```json
{
  "graph_node_id": "local:App\\Models\\Invoice:42",
  "graph_chunk_id": "local:App\\Models\\Invoice:42#chunk:0",
  "vector_collection": "vec_invoices",
  "vector_point_id": "42",
  "qdrant_collection": "vec_invoices",
  "qdrant_point_id": "42"
}
```

Neo4j entity and chunk nodes receive matching Qdrant references. This lets tools, agents, and citations move fluidly from a vector hit to the graph neighborhood, or from a graph node back to the vector store.

### Graph Query Planning

The Neo4j retrieval service uses a planner to classify queries (ownership, dependency, relationship, timeline, etc.) and choose traversal templates. Planned results include:

- `vector_metadata.graph_planned` (bool)
- `vector_metadata.planner_strategy` (strategy name)
- `vector_metadata.planner_score` (0.0–1.0)
- `vector_metadata.planner_score_breakdown` (when `ai-engine.graph.planner_score_breakdown = true`)
- `vector_metadata.relation_path` (array of relation types traversed)
- `vector_metadata.path_length` (hops from seed)

Tuning planner behavior:

```env
AI_ENGINE_GRAPH_MAX_TRAVERSAL_HOPS=2
AI_ENGINE_GRAPH_PLANNER_ENABLED=true
AI_ENGINE_GRAPH_PLANNER_SEED_SCORE_FLOOR=0.65
AI_ENGINE_GRAPH_RELATION_HOP_DECAY=0.9
```

### Knowledge Base Caching & Warming

The graph knowledge base is a derived acceleration layer that caches query plans and retrieval results per access scope:

```bash
php artisan ai:neo4j-kb-warm --from-profiles --canonical-user-id=1
php artisan ai:neo4j-kb-build --profiles-limit=25 --entity-limit=25
```

Schedule background warming in `routes/console.php`:

```php
Schedule::command('ai:neo4j-kb-build --profiles-limit=25 --entity-limit=25')
    ->hourly()
    ->withoutOverlapping()
    ->when(static function (): bool {
        return (bool) config('ai-engine.graph.enabled', false)
            && (string) config('ai-engine.graph.backend', 'neo4j') === 'neo4j'
            && (bool) config('ai-engine.graph.knowledge_base.enabled', true);
    });
```

## Voice, TTS & Transcription

Text-to-speech, speech-to-text, and voice conversion provide audio generation and transcription capabilities across OpenAI, ElevenLabs, Google Cloud, Gemini, local servers, and more. Use these features to enable voice interactions, narration, and audio input processing in your application.

### Text to Speech (TTS)

Generate audio from text using any configured TTS provider.

```php

$audioUrl = $response->getFiles()[0] ?? null;
```

### Database Storage (Transcription Model)

```php
    if ($t->segments) {
```

### Tips

- **Cost-aware routing**: Use `MediaProviderRouter` to select TTS/STT providers by cost, latency, or quality for production workloads.

- **Local first**: Running LocalAI with Whisper and Kokoro avoids API costs and latency; use for privacy-sensitive or high-volume workloads.

## Provider Tools & MCP

Provider tools let your app expose hosted capabilities like web search, code execution, computer use, and remote MCP servers through the Laravel AI Engine request path. The package automatically maps provider-native tool definitions to the right provider SDK format, routes executions through approval and audit layers, and records all runs, artifacts, and decisions.

### Using Provider Tools

Build tools using the provider tool classes under `LaravelAIEngine\Tools\Provider\`. Each tool implements `ProviderToolInterface` and uses a fluent builder pattern:

```php
use LaravelAIEngine\Facades\Engine;
use LaravelAIEngine\Tools\Provider\WebSearch;
use LaravelAIEngine\Tools\Provider\FileSearch;
use LaravelAIEngine\Tools\Provider\CodeInterpreter;
use LaravelAIEngine\Tools\Provider\ComputerUse;
use LaravelAIEngine\Tools\Provider\McpServer;

$response = Engine::engine('openai')
    ->model('gpt-4o')
    ->withTools([
        (new WebSearch())->max(5)->location('San Francisco', 'California', 'USA'),
        (new FileSearch(['vs_abc123', 'vs_xyz789']))->where(['status' => 'active']),
        (new CodeInterpreter())->memoryLimit('8g'),
        (new ComputerUse())->display(1920, 1080)->environment('browser'),
        (new McpServer('docs', 'https://mcp.example.com/docs'))
            ->authorizationToken('bearer-token-xyz'),
    ])
    ->generate('Search and analyze this.');
```

**Supported tools** (namespace `LaravelAIEngine\Tools\Provider\`):
- `WebSearch` — web search with optional domain allow list and location context
- `WebFetch` — fetch page content (Anthropic)
- `FileSearch` — OpenAI vector store search with filters
- `CodeInterpreter` — code execution with files and memory limits (OpenAI/Anthropic/Gemini)
- `ComputerUse` — remote desktop control (OpenAI/Anthropic)
- `McpServer` — remote MCP server with URL and auth
- `HostedShell` — execute shell commands (OpenAI)
- `ApplyPatchTool` — apply file patches with path restrictions (OpenAI)
- `ImageGenerationTool` — image generation (OpenAI)
- `ToolSearch` — search available provider skills (OpenAI)
- `GoogleMapsGrounding` — Google Maps context (Gemini)
- `ProviderSkill` — OpenAI provider skills

### Tool Configuration

Each tool exposes a fluent API. Common patterns:

```php
// WebSearch
(new WebSearch())
    ->max(5)                              // max results
    ->allow(['github.com', 'openai.com']) // restrict to domains
    ->location('London', 'England', 'UK') // optional location context

// FileSearch (OpenAI)
(new FileSearch(['vs_123']))
    ->where(['status' => 'active', 'tag' => 'pdf']) // metadata filters

// CodeInterpreter
(new CodeInterpreter())
    ->withFiles(['file_abc', 'file_xyz'])
    ->memoryLimit('4g') // or '8g', '16g'
    ->container(['type' => 'auto'])

// ComputerUse
(new ComputerUse())
    ->display(1600, 900, 1)       // width, height, display_number
    ->environment('browser')      // or 'server', 'desktop'

// McpServer
(new McpServer('label', 'https://server.example.com'))
    ->authorizationToken('secret-token')
    ->headers(['X-Custom' => 'value'])
    ->connectorId('connector-uuid') // OpenAI connector_id if using connectors

// HostedShell
(new HostedShell())
    ->container(['type' => 'auto'])

// ApplyPatchTool
(new ApplyPatchTool())
    ->workspace('/app')
    ->allowedPaths(['/app/src', '/app/tests'])
```

### Approvals & Access Control

High-risk tools (computer use, MCP servers, code execution) require approval by default. Configure which tools need approval and their risk levels in `config/ai-engine.php`:

```php
'provider_tools' => [
    'approvals' => [
        'enabled' => env('AI_ENGINE_PROVIDER_TOOL_APPROVALS_ENABLED', true),
        'expires_after_minutes' => 0, // 0 = never expire
        'require_for' => ['computer_use', 'mcp_server', 'code_interpreter'],
        'require_for_sensitive_payloads' => true,
        'sensitive_keys' => [
            'password', 'secret', 'token', 'api_key', 'authorization'
        ],
        'sensitive_patterns' => [
            '/sk-[A-Za-z0-9_\-]{16,}/',                    // API keys
            '/-----BEGIN .*PRIVATE KEY-----/',              // PEM keys
            '/\b\d{13,19}\b/',                             // Card numbers
        ],
        'risk_levels' => [
            'computer_use' => 'high',
            'mcp_server' => 'medium',
            'code_interpreter' => 'medium',
            'file_search' => 'low',
            'web_search' => 'low',
        ],
    ],
],
```

When a request hits approval middleware, the package:
1. Creates a pending approval record with a unique `approval_key`
2. Emits an event (captured by your UI or approval handler)
3. Blocks the AI request until approved/rejected

Approve a pending request:

```php
use LaravelAIEngine\Services\ProviderTools\ProviderToolApprovalService;

// In your approval handler (controller, event listener, etc.):
app(ProviderToolApprovalService::class)->approve(
    approvalKey: 'abc-123-xyz',
    actorId: auth()->id(),
    reason: 'Approved by user',
    metadata: ['ip' => request()->ip()]
);

// Reject instead:
app(ProviderToolApprovalService::class)->reject(
    approvalKey: 'abc-123-xyz',
    actorId: auth()->id(),
    reason: 'User rejected'
);
```

After approval, retry the request with approval metadata:

```php
use LaravelAIEngine\DTOs\AIRequest;

$request = AIRequest::make('Generate code', 'openai', 'gpt-4o')
    ->withMetadata([
        'provider_tool_run_id' => $runUuid,
        'provider_tool_approval_keys' => ['abc-123-xyz'],
    ]);

$response = Engine::request($request);
```

### MCP App Tool Adapter

Expose your Laravel app's registered tools and skills to MCP clients via `McpAppToolAdapter`. This adapter lists app tools with JSON schemas and routes calls back to your app:

```php
use LaravelAIEngine\Services\SDK\McpAppToolAdapter;
use LaravelAIEngine\Services\Agent\Tools\ToolRegistry;
use LaravelAIEngine\Services\Agent\AgentSkillRegistry;

$adapter = new McpAppToolAdapter(
    tools: app(ToolRegistry::class),
    skills: app(AgentSkillRegistry::class)
);

// List all tools + skills in MCP format:
$toolList = $adapter->listTools(includeSkills: true);

// Sample response:
// [
//   {
//     'name' => 'send_email',
//     'description' => 'Send an email...',
//     'inputSchema' => {
//       'type' => 'object',
//       'properties' => ['to' => ['type' => 'string'], ...],
//       'required' => ['to', 'subject', 'body']
//     },
//     'metadata' => [
//       'source' => 'tool',
//       'dispatch_name' => 'send_email',
//       'provider_name' => 'send_email',
//       'requires_confirmation' => false
//     ]
//   },
//   {
//     'name' => 'skill.summarize-docs',
//     'description' => 'Summarize documents...',
//     'inputSchema' => {
//       'type' => 'object',
//       'properties' => ['message' => ['type' => 'string']],
//       'required' => ['message']
//     },
//     'metadata' => [
//       'source' => 'skill',
//       'skill_id' => 'summarize-docs',
//       'dispatch_name' => 'skill.summarize-docs',
//       'requires_confirmation' => false
//     ]
//   }
// ]

// Call a tool by name:
$result = $adapter->callTool(
    name: 'send_email',
    arguments: ['to' => 'user@example.com', 'subject' => 'Test'],
    context: $unifiedActionContext
);
```

Tools are serialized with:
- `name` — tool identifier (e.g. 'send_email', 'skill.summarize-docs')
- `description` — tool purpose
- `inputSchema` — JSON Schema for tool arguments
- `metadata.source` — 'tool' or 'skill'
- `metadata.dispatch_name` — internal dispatch key
- `metadata.provider_name` — safe name for provider transmission (alphanumeric + underscore)
- `metadata.requires_confirmation` — whether tool needs user approval

Skills are auto-exposed with `message` as the sole input parameter.

### Provider Tool Payload Mapping

The `ProviderToolPayloadMapper` class converts normalized tool definitions to provider-specific payloads. It's transparent; the engine uses it internally:

```php
use LaravelAIEngine\Services\SDK\ProviderToolPayloadMapper;

$mapper = new ProviderToolPayloadMapper();

// Split tools into provider groups and functions
$split = $mapper->splitForProvider('openai', [
    ['type' => 'web_search', 'max' => 5],
    ['type' => 'code_interpreter'],
    ['type' => 'mcp_server', 'label' => 'docs', 'url' => 'https://...'],
    ['type' => 'function', 'name' => 'custom_fn', ...], // custom functions pass through
]);

// Result:
// [
//   'functions' => [['type' => 'function', 'name' => 'custom_fn', ...]],
//   'tools' => [
//     ['type' => 'web_search_preview', ...],
//     ['type' => 'code_interpreter', ...],
//     ['type' => 'mcp', 'server_label' => 'docs', 'server_url' => 'https://...']
//   ],
//   'mcp_servers' => [],
//   'tool_config' => {},
//   'beta_headers' => []  // Anthropic beta headers when needed
// ]
```

Provider-specific behavior:
- **OpenAI**: Maps to `web_search_preview`, `file_search`, `code_interpreter`, `computer_use_preview`, `mcp`, `image_generation`, `tool_search`, `hosted_shell`, `apply_patch`, `skill`.
- **Anthropic**: Maps to `web_search_*`, `web_fetch`, `code_execution_*`, `computer_*`, `mcp_server` (with beta headers). Supports configurable versions.
- **Gemini**: Maps to `googleSearch`, `urlContext`, `retrieval`, `googleMaps`, `codeExecution`.
- **OpenRouter**: Subset of OpenAI mappings (no computer use, limited image generation).

### Lifecycle: Runs, Approvals & Artifacts

Provider tool work is recorded across four tables:

- **`ai_provider_tool_runs`** — Tool execution records (request/response payloads, status: running/awaiting_approval/completed/failed, conversation_id for OpenAI responses state reuse)
- **`ai_provider_tool_approvals`** — Approval records with `approval_key`, `status` (pending/approved/rejected), risk_level, expiry
- **`ai_provider_tool_artifacts`** — Generated files (links to `ai_media` for downloaded files, provider_file_id for remote artifacts, citation URLs)
- **`ai_provider_tool_audit_events`** — Approval/run lifecycle audit trail (action, actor_id, metadata)

Access them via repositories:

```php
use LaravelAIEngine\Repositories\ProviderToolRunRepository;
use LaravelAIEngine\Repositories\ProviderToolApprovalRepository;
use LaravelAIEngine\Repositories\ProviderToolArtifactRepository;
use LaravelAIEngine\Repositories\ProviderToolAuditRepository;

// Get run with artifacts and approvals:
$run = app(ProviderToolRunRepository::class)->find($uuid);
$run->artifacts;   // HasMany AIProviderToolArtifact
$run->approvals;   // HasMany AIProviderToolApproval
$run->auditEvents; // HasMany AIProviderToolAuditEvent

// List pending approvals:
$pending = app(ProviderToolApprovalRepository::class)
    ->query()
    ->where('status', 'pending')
    ->get();

// Get audit events for a run:
$events = $run->auditEvents()->orderByDesc('created_at')->get();
```

### Artifacts & Remote File Handling

When a provider tool returns downloadable file URLs (from code execution, computer use, etc.), the package extracts them and optionally downloads/stores them:

```php
'provider_tools' => [
    'artifacts' => [
        'enabled' => true,
        'persist_remote_files' => env('AI_ENGINE_PROVIDER_TOOL_PERSIST_REMOTE_ARTIFACTS', true),
        'block_private_urls' => true,  // Prevent downloads from localhost/10.x/172.16-31.x/192.168.x
        'max_remote_bytes' => 50 * 1024 * 1024,
        'allowed_extensions' => ['png', 'jpg', 'mp4', 'wav', 'pdf', 'csv', 'json'],
        'allowed_mime_types' => ['image/*', 'video/*', 'audio/*', 'application/pdf', 'text/*'],
    ],
],
```

Keep `block_private_urls` enabled for user-facing apps to prevent SSRF. Citation URLs are always audited (recorded without downloading).

With `"continue": true`, a continuation job is queued automatically. Or call `/runs/{run}/continue` manually with `"async": false` for immediate execution or `"async": true` to queue.

```json
  "model": "fal-ai/flux/dev",
  "prompt": "A clean product render",
  "input": {
    "image_size": "square_hd"
  },
  "async": false
}
```

Queued jobs persist the run and post results back via webhook to `/fal/catalog/webhook`.

## Multi-tenancy & Scope

The laravel-ai-engine automatically isolates AI operations (RAG, agent memory, data queries) by tenant and workspace, ensuring strict data segregation in multi-tenant architectures. Scopes are resolved from your authenticated user, session, or custom resolvers—then automatically applied to every vector search, structured data query, and memory lookup.

### Scope Basics: Tenant & Workspace

Every AI operation can carry `tenant_id` and/or `workspace_id`. The engine resolves these via the `AIScopeResolver` contract:

```php
// src/Contracts/AIScopeResolver.php
public interface AIScopeResolver {
    /**
     * Resolve trusted execution scope for AI/RAG/agent requests.
     * @return array{tenant_id?:mixed, workspace_id?:mixed}
     */
    public function resolve(mixed $userId = null, array $options = []): array;
}
```

By default, `DefaultAIScopeResolver` reads scope from your authenticated user. Configure which user fields to check in `config/ai-engine.php`:

```php
'scope' => [
    'auto_inject' => env('AI_ENGINE_SCOPE_AUTO_INJECT', true),  // Auto-resolve & merge on every request
    'resolver' => env('AI_ENGINE_SCOPE_RESOLVER', null),        // Custom resolver (class or callable)
    'tenant_user_fields' => [                                    // Check these fields in order
        'tenant_id',
        'organization_id',
        'company_id',
        'team_id',
    ],
    'workspace_user_fields' => [
        'workspace_id',
        'current_workspace_id',
    ],
],
```

### Auto-Injection & Merging

When `ai-engine.scope.auto_inject` is `true` (default), the `AIScopeOptionsService` merges resolved scope into every AI call:

```php
$scopeService = app(\LaravelAIEngine\Services\Scope\AIScopeOptionsService::class);

// Resolved from user + merged with explicit options
// Explicit options win (tenant_id here overrides resolved value)
$merged = $scopeService->merge(auth()->id(), [
    'tenant_id' => 'custom-tenant',
]);
// Result: ['tenant_id' => 'custom-tenant', 'workspace_id' => 'user-workspace-id', ...]
```

For **custom tenant resolution**, implement `AIScopeResolver` and register it:

```php
// app/Services/CustomScopeResolver.php
namespace App\Services;

use LaravelAIEngine\Contracts\AIScopeResolver;

class CustomScopeResolver implements AIScopeResolver {
    public function resolve(mixed $userId = null, array $options = []): array {
        $tenant = session('current_tenant');
        return [
            'tenant_id' => $tenant?->id,
            'workspace_id' => $tenant?->workspace_id,
        ];
    }
}
```

Then register in `config/ai-engine.php`:

```php
'scope' => [
    'resolver' => \App\Services\CustomScopeResolver::class,
],
```

Or use a callable:

```php
'scope' => [
    'resolver' => fn($userId, $options) => [
        'tenant_id' => session('tenant_id'),
        'workspace_id' => $options['workspace_id'] ?? session('workspace_id'),
    ],
],
```

### Data Query Scoping

The built-in `data_query` tool filters structured data by scope columns. Configure which columns represent tenant/workspace/user ownership:

```php
// config/ai-engine.php
'data_query' => [
    'scope_columns' => ['user_id', 'workspace_id', 'tenant_id'],  // Checked in order
    'require_scope' => env('AI_ENGINE_DATA_QUERY_REQUIRE_SCOPE', true),  // Fail-closed by default
    'models' => [
        'invoices' => [
            'class' => \App\Models\Invoice::class,
            'aliases' => ['invoice', 'invoices'],
            'list' => ['id', 'invoice_number', 'total', 'status'],
            'status_column' => 'status',
            'statuses' => ['paid', 'draft', 'overdue'],
            'filterable' => ['customer_id', 'status'],
            // Mark intentionally-unscoped models (public catalog)
            'public' => false,
        ],
    ],
],
```

When an agent runs `data_query` to "count invoices for Acme Corp", the engine:

1. Resolves `tenant_id` from context / request options
2. Checks if the `invoices` table has a `tenant_id` column
3. Applies `WHERE tenant_id = ?` to all queries
4. Returns a failure if no scope could be applied and `require_scope` is true

```php
// Example: agent context with scope
$context = new \LaravelAIEngine\DTOs\UnifiedActionContext(
    sessionId: $session->id,
    userId: auth()->id(),
    conversationHistory: [],
    metadata: ['workspace_id' => $workspace->id, 'tenant_id' => $org->id],
);

// data_query tool automatically applies these scopes to any database query
```

To allow a model to be queried without scope (e.g., public product catalog), set `'public' => true`:

```php
'data_query' => [
    'models' => [
        'products' => [
            'class' => \App\Models\Product::class,
            'public' => true,  // Bypass require_scope check
        ],
    ],
],
```

### Vector Search & RAG Scoping

Models using the `Vectorizable` trait automatically capture scope in metadata:

```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use LaravelAIEngine\Traits\Vectorizable;

class Document extends Model {
    use Vectorizable;

    protected $fillable = ['title', 'content', 'tenant_id', 'workspace_id', 'user_id'];
}
```

When indexing, metadata is auto-populated from model fields:

```php
// When Document is indexed, this metadata is stored with the vector
public function getVectorMetadata(): array {
    // Trait auto-extracts from model attributes
    // Checks: tenant_id, organization_id, company_id, team_id (in order)
    // Checks: workspace_id, current_workspace_id (in order)
    // Returns: ['tenant_id' => 1, 'workspace_id' => 'workspace-123']
}
```

Vector searches are automatically scoped by the `VectorSearchService` using metadata filters. During RAG retrieval, only vectors matching the current tenant/workspace are returned.

To ensure vectors are indexed with scope metadata, your models must have scope columns:

```php
// Migration
Schema::create('documents', function (Blueprint $table) {
    $table->id();
    $table->string('title');
    $table->text('content');
    $table->unsignedBigInteger('tenant_id')->index();  // RAG will filter on this
    $table->uuid('workspace_id')->index();
    $table->unsignedBigInteger('user_id')->nullable();
    $table->timestamps();
});

// Model knows how to extract scope
class Document extends Model {
    use Vectorizable;
    // trait auto-reads tenant_id, workspace_id from attributes
}
```

### Multi-Database Tenancy

For complete data isolation where each tenant has their own database, enable multi-DB tenancy in `config/vector-access-control.php`:

```php
// config/vector-access-control.php
return [
    'multi_db_tenancy' => env('AI_ENGINE_MULTI_DB_TENANCY', false),
    'multi_db_collection_strategy' => env('AI_ENGINE_MULTI_DB_COLLECTION_STRATEGY', 'prefix'),
    // 'prefix' => tenant_slug_vec_documents
    // 'suffix' => vec_documents_tenant_slug
    // 'separate' => tenant_slug/vec_documents
    
    'tenant_resolver' => env('AI_ENGINE_TENANT_RESOLVER', 'session'),
    // 'config' => database connection name from config
    // 'session' => reads session('tenant_id')
    // 'database' => current database connection name
    // 'custom' => uses custom_tenant_resolver class
    
    'tenant_session_key' => 'tenant_id',
];
```

When multi-DB is enabled, the `MultiTenantVectorService` isolates vector collections per tenant:

```php
$tenantService = app(\LaravelAIEngine\Services\Tenant\MultiTenantVectorService::class);

// Each tenant gets separate vector collections
$collectionName = $tenantService->getTenantCollectionName('vec_documents');
// With 'prefix' strategy: 'acme_corp_vec_documents'

// Metadata is automatically tagged with tenant info
$metadata = $tenantService->buildTenantMetadata();
// Returns: ['tenant_id' => 'acme', 'tenant_slug' => 'acme', 'scope_key' => 'sha1hash']
```

For custom tenant resolution (e.g., Spatie Multitenancy, Stancl Tenancy), implement `TenantResolverInterface`:

```php
namespace App\Services;

use LaravelAIEngine\Contracts\TenantResolverInterface;
use Spatie\Multitenancy\Models\Tenant;

class SpatieMultitenancyResolver implements TenantResolverInterface {
    public function getCurrentTenantId(): ?string {
        $tenant = Tenant::current();
        return $tenant?->id;
    }

    public function getCurrentTenantSlug(): ?string {
        return Tenant::current()?->slug;
    }

    public function getTenantConnection(): ?string {
        return Tenant::current()?->database;
    }

    public function hasTenant(): bool {
        return Tenant::current() !== null;
    }
}
```

Then register in `config/vector-access-control.php`:

```php
'tenant_resolver' => 'custom',
'custom_tenant_resolver' => \App\Services\SpatieMultitenancyResolver::class,
```

### Agent Memory Scoping

Conversation memory is scoped by the same tenant/workspace rules. Each workspace gets isolated memory retrieval:

```php
// src/DTOs/ConversationMemoryQuery.php
$query = new \LaravelAIEngine\DTOs\ConversationMemoryQuery(
    message: 'what language should I use?',
    scopeType: 'workspace',      // or 'tenant'
    scopeId: 'workspace-123',     // current workspace
    sessionId: $session->id,
);

// Memory is keyed by scope_hash(tenant_id, workspace_id)
// So workspace-a and workspace-b never see each other's memories
```

When compacting conversation context (extracting long-term facts), the extracted memories are written under the **current request's scope**, not any cached context:

```php
$compactor = app(\LaravelAIEngine\Services\Agent\ConversationContextCompactor::class);

$compactor->compact(
    $context,  // cached context (may have stale workspace_id)
    ['workspace_id' => $request->user()->current_workspace_id]  // CURRENT workspace
    // Extracted memories go under current workspace, not stale one
);
```

### Fail-Closed Access Control

By default, operations are **fail-closed**: if no scope can be applied, the request is refused:

```php
// Data query without scope returns an error
$result = $tool->execute([], new UnifiedActionContext(
    sessionId: $session->id,
    userId: null,  // No authenticated user
    metadata: [],  // No tenant/workspace provided
));
// ActionResult::failure("Access to 'invoices' is blocked: no scope could be applied")
```

Control this behavior with config flags:

```php
'data_query' => [
    'require_scope' => true,     // Fail if no scope applied (default)
],

'vector' => [
    'require_structured_scope' => true,  // Fail if RAG finds no scope
],
```

To bypass scoping for intentionally-public data, mark models explicitly:

```php
'data_query' => [
    'models' => [
        'public_docs' => [
            'class' => \App\Models\PublicDocument::class,
            'public' => true,  // Skip require_scope check
        ],
    ],
],
```

### Scope Aliases

The engine normalizes scope field names for convenience:

```php
// These are equivalent:
$options1 = ['tenant_id' => 'org-1'];
$options2 = ['tenant' => 'org-1'];  // Alias: normalized to tenant_id

$options3 = ['workspace_id' => 'ws-2'];
$options4 = ['workspace' => 'ws-2'];  // Alias: normalized to workspace_id
```

This is handled by `AIScopeOptionsService::normalizeAliases()`, so both forms work everywhere.

### Configuration Summary

| Key | Default | Purpose |
|-----|---------|---------|
| `ai-engine.scope.auto_inject` | `true` | Auto-resolve & merge scope on every call |
| `ai-engine.scope.resolver` | `null` | Custom `AIScopeResolver` class or callable |
| `ai-engine.scope.tenant_user_fields` | `['tenant_id', 'organization_id', 'company_id', 'team_id']` | User fields checked for tenant scope |
| `ai-engine.scope.workspace_user_fields` | `['workspace_id', 'current_workspace_id']` | User fields checked for workspace scope |

| `vector-access-control.multi_db_tenancy` | `false` | Isolate vectors per tenant in separate collections |
| `vector-access-control.tenant_resolver` | `'session'` | How to resolve current tenant (`'config'`, `'session'`, `'database'`, or `'custom'`) |

## Learning & Design Generation

The Learn layer stores and retrieves reusable design examples, workflows, rules, and guidance scoped to tenants, workspaces, or sessions. It works as a generic knowledge repository: learned sources are extracted into sections, optionally indexed for semantic retrieval, and later searched by agents or design generators to influence output.

### Learning & Storing Knowledge

Import knowledge from text, files, URLs, or curated getdesign design packs using the `Engine::learn()` fluent API:

```php
use LaravelAIEngine\Facades\Engine;

// Learn from inline text
$result = Engine::learn()
    ->fromText(
        'Invoices require customer email, due date, tax rules, and line item totals. '
        . 'Always show a subtotal, tax amount, and grand total. Use a clean two-column layout.'
    )
    ->type('workflow')
    ->scope(tenantId: $tenantId, workspaceId: $workspaceId)
    ->save();

// Learn from a file (must be stored on a configured disk or local path)
$result = Engine::learn()
    ->fromFile('./docs/design-system.md')
    ->type('design')
    ->scope(workspaceId: $workspaceId)
    ->index()  // Add to vector store for semantic search
    ->save();

// Learn from a URL (getdesign API, or generic crawl if enabled)
$result = Engine::learn()
    ->fromUrl('https://example.com/design-guide')
    ->using('getdesign')
    ->type('design')
    ->scope(workspaceId: $workspaceId)
    ->index()
    ->save();

// Learn from a curated getdesign slug (runs CLI: npx getdesign@latest add {slug})
$result = Engine::learn()
    ->fromDesignSlug('bmw-m')
    ->scope(tenantId: $tenantId, workspaceId: $workspaceId)
    ->index(storeName: 'Premium Design Examples')
    ->save();
```

### Searching Learned Context

Retrieve learned sources by query and scope:

```php
// Search scoped to a workspace
$matches = Engine::learn()->search(
    'premium automotive layout with red CTAs and performance cards',
    scope: ['workspace_id' => $workspaceId],
    limit: 3,
    type: 'design'  // Optional: filter by knowledge type
);

// Each match includes score, source metadata, and extracted item content

    echo $match->score;  // Semantic similarity (0-1)
```

### Storage & Scope

Learned knowledge is stored in two database tables:

- `ai_learn_sources` — Raw source, metadata, vector store link, scope columns
- `ai_learned_items` — Extracted sections with kind, title, content, confidence

Scope columns are flexible strings (`user_id`, `tenant_id`, `workspace_id`, `session_id`), so UUID, ULID, integer, and external identifiers all work.

**Scope isolation:** A learned source saved to `workspace_id='acme'` will not be returned when searching `workspace_id='other'` unless explicitly saved without that constraint. Use this to keep shared design systems at the tenant or global scope and project-specific examples at workspace scope.

### Item Classification

Extracted items are automatically classified by kind:

- `component` — Mentions "component"
- `example` — Mentions "example" or has "before/after"
- `token` — Mentions color, typography, or spacing
- `rule` — Mentions "rule," "guideline," "must," or "should"
- `section` — Default catch-all

### Design Generation Pipeline

When you call `generateDesign()`:

1. Retrieve scoped learned sources matching the prompt (vector similarity or SQL).
2. Build an AI prompt that includes matched sources, instructions to preserve design tokens and layout geometry, and instructions to sanitize proprietary brand names.
3. Call the configured AI engine (OpenAI, Anthropic, Gemini, etc.) to generate HTML or Markdown.
4. Normalize output (add doctype, wrap HTML, strip markdown fences).
5. Sanitize copied source brand/font terms to generic fallbacks.
6. Optionally compose with HTML layout wrapping and media bands.
7. Return content + match metadata for audit and debugging.

The generator is generic—it takes learned context and a user prompt, and produces a design. It does not know about invoices, BMWs, dashboards, or domain logic. Those details come from the prompt and learned context.

## Model Council

Run a single prompt across multiple AI models simultaneously and compare their responses side-by-side. Each model member executes independently—if one fails (provider error, bad model, network issue), the rest continue unaffected. Members run sequentially; the council's value is comprehensive comparison, not throughput.

### Basic Usage

Post to `POST /api/v1/agent/council` with a prompt, a list of council members (each specifying a model and optional engine), and optional parameters:

```php
// PHP client example (e.g., in a Livewire component or controller)
$response = Http::post('http://localhost/api/v1/agent/council', [
    'prompt' => 'Compare the pros and cons of React vs Vue for a startup.',
    'members' => [
        ['engine' => 'openai', 'model' => 'gpt-4o'],
        ['engine' => 'anthropic', 'model' => 'claude-3-5-sonnet-20241022'],
        ['engine' => 'gemini', 'model' => 'gemini-2.5-flash'],
    ],
    'options' => [
        'temperature' => 0.7,
        'max_tokens' => 500,
    ],
]);

// Response structure:
// {
//   "success": true,
//   "data": {
//     "prompt": "Compare the pros and cons...",
//     "members": [
//       {
//         "engine": "openai",
//         "model": "gpt-4o",
//         "success": true,
//         "content": "React is...",
//         "error": null,
//         "usage": { "prompt_tokens": 40, "completion_tokens": 210 },
//         "credits_used": 1.2,
//         "metadata": {}
//       },
//       {
//         "engine": "anthropic",
//         "model": "claude-3-5-sonnet-20241022",
//         "success": true,
//         "content": "React excels at...",
//         "error": null,
//         "usage": { "prompt_tokens": 40, "completion_tokens": 215 },
//         "credits_used": 0.8,
//         "metadata": {}
//       },
//       // ... additional members
//     ],
//     "count": 3,
//     "succeeded": 3
//   }

```

### Members Format

Each member is an object with:

- **`model`** (required, string, max 255 chars) — The model identifier (e.g., `gpt-4o`, `claude-3-5-sonnet-20241022`). Empty or missing models are silently skipped.
- **`engine`** (optional, string, max 255 chars) — The provider/engine (`openai`, `anthropic`, `gemini`, `bedrock`, etc.). Defaults to your app's default engine if omitted.
- **`system_prompt`** (optional, string, max 4000 chars) — A custom system prompt for this member only. Falls back to the `options.system_prompt` if provided there.

```php
'members' => [
    // Explicit engine + model
    ['engine' => 'openai', 'model' => 'gpt-4o'],
    
    // Use default engine, custom system prompt
    ['model' => 'gpt-4o-mini', 'system_prompt' => 'You are a code reviewer.'],
    
    // Another provider
    ['engine' => 'anthropic', 'model' => 'claude-3-5-sonnet-20241022'],
]
```

### Options

Global defaults applied to all members (unless overridden per-member):

```php
'options' => [
    'system_prompt' => 'You are a helpful expert.',  // Fallback for members without one
    'temperature' => 0.8,                             // Sampling randomness (0–2)
    'max_tokens' => 1024,                             // Output limit per model
]
```

### Request Validation

The `ModelCouncilRequest` enforces:

- **`prompt`** — Required, max 8000 characters.
- **`members`** — Required array, min 1 member, max 8 members per request.
  - Each member's `model` is required and max 255 characters.
  - Each member's `engine` is optional, max 255 characters.
  - Each member's `system_prompt` is optional, max 4000 characters.
- **`options`** — Optional object.
  - `system_prompt` — max 4000 characters.
  - `temperature` — numeric, 0–2.
  - `max_tokens` — integer, 1–32000.
- **`user_id`** — Optional identifier (max 255 chars). If not provided, defaults to the authenticated user's ID.

### Behind the Scenes

The endpoint delegates to `LaravelAIEngine\Services\ModelCouncilService::run()`, which:

1. Iterates through each member in sequence.
2. Constructs an `AIRequest` DTO with the prompt, engine, model, system prompt, and options.
3. Calls `AIEngineService::generateText()` to fetch that model's response.
4. Captures success, content, error, usage metrics, and credits consumed.
5. **On member failure** (provider error, network timeout, exception), logs a warning but continues with the next member.
6. Returns an array of result objects, one per member, preserving order.

### Response Fields per Member

Each result object contains:

- **`engine`** (string) — The engine used (e.g., `openai`).
- **`model`** (string) — The model used.
- **`success`** (bool) — Whether the request succeeded.
- **`content`** (string) — The model's text response (empty on failure).
- **`error`** (string|null) — Error message if `success === false`.
- **`usage`** (object|null) — Token counts: `{ prompt_tokens, completion_tokens }` (null on failure).
- **`credits_used`** (float) — Credits consumed (0 on failure).
- **`metadata`** (object) — Provider-specific metadata (finish_reason, etc.).

### Resilience & Isolation

If a single member fails—bad API key, unsupported model, provider outage, timeout—its result will have `success: false` and an error message. Other members' requests are not affected. The top-level response remains `success: true` as long as the council endpoint itself didn't crash (e.g., validation error).

```php
// Example partial failure:
{
  "success": true,
  "data": {
    "count": 3,
    "succeeded": 2,  // Only 2 of 3 members succeeded
    "members": [
      { "success": true, "content": "React excels...", ... },
      { "success": false, "error": "401 Unauthorized", "content": "" },
      { "success": true, "content": "Vue is lightweight...", ... }
    ]
  }
}
```

### Logging

Failed council members are logged to the `ai-engine` channel at the `warning` level with context:

```php
Log::channel('ai-engine')->warning('Model council member failed', [
    'engine' => 'anthropic',
    'model' => 'claude-3-5-sonnet-20241022',
    'exception' => 'API rate limit exceeded',
    'exception_class' => 'RuntimeException',
]);
```

### Usage in PHP/Laravel

```php
use Illuminate\Support\Facades\Http;

$council = Http::post('/api/v1/agent/council', [
    'prompt' => 'Write a haiku about Laravel.',
    'members' => [
        ['engine' => 'openai', 'model' => 'gpt-4o'],
        ['engine' => 'anthropic', 'model' => 'claude-3-5-sonnet-20241022'],
        ['engine' => 'gemini', 'model' => 'gemini-2.5-flash'],
    ],
    'options' => [
        'max_tokens' => 100,
        'temperature' => 0.9,
    ],
]);

if ($council['success']) {
    foreach ($council['data']['members'] as $member) {
        echo "## {$member['model']} ({$member['engine']})\n";
        if ($member['success']) {
            echo $member['content'] . "\n";
            echo "Tokens: {$member['usage']['completion_tokens']}\n";
        } else {
            echo "ERROR: {$member['error']}\n";
        }
    }
    echo "\nSuccessful responses: {$council['data']['succeeded']} / {$council['data']['count']}\n";
}
```

### Middleware & Authentication

The council route is registered under the `api/v1/agent` prefix and uses the `agent` middleware group (configurable in `ai-engine.php`):

```php
// In routes/api.php:
Route::post('/council', [ModelCouncilApiController::class, 'run'])
    ->name('ai-engine.agent.api.council.run');
```

Middleware stack defaults to `['api', SetRequestLocaleMiddleware::class, StandardizeApiResponseMiddleware::class]` but can be customized via `config/ai-engine.php`:

```php
'api' => [
    'middleware' => [
        'append' => [],  // Add middleware to all routes
        'groups' => [
            'agent' => [],  // Specific to agent group
        ],
        'replace' => [
            'agent' => ['api', 'auth:sanctum'],  // Override completely
        ],
    ],
],
```

### Cost Estimation

Because members run sequentially and each model is charged independently, the total cost is the sum of each successful member's credits:

```php
$totalCredits = array_sum(array_map(
    fn ($r) => $r['credits_used'] ?? 0,
    $response['data']['members']
));
```

Use the council sparingly for real-time UX (since all members run sequentially), and reserve it for comparison workflows where latency is acceptable (e.g., batch analysis, offline reports).

## Structured Collection Flows

Structured collection enables guided, multi-turn chat to gather a target JSON object field-by-field. The system asks for missing required fields in the user's language, confirms before completion, and dispatches a callback with the final payload—all while keeping JSON keys and enum values canonical.

### Define a Collection

Use `StructuredCollectionDefinition::make()` with a fluent builder API:

```php
use LaravelAIEngine\DTOs\StructuredCollectionDefinition;

$collection = StructuredCollectionDefinition::make('lead_capture')
    ->description('Collect lead details from chat.')
    ->addText('name', required: true, description: 'Lead name')
    ->addEmail('email', required: true)
    ->addSelect('level', [
        ['value' => 'beginner', 'labels' => ['en' => 'Beginner', 'ar' => 'مبتدئ']],
        ['value' => 'advanced', 'labels' => ['en' => 'Advanced', 'ar' => 'متقدم']],
    ], required: true)
    ->addTextarea('notes')
    ->confirmBeforeComplete()
    ->closeOnComplete()
    ->callbackUrl('https://app.test/api/ai/lead-callback');
```

The builder includes convenience methods for common field types:

- **Text fields**: `addText()`, `addTextarea()`, `addEmail()`, `addUrl()`, `addPhone()`, `addFile()`, `addHidden()`
- **Numbers**: `addNumber()`, `addInteger()`
- **Dates/times**: `addDate()`, `addDateTime()`, `addTime()`
- **Booleans**: `addBoolean()`
- **Choices**: `addSelect()`, `addMultiSelect()`, `addRadio()`, `addCheckboxes()`
- **Structured**: `addJson()`, `addArray()`, `addObject()`
- **Generic**: `addField(name, type, required, description, format, enum, metadata)`

All accept optional `description` and `metadata` parameters for UI hints and localization.

### Field Types & Options

Select-like fields (`addSelect`, `addMultiSelect`, `addRadio`, `addCheckboxes`) accept an options array. Each option can be a string or an object with `value` and optional `label` (English) or `labels` (multi-language map):

```php
->addSelect('priority', [
    'low',
    'medium',
    'high',
], required: true)

->addSelect('tier', [
    ['value' => 'free', 'label' => 'Free Plan'],
    ['value' => 'pro', 'labels' => ['en' => 'Pro Plan', 'ar' => 'خطة احترافية']],
], required: true)
```

The AI stores canonical values (`free`, `pro`) in the final payload; the frontend presents localized labels to the user.

### Multi-Turn Collection Flow

Pass the collection to `ChatService::processMessage()` via `extraOptions`:

```php
$response = app(\LaravelAIEngine\Services\ChatService::class)->processMessage(
    message: 'اسمي أحمد ومستواي مبتدئ',
    sessionId: 'lead-session-123',
    userId: 'user-uuid',
    extraOptions: [
        'collection' => $collection->toArray(),
    ],
);
```

The system:
1. **Extracts** fields from the user's message (in any language)
2. **Asks** for missing required fields (replies in the user's detected language)
3. **Confirms** the collected data (if `confirmBeforeComplete()` is true, which is the default)
4. **Dispatches** a callback (event or HTTP POST) with the final payload
5. **Closes** the session (if `closeOnComplete()` is true, which is the default)

Each response includes the collection status and missing fields under `metadata['collection']`:

```json
{
  "content": "Thank you! What is your email?",
  "metadata": {
    "collection": {
      "name": "lead_capture",
      "status": "collecting",
      "data": {"name": "Ahmed"},
      "missing_fields": ["email", "level"],
      "language": "ar",
      "fields": [
        {
          "name": "level",
          "type": "string",
          "ui": "select",
          "required": true,
          "options": [
            {"value": "beginner", "label": "مبتدئ"},
            {"value": "advanced", "label": "متقدم"}
          ]
        }
      ]
    }
  }
}
```

### Collection Lifecycle

The collection session is stored in cache (keyed by `agent_structured_collection:{sessionId}:{userId}`) for the duration configured in `ai-agent.structured_collection.ttl_seconds` (default: 3600 seconds / 1 hour). Each user message updates the state; when all required fields are present and the user confirms, the collection completes.

Check if a collection is active:

```php
$service = app(\LaravelAIEngine\Services\Agent\StructuredCollectionSessionService::class);
if ($service->isActive($sessionId, $userId)) {
    // Collection is in progress
}
```

### Completion Callback

When the user confirms, the callback receives a JSON payload:

```php
->callbackUrl('https://app.test/api/ai/lead-callback', method: 'POST', headers: [
    'Authorization' => 'Bearer '.config('services.ai_callback.token'),
])
```

The callback body:

```json
{
  "session_id": "lead-session-123",
  "user_id": "user-uuid",
  "status": "completed",
  "collection": "lead_capture",
  "data": {
    "name": "Ahmed",
    "email": "ahmed@example.com",
    "level": "beginner"
  },
  "metadata": {
    "language": "ar",
    "confirmed": true,
    "completed_at": "2026-05-18T01:30:00+03:00",
    "definition": { /* full StructuredCollectionDefinition::toArray() */ }
  }
}
```

Or use event-based dispatch instead of HTTP:

```php
->callbackEvent()
```

This fires `LaravelAIEngine\Events\AgentStructuredCollectionCompleted` (which receives the same payload). Listeners in your app can then process the collected data.

### Preview Rendering

Enable optional form previews in the chat response:

```php
->withPreview('html')
```

Preview modes:
- `'html'` (default): Quick chat-friendly form preview plus a component placeholder
- `'component'`: Component metadata only (frontend renders with custom component)
- `'schema'`: Pure schema payload (no HTML, frontend decides rendering)

Configure defaults in `config/ai-agent.php`:

```php
'structured_collection' => [
    'preview' => [
        'enabled' => false,  // Enable by default
        'mode' => 'component',
        'assets' => [
            'css' => ['/vendor/ai-engine/structured-collection.css'],
            'js' => ['/vendor/ai-engine/structured-collection.js'],
        ],
    ],
],
```

### Configuration

Key settings in `config/ai-agent.php` under `'structured_collection'`:

| Key | Default | Purpose |
|-----|---------|---------|
| `ttl_seconds` | `3600` | Session cache lifetime |
| `max_tokens` | `900` | AI response token limit |
| `temperature` | `0.1` | AI temperature (low = deterministic) |
| `callback_timeout` | `10` | HTTP callback timeout in seconds |
| `preview.enabled` | `false` | Enable preview rendering by default |
| `preview.mode` | `component` | Default preview mode |
| `preview.assets` | `[...]` | CSS/JS asset paths for previews |

### REST API Usage

Post to the chat endpoint with collection embedded:

```json
POST /api/ai/chat
{
  "message": "I'm Ahmed, beginner level",
  "session_id": "lead-session-123",
  "memory": false,
  "actions": false,
  "use_rag": false,
  "collection": {
    "name": "lead_capture",
    "description": "Collect lead details from chat.",
    "schema": {
      "type": "object",
      "required": ["name", "email", "level"],
      "properties": {
        "name": {"type": "string", "metadata": {"ui": "text"}},
        "email": {"type": "string", "format": "email", "metadata": {"ui": "email"}},
        "level": {
          "type": "string",
          "enum": ["beginner", "advanced"],
          "metadata": {
            "ui": "select",
            "options": [
              {"value": "beginner", "labels": {"en": "Beginner", "ar": "مبتدئ"}},
              {"value": "advanced", "labels": {"en": "Advanced", "ar": "متقدم"}}
            ]
          }
        }
      }
    },
    "confirm_before_complete": true,
    "close_on_complete": true,
    "callback": {"type": "event"}
  }
}
```

The response includes `metadata.collection` with the current state, missing fields, and field presentation info (names, types, UI hints, options, descriptions).

### Cancellation & Skipping

If a user says "cancel" or similar intent, the session fires `user_cancelled: true`, clears the cache, and returns status `'cancelled'`. To disable confirmation or auto-close behavior:

```php
->confirmBeforeComplete(false)
->closeOnComplete(false)
```

This keeps the session open after completion so you can extend or re-run without restarting.

### Multi-Language & Localization

The AI detects the user's language and replies in that language. It **never translates JSON keys or enum values**—those remain canonical. The `fields` array in the response includes locale-aware labels for select options via the `StructuredCollectionFieldPresenter`, which respects the `labels` map and falls back to humanized `value` if no label is found.

## Admin, Catalog, Diagnostics & SDK

The laravel-ai-engine package provides a unified operational surface for managing AI engines, models, and agent infrastructure. This section covers four complementary domains: the admin UI for cross-tenant governance, the engine catalog for introspecting available models, the diagnostics commands for health-checking the orchestrator, and the SDK services for advanced integrations.

### Admin UI

The admin UI is a fully-featured dashboard for managing agent runs, provider tool approvals, policies, and the agent manifest. Enable it with:

```php
// config/ai-engine.php
'admin_ui' => [
    'enabled' => env('AI_ENGINE_ENABLE_ADMIN_UI', false),
    'route_prefix' => env('AI_ENGINE_ADMIN_PREFIX', 'ai-engine/admin'),
    'middleware' => ['web'],
    'access' => [
        'allowed_user_ids' => explode(',', env('AI_ENGINE_ADMIN_ALLOWED_USER_IDS', '')),
        'allowed_emails' => explode(',', env('AI_ENGINE_ADMIN_ALLOWED_EMAILS', '')),
        'allowed_ips' => explode(',', env('AI_ENGINE_ADMIN_ALLOWED_IPS', '')),
        'allow_any_authenticated' => env('AI_ENGINE_ADMIN_ALLOW_ANY_AUTHENTICATED', false),
        'allow_localhost' => env('AI_ENGINE_ADMIN_ALLOW_LOCALHOST', false),
    ],
],
```

Access is fail-closed: the middleware (`LaravelAIEngine\Http\Middleware\AdminAccessMiddleware`) checks authenticated user IDs/emails against an allowlist, or IP ranges if `allow_localhost` is set. Forged audit trails are prevented by reading authenticated identity only — the controller never trusts a client-supplied actor ID.

Routes are registered at `/ai-engine/admin` (configurable). The dashboard surfaces:

- **Agent Runs** (`/ai-engine/admin/agent-runs`): paginated list of agent runs with status filtering; detail view with steps, artifacts, and audit events. Actions include resume, retry (with custom message/reason), and cancel, all dispatched through `AgentRunRuntimeControlService`.
- **Provider Tools** (`/ai-engine/admin/provider-tools`): runs, approvals (pending/approved/rejected), and artifacts. Approve/reject actions update the `ai_provider_tool_approvals` table and optionally queue `ContinueProviderToolRunJob` asynchronously.
- **Policies** (`/ai-engine/admin/policies`): RAG prompt decision policy versions, creation, and activation via `RAGPromptPolicyService`.
- **Manifest Manager** (`/ai-engine/admin/manifest`): edit agent model configs, tools, and filters; import/export as JSON; scaffold new entries via `ai:scaffold` command.

Behind the scenes, three service classes handle admin operations:

```php
// Approve a provider tool (admin context)
$service = app(AdminProviderToolOperationsService::class);
$service->approve(
    ['approval_key' => $approvalKey, 'reason' => 'User approved', 'continue' => true],
    $approvalsService,
    $continuationsService,
    $jobs
);

// Resume or retry an agent run
$runService = app(AdminAgentRunOperationsService::class);
$runService->resume($runId, ['message' => 'Resume requested'], $controlService);

// Create or activate a policy
$policyService = app(AdminPolicyOperationsService::class);
$policyService->create([
    'policy_key' => 'decision',
    'name' => 'V2 Fallback',
    'status' => 'active',
    'rollout_percentage' => 50,
], $ragPolicyService, $policyConfig);
```

### Engine Catalog Service

The `LaravelAIEngine\Services\Catalog\EngineCatalogService` introspects all configured engines and models, merging config-defined models with database records (from `ai_models` table populated by `ModelToolConfig` discovery).

Three public methods expose the catalog:

```php
use LaravelAIEngine\Services\Catalog\EngineCatalogService;

$catalog = app(EngineCatalogService::class);

// All models, sorted by engine then model_id
$models = $catalog->flatModels();  // [['engine' => 'openai', 'model_id' => 'gpt-4o', ...], ...]

// Filter by engine
$openaiModels = $catalog->flatModels('openai');

// Engine summaries: name, configured status, default model, model count
$engines = $catalog->engines();
// [['engine' => 'openai', 'name' => 'OpenAI', 'configured' => true, 'model_count' => 11, ...], ...]

// Full nested structure: engines with nested model arrays
$nested = $catalog->catalog();
```

Models from config are marked `'source' => 'config'`; models from the database table are `'source' => 'database'`. Each entry includes `engine`, `provider`, `model_id`, `name`, `capabilities`, and boolean flags for streaming/vision/function-calling support.

The catalog is exposed via API routes (`/api/v1/ai`):

```bash
# All models, optionally filtered by engine
GET /api/v1/ai/models?engine=openai
# Response: { "success": true, "data": { "models": [...], "count": 3, "engine": "openai" } }

# Engine summaries
GET /api/v1/ai/engines
# Response: { "success": true, "data": { "engines": [...], "count": 14, "default_engine": "openai", "default_model": "gpt-4o" } }

# Nested engines + models
GET /api/v1/ai/engines-with-models
```

These routes are handled by `LaravelAIEngine\Http\Controllers\Api\EngineCatalogController`.

### SDK Services for Advanced Integrations

The SDK layer provides programmatic access to observability, file stores, vector stores, realtime sessions, and traces — all built on native Anthropic SDK primitives and compatible with `laravel/ai`.

#### Trace Recording

```php
use LaravelAIEngine\Services\SDK\TraceRecorderService;

$traces = app(TraceRecorderService::class);

// Record a span with metadata
$spanId = $traces->start('fetch_documents', ['query' => 'invoices']);
// ... do work ...
$traces->end($spanId, 'ok', ['count' => 5]);

// One-shot recording
$traces->record('validate_schema', ['schema' => 'invoice', 'valid' => true]);

// Flush and export
$allTraces = $traces->flush();
// Each trace: ['id', 'name', 'started_at', 'ended_at', 'duration_ms', 'status', 'metadata']
```

Exporters are dispatched via `LaravelAIEngine\Services\SDK\ObservabilityExporterService`, which reads `ai-engine.observability.exporters` (array of classes implementing `LaravelAIEngine\Contracts\ObservabilityExporter`):

```php
// config/ai-engine.php
'observability' => [
    'exporters' => [
        LaravelAIEngine\Services\SDK\Exporters\LogObservabilityExporter::class,
        LaravelAIEngine\Services\SDK\Exporters\OpenTelemetryObservabilityExporter::class,
        // Custom exporter
        \App\AI\Exporters\DatadogExporter::class,
    ],
],
```

Each exporter's `export(string $type, array $payload)` is called best-effort (exceptions caught).

#### File & Vector Stores

```php
use LaravelAIEngine\Services\SDK\FileStoreService;
use LaravelAIEngine\Services\SDK\VectorStoreService;

$files = app(FileStoreService::class);
$vectors = app(VectorStoreService::class);

// File store drivers for OpenAI and Anthropic
$fileStore = $files->driver('openai');
$upload = $fileStore->upload('/path/to/report.pdf');

// Vector store drivers
$vectorStore = $vectors->driver('openai');  // Uses OpenAI's hosted vector store
$vs = $vectorStore->create(['name' => 'docs']);
$vectorStore->addFiles($vs->id, [$upload['id']]);
```

Drivers live in `Services/SDK/{FileStores,VectorStores}/*Driver.php`; they wrap native SDK APIs.

## Federation (Nodes) & Security

Federation lets you scale AI across multiple Laravel applications. The optional `laravel-ai-engine-federation` package provides deterministic node routing where ownership metadata decides which app executes an operation—master node plans, child nodes execute their owned data and actions. Security is enforced through JWT-based inter-node authentication and fail-closed IDOR guards on both routing and data access.

### Enable Node Federation

Node federation is enabled by default but requires the `laravel-ai-engine-federation` package. Install it:

```bash
composer require laravel-ai-engine-federation
php artisan migrate
```

Configure your deployment topology in `.env`:

```env
# Master node orchestrates routing decisions
AI_ENGINE_NODES_ENABLED=true
AI_ENGINE_IS_MASTER=true
AI_ENGINE_NODE_NAME=MasterNode
AI_ENGINE_NODE_SLUG=master
AI_ENGINE_NODE_LABEL=master

# Shared secret for all nodes (use a strong random value)
AI_ENGINE_JWT_SECRET=your_shared_jwt_secret_min_32_chars
AI_ENGINE_SHARED_SECRET=${AI_ENGINE_JWT_SECRET}

# Child nodes point back to master
# AI_ENGINE_IS_MASTER=false
# AI_ENGINE_MASTER_URL=https://master.example.com
# AI_ENGINE_NODE_NAME=AccountingNode
# AI_ENGINE_NODE_SLUG=accounting
# AI_ENGINE_NODE_LABEL=child
```

All nodes **must use the same `AI_ENGINE_JWT_SECRET`**. Child nodes must be reachable at their configured `url` and expose the node API contract.

### Register Child Nodes on Master

Use a centralized manifest file to define your node topology. Create `config/ai-engine-nodes.json`:

```json
{
  "nodes": [
    {
      "name": "Accounting Service",
      "slug": "accounting",
      "type": "child",
      "url": "https://accounting.example.com",
      "capabilities": ["search", "actions", "rag"],
      "status": "active",
      "weight": 1
    },
    {
      "name": "CRM Service",
      "slug": "crm",
      "type": "child",
      "url": "https://crm.example.com",
      "capabilities": ["search", "actions"],
      "status": "active",
      "weight": 1
    }
  ]
}
```

Sync nodes from the manifest (with a dry-run first):

```bash
# Dry-run: show what would change
php artisan ai:nodes-sync --file=config/ai-engine-nodes.json

# Auto-fix minor issues (normalize slugs, etc.)
php artisan ai:nodes-sync --file=config/ai-engine-nodes.json --autofix

# Apply changes, ping each node, prune orphaned entries
php artisan ai:nodes-sync --file=config/ai-engine-nodes.json --apply --prune --ping --force
```

### Node API Contract

Each node (master and child) exposes:

- `GET /api/ai-engine/health` — public, returns `{status, ready, version, capabilities}`
- `GET /api/ai-engine/manifest` — public, returns collections, keywords, domains owned by this node
- `POST /api/ai-engine/search` — requires JWT/API-key auth, executes vector/RAG search
- `POST /api/ai-engine/chat` — requires JWT/API-key auth, routes a conversation turn
- `POST /api/ai-engine/tools/execute` — requires JWT/API-key auth, executes a tool

Routes are defined in the federation package's `routes/node-api.php` and protected by `NodeAuthMiddleware`.

### Node Authentication (JWT & API Keys)

Node-to-node requests use JWT tokens (preferred) or API keys. The `NodeAuthMiddleware` validates tokens in this order:

1. **JWT (preferred):** Extract from `Authorization: Bearer <token>` or `X-Node-Token` header. Validates signature and expiry using the shared `AI_ENGINE_JWT_SECRET`. Contains claims: `iss`, `sub` (node id), `node_slug`, `node_name`, `capabilities`, `type`, `exp`.

2. **Shared secret:** If `AI_ENGINE_SHARED_SECRET` matches the `Authorization` header exactly, authenticate without JWT validation (useful for internal-only deployments).

3. **API key:** Fall back to `X-API-Key` header or `?api_key` query param, validated against the node's stored key in the `ai_nodes` table.

```php
// In NodeAuthService::validateToken() - called by middleware
$nodeData = $this->authService->validateToken($token);
if ($nodeData) {
    // JWT is valid; check node status in database (master only)
    // or trust JWT claims (child nodes)
}
```

The middleware attaches the authenticated node to `$request->attributes->set('node', $node)` so routes can enforce ownership.

### Generate JWT Tokens for Child Nodes

When registering a node, generate a reusable JWT for that node to use when calling the master:

```php
use LaravelAIEngine\Services\Node\NodeAuthService;
use LaravelAIEngine\Models\AINode;

$node = AINode::where('slug', 'accounting')->firstOrFail();
$authService = app(NodeAuthService::class);
$token = $authService->generateToken($node, expiresIn: 3600);

// Child node uses this token in requests:
// Authorization: Bearer <token>
```

TTL defaults to `ai-engine.nodes.jwt.ttl` (1 hour). JWT library is auto-detected (firebase/php-jwt or tymon/jwt-auth).

### Routing Decisions: `route_to_node` Tool

The agent has a built-in `route_to_node` tool when the federation package is installed. It lets the AI planner explicitly route an entire turn to a remote node that owns a specific domain:

```json
{
  "action": "route_to_node",
  "resource_name": "accounting",
  "params": {
    "node": "accounting",
    "query": "what is my outstanding balance?"
  }
}
```

The tool is called only when:
- The request clearly belongs to a known remote node (determined by AI or routing config).
- The node is active and healthy (checked by `getActiveNodes()`).
- The AI planner's ownership metadata indicates remote ownership.

Under the hood, `RouteToNodeTool::handle()` calls `NodeSessionManager::routeToNode()`, which:

1. Looks up the node by slug or name.
2. Generates a JWT token for the call.
3. `POST`s to the node's `/api/ai-engine/chat` endpoint.
4. Returns the node's response or an error.

Routing is deterministic—no guessing. If ownership cannot be determined, the local RAG/tools are used.

### Node Configuration

Core settings in `config/ai-engine.php` under `nodes`:

```php
'nodes' => [
    // Enable/disable node federation
    'enabled' => env('AI_ENGINE_NODES_ENABLED', true),

    // Is this app a master or child?
    'is_master' => env('AI_ENGINE_IS_MASTER', true),

    // Master URL (for child nodes to call back)
    'master_url' => env('AI_ENGINE_MASTER_URL'),

    // Local node identity
    'local' => [
        'name' => env('AI_ENGINE_NODE_NAME', env('APP_NAME', 'Laravel')),
        'slug' => env('AI_ENGINE_NODE_SLUG'),
        'label' => env('AI_ENGINE_NODE_LABEL', env('AI_ENGINE_IS_MASTER') ? 'master' : 'client'),
        'aliases' => explode(',', env('AI_ENGINE_NODE_ALIASES', '')),
    ],

    // JWT configuration
    'jwt' => [
        'secret' => env('AI_ENGINE_JWT_SECRET', env('APP_KEY')),
        'library' => env('AI_ENGINE_JWT_LIBRARY', null), // auto-detect: 'firebase' or 'tymon'
        'algorithm' => env('AI_ENGINE_JWT_ALGORITHM', 'HS256'),
        'ttl' => env('AI_ENGINE_JWT_TTL', 3600), // 1 hour
        'issuer' => env('AI_ENGINE_JWT_ISSUER', env('APP_URL')),
        'audience' => env('AI_ENGINE_JWT_AUDIENCE', null),
    ],

    // Shared secret (fallback, used by all nodes)
    'shared_secret' => env('AI_ENGINE_SHARED_SECRET', env('AI_ENGINE_JWT_SECRET', env('APP_KEY'))),

    // Node capabilities advertised in health/manifest
    'capabilities' => ['search', 'actions', 'rag'],

    // Health checks & availability
    'health_check_interval' => env('AI_ENGINE_HEALTH_CHECK_INTERVAL', 300), // seconds
    'request_timeout' => env('AI_ENGINE_REQUEST_TIMEOUT', 30),
    'verify_ssl' => env('AI_ENGINE_VERIFY_SSL', true),

    // Circuit breaker (fail-fast on broken nodes)
    'circuit_breaker' => [
        'failure_threshold' => env('AI_ENGINE_CB_FAILURE_THRESHOLD', 5),
        'success_threshold' => env('AI_ENGINE_CB_SUCCESS_THRESHOLD', 2),
        'timeout' => env('AI_ENGINE_CB_TIMEOUT', 60),
    ],

    // Routing mode
    'search_mode' => env('AI_ENGINE_NODE_SEARCH_MODE', 'routing'), // 'routing' or 'federated'

    // Fallback on node failure
    'routing' => [
        'local_fallback_on_failure' => env('AI_ENGINE_ROUTE_LOCAL_FALLBACK_ON_FAILURE', false),
        'local_fallback_notice' => env('AI_ENGINE_ROUTE_LOCAL_FALLBACK_NOTICE', 'Remote node is unavailable...'),
    ],
],
```

### Security Model: Ownership & IDOR Prevention

Federation security rests on three pillars:

#### 1. Ownership Metadata (Deterministic Routing)

Each collection/model is explicitly mapped to a node in the manifest. The master checks ownership before routing:

```php
// NodeRegistryService::findNodeForCollection()
// Returns the node that owns a collection, or null if local
$node = app(NodeRegistryService::class)->findNodeForCollection(Invoice::class);
if ($node) {
    // Route to remote node
} else {
    // Use local RAG/tools
}
```

No ambiguity, no AI guessing. If a collection is not registered to any node, it's assumed to be local.

#### 2. JWT & Rate Limiting

All inter-node calls require a valid JWT or API key:

```php
// NodeAuthMiddleware validates every protected request
// Checks: signature, expiry, node status, rate limit
```

The `NodeAuthMiddleware` (in `routes/node-api.php`) rate-limits authenticated requests to `60 per minute` per node.

#### 3. Fail-Closed IDOR Guards

When retrieving data across nodes, the system enforces **user/tenant scope**:

- **Agent runs** (`GET /agent-runs`, etc.) are scoped by `user_id`; cross-user access returns HTTP 403.
- **Data queries** fail closed: a query with no scope (no `user_id`, `tenant_id`, or `workspace_id`) is blocked unless explicitly marked `public`.
- **Vector/RAG retrieval** respects scope columns (`user_id`, `tenant_id`, `workspace_id`) during filtering.

See [Security Guide](docs/security.mdx) for details on scope enforcement.

### Monitoring Node Health

Use the `ai:node-ping` and `ai:doctor` commands:

```bash
# Ping all nodes
php artisan ai:node-ping --all

# Check a single node
php artisan ai:node-ping --slug=accounting

# Full orchestrator diagnostics
php artisan ai:doctor --json
```

The output shows node availability, response time, and circuit-breaker status. If a node is down, the `circuit_breaker` opens and temporarily fails fast (before timeout).

### Optional: Federated Search

Set `AI_ENGINE_NODE_SEARCH_MODE=federated` to search all active nodes in parallel and merge results:

```env
AI_ENGINE_NODE_SEARCH_MODE=federated
AI_ENGINE_MERGE_STRATEGY=score  # or round_robin, node_priority, diversity, hybrid
AI_ENGINE_MERGE_DEDUPLICATION=true
```

In routing mode (default), the planner chooses one node based on ownership. In federated mode, all relevant nodes are queried and results are ranked/merged. Federated mode is more comprehensive but slower and more complex.

### Troubleshooting

- **Node not found during sync:** Check the node's public `/api/ai-engine/health` endpoint is reachable and returns `{status: healthy}`.
- **Authentication failures in logs:** Verify `AI_ENGINE_JWT_SECRET` matches on all nodes and no token is expired.
- **Stale nodes in database:** Use `ai:node-cleanup --status=error --days=0 --apply --force` to remove nodes that have not pinged in `N` days.
- **Circuit breaker open:** Wait `AI_ENGINE_CB_TIMEOUT` seconds or manually reset with `AINode::updateStatus($node, 'active')`.

## Specialized Providers (search, stock images, plagiarism)

Beyond chat/LLM engines, several drivers expose non-text capabilities. They are invoked the
same way as any engine — `AIEngineService::generate(new AIRequest(..., engine: '<name>'))` —
where the engine name is a `LaravelAIEngine\Enums\EngineEnum` case value.

| Capability | Engine name | Driver |
| --- | --- | --- |
| Web search | `serper`, `perplexity` | `SerperEngineDriver`, `PerplexityEngineDriver` |

| Image generation | `midjourney` | `MidjourneyEngineDriver` |
| Plagiarism check | `plagiarism_check` | `PlagiarismCheckEngineDriver` |
| Extra LLM providers | `azure`, `xai` (Grok), `deepseek` | `AzureEngineDriver`, `GrokEngineDriver`, `DeepSeekEngineDriver` |

```php
use LaravelAIEngine\Services\AIEngineService;
use LaravelAIEngine\DTOs\AIRequest;

// Web search
$results = app(AIEngineService::class)->generate(new AIRequest(

$photos = app(AIEngineService::class)->generate(new AIRequest(
    prompt: 'mountain landscape at dawn', engine: 'unsplash',
));
```

Each driver needs its credentials under `ai-engine.engines.<name>.api_key`. Run
`php artisan ai:doctor` to see which providers are configured. (Grok is the `Xai` engine case,

## Field Assistants

During structured collection (and guided forms), four built-in tools help a user fill a single
